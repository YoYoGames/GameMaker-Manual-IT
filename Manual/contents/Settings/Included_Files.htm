File inclusi

GameMaker non si limita solo a assets che si aggiunge nel Browser Asset, in quanto è anche possibile aggiungere file al progetto come Included Files in modo che possano essere accessibili durante l'esecuzione del gioco. Questi sono semplicemente dei file che sono inclusi nel vostro pacchetto di gioco, quindi potete farne ciò che volete.

I tipi di file da aggiungere dipendono da ciò che si vuole fare, ma cose come i file di default *.ini, sprites e immagini di sfondo, o fogli di calcolo sono comunemente usati come file inclusi.

Aggiungere i file inclusi

Il modo più semplice per aggiungere un file incluso è quello di trascinarlo dal tuo Explorer/Finder in IDE. Ti verrà chiesto di confermare se il file trascinato deve essere aggiunto a Included Files. Dopo aver premuto Yes, si aprirà la finestra Included Files, contenente il file appena aggiunto.

Finestra dei file inclusi

Aprite il menu Extras di Asset Browser (Fig. 1), per aprire manualmente la finestra Included Files (Fig. 2):

Questa finestra ti mostra solo i file inclusi che sono presenti nel tuo progetto. Per riorganizzare o rinominare i tuoi file, dovrai modificare la directory effettiva in cui questi file sono memorizzati.

Cliccando su Open In Explorer/Finder si aprirà la cartella datafiles nella directory del vostro progetto, che contiene tutti i file inclusi. Puoi aggiungere e rimuovere file in questa cartella come vuoi, e l'interfaccia in GameMaker si aggiornerà automaticamente per riflettere la struttura nella tua cartella datafiles.

Una volta aggiunti i file all'editor, potete fare doppio clic su per aprire le Proprietà del file:

Questa finestra mostra il nome del file, la sua dimensione su disco, e anche le piattaforme su cui dovrebbe essere esportato quando compilate/testate il vostro gioco. Avete dei pulsanti per selezionare "All" o "None", o potete scendere nella lista e selezionare/deselezionare gli obiettivi su cui volete che il file venga esportato.

Per ulteriori informazioni su come i file inclusi sono memorizzati con il tuo gioco e su come vi si accede, vedi File System.

IMPORTANTE Quando includete i file audio, dovreste assicurarvi che i file evitino nomi che si scontrano con lo streaming del suono assets. Per esempio, un suono asset chiamato "main_music" farà sì che un file incluso chiamato "main_music.ogg" diventi indefinito quando il progetto viene eseguito. Questo perché i file in streaming sono memorizzati anche al di fuori del bundle principale del progetto, quindi bisogna fare attenzione in queste situazioni.

Potete anche includere i file DLL (su Windows) o DyLib (su macOS) come file inclusi e poi creare le vostre funzioni personalizzate per gestirli usando external_define(). Queste possono poi essere usate chiamando la external_call() funzione.

NOTA L'uso dei file inclusi per creare funzionalità extra in questo modo è incompatibile con il sistema delle estensioni, poiché non è possibile usare le external_*() con i file aggiunti in un extension. Extensions assets ha già un metodo per aggiungere costanti e funzioni.
