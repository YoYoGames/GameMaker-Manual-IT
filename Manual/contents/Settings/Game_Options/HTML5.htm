Opzioni di gioco HTML5

Questa sezione delinea le diverse opzioni a vostra disposizione che controllano come i vostri progetti di gioco HTML5 saranno compilati. Le diverse sezioni sono:

Generale

In questa sezione hai dei campi per riempire le seguenti informazioni sul tuo gioco:

  • Browser Title: qui puoi dare un testo da usare per la barra del titolo della scheda del tuo gioco nel browser.
  • Versione: Il numero di versione del tuo gioco.
  • Nome della cartella: Il nome della cartella in cui saranno memorizzati i file del tuo gioco HTML5 (l'impostazione predefinita è "html5game").
  • Nome di uscita: Questo è il nome del file HTML di output e può essere cambiato in qualcosa di diverso dal predefinito "index.html".

Sotto i campi delle informazioni di gioco hai anche le seguenti opzioni:

  • Uscita di debug alla console: Questo invierà tutto l'output di debug alla console JS del vostro browser. Questo è disattivato per impostazione predefinita.
  • Visualizza cursore: Questo disabiliterà il cursore del mouse quando è sopra la tela di gioco. Questo è attivo per impostazione predefinita.
  • Display "Running Outside Server" Alert: Se stai testando un gioco localmente (cioè: cliccando sul file indice dal tuo computer piuttosto che su un server - non raccomandato), allora normalmente ottieni un avviso che ti informa che le cose non funzioneranno come ti aspetti a causa della sicurezza del browser per i file locali. Puoi sopprimere questo avviso deselezionando questa opzione. È attiva per impostazione predefinita.

Sotto le impostazioni generali, avete le opzioni avanzate , che sono:

  • File incluso come index.html: Questo serve per aggiungere un file "index.html" personalizzato da usare al posto di quello predefinito che GameMaker genera di default. Questo può essere molto utile se avete implementato il design di un sito web o volete includere certe caratteristiche in tutte le vostre build HTML5. Il file da utilizzare deve essere incluso come file incluso nel progetto di gioco.
  • Estensione barra di caricamento: Questo cambierà la barra di caricamento predefinita con una che è stata aggiunta al vostro progetto GameMaker come estensione.
  • Prepend output .js: qui puoi prependere l'output di *.js con qualsiasi JavaScript extra necessario per il tuo gioco. Per esempio, un'applicazione di Chrome Store potrebbe richiedere di aggiungere window.localStorage = undefined; all'output JS, per cui useresti questa opzione.

Analytics

Questa sezione ti permette di aggiungere analisi al tuo gioco, consentendoti di utilizzare alcuni prodotti di terze parti per monitorare il tuo gioco e le sue vendite o giocate. Puoi specificare se utilizzare Flurry Analytics o Google Analytics per monitorare il tuo gioco. A causa della natura di queste funzioni, puoi averne solo una attiva alla volta, e una volta che hai contrassegnato l'opzione Abilita Flurry Analytics o Abilita Google Analytics dovresti anche aggiungere l'ID unico che ti è stato assegnato nell'area appropriata.

Per informazioni sulle funzioni che puoi usare per inviare dati analitici, vedi qui.

Grafica

Qui puoi cambiare i seguenti dettagli relativi a come il tuo gioco verrà visualizzato (nota che non tutti i browser visualizzeranno il tuo gioco allo stesso modo, quindi prova queste opzioni su quanti più browser possibile per assicurarti che abbiano l'effetto desiderato):

  • Consenti il passaggio a schermo intero: Selezionando questo permetterà all'utente di commutare il gioco dalla modalità a finestra a quella a schermo intero usando / + "F" su Mac e F10 su Windows. Questo è attivo per impostazione predefinita.
  • Interpola i colori tra i pixel: Attiva l'interpolazione lineare, che fondamentalmente "liscia" i pixel. Per una grafica a pixel nitidi, dovrebbe essere disattivata, ma se hai delle belle miscele alfa e una grafica dai bordi smussati è meglio lasciarla attivata. Questo è attivo di default.
  • Centra il gioco nel browser: Selezionando questa opzione, la schermata di gioco verrà centrata automaticamente all'interno del browser che esegue il gioco. Questo è disattivato per impostazione predefinita.
  • Use Built In Particles: Questa opzione esporterà il sprites richiesto per il sistema di particelle integrato insieme al tuo gioco. Questi sprites sono memorizzati in pagine individuali texture, il che significa che non sono il metodo più efficiente per generare particelle per questa piattaforma, quindi si raccomanda che (a meno che non si usi WebGL) si deselezioni questa opzione e si usi la propria particella sprites che sarà memorizzata nelle pagine principali texture tutte insieme per il tuo gioco. Questo è attivo per impostazione predefinita.
  • Use Built In Font: Questa opzione esporterà il GameMaker costruito in font insieme al resto dei file di gioco. Questo aumenterà leggermente la dimensione delle pagine di texture e richiederà una piccola quantità di memoria extra, quindi se il vostro gioco non usa l'Arial incorporato font, questo dovrebbe essere disabilitato. È attivo per impostazione predefinita.
  • WebGL: Scegli se switch disattivare il supporto WebGL, renderlo opzionale (rilevamento automatico), o renderlo un requisito per il tuo gioco. Va notato che per testare le prestazioni in tutte le circostanze, si dovrebbe avere questo come disabilitato, poi normalmente impostato su auto-rilevamento per la pubblicazione. Farlo diventare un requisito può ridurre considerevolmente la compatibilità del tuo gioco con diversi tipi di browser. Questo è impostato su rilevamento automatico per impostazione predefinita.
  • Scalatura: Il tuo gioco può essere configurato per scalare automaticamente la tela di disegno per mantenere le proporzioni all'interno del browser, oppure puoi scegliere di farlo girare a schermo intero. L'opzione di scala completa non farà il gioco a schermo intero nel browser, ma piuttosto allungherà ciò che viene disegnato per adattarlo alla dimensione della tela, come definito dal primo room del gioco. Questo è impostato per mantenere il rapporto di aspetto per impostazione predefinita.

Infine c'è l'opzione per impostare la dimensione della pagina della texture. La dimensione predefinita (e più compatibile) è 2048x2048, ma si può scegliere tra 256x256 e 8192x8192. C'è anche un pulsante contrassegnato da Anteprima che genererà le pagine texture per questa piattaforma e poi aprirà una finestra in modo che tu possa vedere come appaiono. Questo può essere molto utile se vuoi vedere come sono strutturate le pagine texture e per evitare di avere pagine texture più grandi (o più piccole) del necessario. Per maggiori informazioni sulle pagine di texture, si veda qui.

NOTA: Sii consapevole che più grande è la dimensione della pagina texture, meno compatibile sarà il tuo gioco con diversi browser e dispositivi.

Immagini

Questa sezione è dove puoi definire la Favicon e lo Splash Screen. L'icona dovrebbe essere creata come file *.ico e può essere di 16x16, 32x32, 48x48, o 64x64 pixel di dimensione, e 8-bit, 24-bit, o 32-bit di profondità di colore. Lo splash screen dovrebbe avere le dimensioni della tela che conterrà il tuo gioco e sovrascriverà qualsiasi barra di caricamento se spunti l'opzione Use Splash Screen (cioè: puoi avere una barra di caricamento O uno splash screen, ma non entrambi). Immagini più piccole o più grandi sono permesse, ma saranno ridimensionate per adattarsi. Lo splash screen dovrebbe essere un'immagine *.png.

Vale la pena notare che GameMaker ha uno strumento Project Image Generator che può essere utilizzato per creare automaticamente tutte le immagini necessarie per tutte le diverse piattaforme di destinazione per le quali il gioco viene compilato. Se usate questo strumento, dovreste rivedere le immagini create per assicurarvi che siano quelle richieste.

Sociale

La scheda sociale è usata per impostare la comunicazione con Facebook nel tuo gioco. Se hai bisogno delle funzioni di Facebook, devi cliccare sulla casella "Usa Facebook" e poi fornire l'ID dell'applicazione Facebook e il nome di visualizzazione dell'applicazione Facebook. Si noti che tutte le funzionalità di Facebook sono accessibili tramite un extension, e quando si seleziona per la prima volta la casella Usa Facebook verrà richiesto di scaricare e installare il Facebook extension dal Marketplace (potrebbe essere necessario accedere al proprio account YoYo prima di poterlo scaricare).
