Opzioni di gioco Android

Questa sezione illustra le diverse opzioni disponibili per controllare la compilazione dei progetti di gioco di Android. Le diverse sezioni sono:

Generale

La prima parte della sezione Generale è dedicata alle Informazioni sul prodotto e alle Impostazioni di creazione. È necessario compilare le informazioni sul prodotto, che comprendono il nome visualizzato del progetto e le informazioni sul pacchetto, che consistono in dominio, azienda e prodotto. Le informazioni sul pacchetto possono essere composte solo da lettere dalla A alla Z e da numeri da 0 a 9, senza simboli o altri caratteri speciali e senza caratteri maiuscoli. Questi dati verranno utilizzati per generare un ID pacchetto nel formato "com.company.myappname".

È quindi necessario impostare gli strumenti di compilazione e le versioni SDK Target, Minimum e Compile.

NOTA: Esiste anche un'opzione per la versione "Support Library" da utilizzare. A partire dalla versione 2.2.4 di GameMaker, questa opzione non è più necessaria e i valori utilizzati saranno ignorati. L'impostazione viene mantenuta per l'uso con progetti legacy che richiedono un runtime precedente alla 2.2.4.

Gli strumenti di compilazione (e la libreria di supporto, se richiesta) vengono utilizzati quando GameMaker costruisce il gioco per il target Android e devono essere impostati sui valori appropriati in base agli strumenti installati con Android Studio.

  • L'SDK di destinazione indica che l'applicazione è stata testata sulla versione specificata (fino a quella inclusa). Questo serve semplicemente a dare al sistema operativo Android un'idea di come dovrebbe gestire l'applicazione in termini di funzionalità del sistema operativo. Per tutti gli scopi pratici, la maggior parte delle applicazioni vorrà impostare Target SDK sull'ultima versione rilasciata dell'API (come usato dall'impostazione Compile SDK). In questo modo, l'app verrà visualizzata al meglio sui dispositivi Android più recenti. Si noti che da novembre 2019 Google ha reso il livello API 28 l'obiettivo richiesto.
  • La versione minima dell'SDK è il livello minimo di API che consente di eseguire il progetto. A partire da novembre 2019 Google ha reso il livello API 16 il minimo assoluto richiesto, ma se si aggiunge extensions potrebbe essere necessario un livello API minimo più alto.
  • La versione dell'SDK di compilazione è la versione dell'API con cui viene compilato il progetto. Ciò significa che è possibile utilizzare le funzioni dell'API Android incluse in quella versione dell'API. Se si cerca di utilizzare le funzioni dell'API 16, ad esempio, ma si imposta Compile SDK 15, si otterrà un errore di compilazione. Tuttavia, se si imposta Compile SDK su 28, è possibile eseguire l'applicazione su un dispositivo API 26 (e anche su tutte le altre versioni precedenti).

È possibile selezionare una delle impostazioni predefinite facendo clic sul pulsante Prepara i valori SDK al livello API scelto, che aprirà un elenco di API diverse tra cui scegliere. Selezionando una di esse, tutti i file SDK verranno popolati con i valori appropriati, che potranno essere utilizzati o modificati a seconda delle esigenze.

NOTA: Per compilare correttamente i progetti, è necessario aver installato le API appropriate in Android SDK Manager o tramite Android Studio.

La seconda parte della sezione Generale riguarda le architetture della CPU per cui costruire, con le seguenti opzioni disponibili (si possono selezionare tutte o solo quelle necessarie, ma si noti che ognuna di esse aumenterà la dimensione finale dell'apk compilato):

  • Costruire per ARMV5 (spuntato per impostazione predefinita)
  • Costruire per ARMV7 (Selezionato per impostazione predefinita)
  • Costruire per x86 (selezionato per impostazione predefinita)
  • Costruire per Mips (selezionato per impostazione predefinita)
  • Costruire per ARM64
  • Costruire per x86_64

Dopo averla impostata, è possibile selezionare i diversi orientamenti su cui il gioco può essere eseguito per ottenere la migliore esperienza utente. Le opzioni disponibili sono:

  • Ritratto
  • Ritratto capovolto
  • Paesaggio
  • Paesaggio capovolto

Infine, sono disponibili le seguenti opzioni varie:

  • Abilita il supporto Bluetooth/iCade: Aggiunge il supporto per i gamepad Bluetooth o iCade al progetto di gioco. L'opzione è attiva per impostazione predefinita.
  • Esegui analisi codice Lint: Quando questa opzione è attivata, gli strumenti di compilazione eseguono un'analisi del codice lint sul progetto durante la compilazione, fornendo un controllo più approfondito degli errori e mostrando più informazioni nella finestra di output durante la compilazione del progetto (verrà creato anche un file LOG, con l'indirizzo path al file mostrato alla fine della compilazione nella finestra di output). Si noti che non tutto ciò che viene segnalato come errore dall'analisi di lint è necessariamente problematico per il gioco, e l'attivazione di questa opzione può significare che il gioco non viene più compilato, anche se quando è disattivata viene compilato ed eseguito correttamente. Essenzialmente questo è solo per debugging e l'opzione è disattivata per impostazione predefinita.
  • Posizione di installazione: Qui è possibile impostare il percorso di installazione predefinito per il gioco. Si può impostare su Automatico, nel qual caso il gioco verrà installato nella posizione scelta dall'utente sul proprio dispositivo, oppure su Preferito esterno, che cercherà sempre di installare il gioco nella memoria esterna del dispositivo.

È inoltre possibile impostare il margine di riposo di Android. Questa opzione è legata alla riduzione dello stuttering quando il gioco viene eseguito su sistemi specifici. In pratica, se il gioco viene eseguito più velocemente della velocità di room, GameMaker "dormirà" per il tempo rimanente, ma questo sonno può essere piuttosto impreciso e spesso si finisce per dormire più a lungo del necessario, facendo sì che i fotogrammi richiedano più tempo e causando lo stuttering. Per ovviare a questo problema, si può dormire per un tempo inferiore e poi rimanere in uno stretto loop per il resto del tempo, in modo da renderlo più accurato - anche se il problema di rimanere in un loop è che fa aumentare l'utilizzo della CPU, che a sua volta può far aumentare la temperatura della CPU e far consumare più batteria al dispositivo. Per impostazione predefinita, questo valore è impostato su 4 e nel 99,99% dei casi andrà bene, ma per i dispositivi di fascia bassa o per i dispositivi con molti processi in background in esecuzione questa potrebbe non essere la soluzione ideale e potrebbe essere necessario un valore da 5 a 10. Si noti tuttavia che si tratta di una configurazione specifica del dispositivo e che ciò che funziona sul proprio dispositivo potrebbe non essere appropriato per un altro utente.

Icone adattive

Android 8.0 (livello API 26) ha introdotto le icone del launcher adattive per i giochi, che possono essere visualizzate in una varietà di forme su diversi modelli di dispositivi. Per questo motivo, è necessario fornire un certo numero di immagini di primo piano e di sfondo per visualizzare le icone a dimensioni diverse. Le dimensioni delle icone sono:

  • 81x81 pixel (LDPI)
  • 108x108 pixel MDPI)
  • 162x162 pixel (HDPI)
  • 216x216 pixel (XHDPI)
  • 324x324 pixel (XXHDPI)
  • 432x432 pixel (XXXHDPI)

È possibile fornire singolarmente ogni immagine di sfondo e di fondo, oppure scegliere di generare le icone da un file. Ciò significa che GameMaker genererà le immagini delle icone richieste da un file di base (XXXHDPI). È necessario fornire un'immagine per ogni primo piano e sfondo, quindi fare clic sul pulsante Generate e le icone verranno create. Le immagini devono essere tutte in formato *.png.

È importante capire che le icone adattive possono essere mascherate su qualsiasi dispositivo e quindi è possibile vedere come appariranno con queste maschere dal menu Anteprima icone adattive:

Si noti che questa è semplicemente un'anteprima di come potrebbe apparire l'icona con la maschera selezionata e l'impostazione non influisce sul modo in cui le icone vengono esportate e visualizzate, poiché ciò dipende dal dispositivo e dal sistema operativo su cui verrà eseguito il gioco. Si noti inoltre che queste icone non vengono create per voi come parte della generazione di immagini effettuata quando si usa lo strumento Generatore di immagini del progetto.

Grafica

Qui è possibile modificare i seguenti dettagli relativi alla visualizzazione della partita, con le seguenti opzioni:

  • Interpola i colori tra i pixel: Attiva l'interpolazione lineare, che in pratica "smussa" i pixel. Per una grafica di pixel nitida, dovrebbe essere disattivata, ma se si hanno delle belle miscele alfa e una grafica dai bordi levigati è meglio lasciarla attivata. Questa opzione è attiva per impostazione predefinita.
  • Profondità colore schermo: Si può impostare la profondità di colore per il rendering a 16 o 24 bit. Questo influisce sulla compatibilità con alcuni vecchi dispositivi se è impostato a 24 bit e aumenta la memoria ashmem necessaria (questa è la memoria condivisa sui dispositivi AmazonFire).
  • Supporto dispositivo: Questa opzione stabilisce se supportare solo i dispositivi che hanno una GPU dedicata o tutti i dispositivi. In pratica, se il vostro gioco ha funzioni di disegno avanzate come la fusione alfa, la fusione additiva, le superfici, ecc... dovreste limitarlo ai dispositivi dotati di GPU, altrimenti rischiate di ricevere commenti negativi e una valutazione insufficiente per il vostro gioco a causa di errori grafici o crash su dispositivi più vecchi.
  • Scala: Qui si può scegliere di scalare l'area di disegno mantenendo il rapporto di aspetto all'interno dello schermo del dispositivo, aggiungendo un "padding" intorno ai bordi per adattarlo allo schermo, oppure si può scegliere di avere l'area di disegno allungata per adattarla allo schermo del dispositivo.

Infine, è possibile impostare la dimensione della pagina della texture. La dimensione predefinita (e più compatibile) è 2048x2048, ma si può scegliere tra 256x256 e 8192x8192. C'è anche un pulsante con la dicitura Anteprima che genera le pagine di texture per questa piattaforma e apre una finestra per vederne l'aspetto. Questo può essere molto utile per vedere come sono strutturate le pagine texture e per evitare di avere pagine texture più grandi (o più piccole) del necessario. Per ulteriori informazioni sulle pagine di texture, vedere qui.

NOTA: Tenere presente che più grandi sono le dimensioni della pagina texture, meno compatibile sarà il gioco con i diversi browser e dispositivi.

Icone

Questa sezione consente di aggiungere le varie immagini delle icone di cui il progetto avrà bisogno per le diverse pagine del negozio e per i dispositivi. Tutte le immagini delle icone devono essere create come file *.png a 24 bit delle dimensioni appropriate indicate.

Vale la pena notare che GameMaker dispone di uno strumento Generatore di immagini di progetto che può essere utilizzato per creare automaticamente tutte le immagini necessarie per tutte le diverse piattaforme di destinazione per cui il gioco viene compilato. Se si utilizza questo strumento, è necessario rivedere le immagini create per assicurarsi che siano quelle richieste. Lo strumento genera sia icone che immagini (vedere la sezione seguente).

Immagini

Qui è possibile impostare il colore del bordo che verrà utilizzato come sfondo per le schermate splash che non sono impostate per riempire l'intero schermo. È inoltre possibile aggiungere al file finale del gioco delle schermate di emergenza, sia in modalità orizzontale che verticale , che verranno visualizzate durante il caricamento del gioco sul dispositivo. Questa schermata deve essere un file *.png a 24 bit e si consiglia di avere le stesse dimensioni della prima room (o vista) del gioco. Se si desidera che lo splash screen venga mostrato per un tempo specifico, è possibile impostarlo da 0 a 10 secondi (il tempo predefinito di 0 significa che verrà mostrato solo per la durata del caricamento di asset ). È inoltre possibile impostare la modalità di visualizzazione della schermata iniziale dalle seguenti opzioni:

  • Mantieni rapporto d'aspetto: Mantiene il rapporto d'aspetto dell'immagine rispetto allo schermo del dispositivo, aggiungendo un "padding" intorno ai bordi dove necessario.
  • Ritaglia per adattarsi: Consente di mantenere l'immagine 1:1 con lo schermo del dispositivo e di ritagliare (o imbottire) i bordi come richiesto.
  • Allunga: L'immagine viene allungata per riempire lo schermo del dispositivo (e può distorcere l'immagine visualizzata).

Infine, è possibile impostare l'immagine del banner per il gioco sui dispositivi TV Android, che è anche a 24 bit *.png.

Imballaggio

Una delle caratteristiche dell'utilizzo di Google Play per i giochi di Android è la possibilità di concedere licenze per i giochi. Ciò significa che i vostri giochi saranno controllati da Google in modo che solo le persone che li hanno scaricati attraverso il loro mercato potranno giocarci, evitando così che vengano distribuite copie illegali del vostro prodotto.

Per attivare questa funzione nel gioco, è necessario copiare la propria chiave pubblica nella casella denominata Google Licencing Public Key. Questa licenza può essere trovata accedendo alla pagina degli sviluppatori di Google e selezionando il gioco che si desidera modificare. Una volta nella pagina del profilo del gioco, scorrere fino alla sezione "Servizi e API" e lì si dovrebbe trovare la chiave pubblica necessaria per attivare questa funzionalità.

Una volta attivato, se qualcuno che non è autorizzato a possedere il gioco tenta di giocarci, verrà visualizzato un messaggio a comparsa che informerà che la licenza non può essere trovata sul dispositivo e darà la possibilità di riprovare e testare di nuovo o di uscire dal gioco.

Un'altra caratteristica delle licenze è che il gioco può utilizzare le espansioni APK. Si tratta di un modo per aggirare la dimensione massima di 50 MB imposta da Google Play (e solo da Google Play). Il gioco verrà creato in due parti: un piccolo file *.apk e un file *.zip più grande che conterrà tutti i giochi assets, entrambi caricati sullo store. Quando l'utente esegue il gioco per la prima volta, lo zip viene scaricato e scompattato. Ulteriori dettagli su questa opzione sono disponibili qui.

È inoltre possibile utilizzare Google Dynamic Asset Delivery per creare bundle di app grandi fino a 1 GB, utilizzando l'opzione di esportazione *.aab. Questo pacchetto contiene tutti i giochi assets come "install-time" assets che vengono installati insieme all'applicazione stessa. Ulteriori dettagli su questa funzione sono disponibili qui.

Infine, è possibile preparare il pacchetto finale dell'applicazione come gioco per Android TV (selezionare questa opzione solo se il gioco è conforme alle linee guida di Android TV).

Permessi

Da questa scheda è possibile modificare alcuni dei permessi che il gioco può richiedere all'interno del Manifesto Android. GameMaker assegnerà automaticamente i permessi in base alle funzioni utilizzate dal gioco; tuttavia, è possibile che il gioco richieda un permesso che non è stato assegnato correttamente (ad esempio, quando si utilizza il sito extensions); in tal caso, selezionare la casella del permesso richiesto.

L'elenco seguente illustra i permessi disponibili:

  • WRITE_EXTERNAL_STORAGE: Permette al gioco di scrivere sulla memoria esterna.
  • READ_PHONE_STATE: Consente l'accesso in sola lettura allo stato del telefono.
  • ACCESS_NETWORK_STATE: Consente al gioco di accedere alle informazioni sulle reti.
  • INTERNET: Permette al gioco di aprire i socket di rete.
  • BLUETOOTH: Permette al gioco di connettersi ai dispositivi Bluetooth accoppiati.
  • RECORD_AUDIO: Permette al gioco di registrare l'audio proveniente dal microfono.

NOTA: se non siete sicuri di averne bisogno, probabilmente dovreste lasciarli deselezionati per impostazione predefinita e lasciare che GameMaker si occupi dei permessi per il vostro gioco.

La sezione Permessi di Android contiene anche un'area in cui è possibile iniettare i permessi nei tag dell'applicazione Android in Androidmanifest.xml. L'elenco completo dei tag disponibili si trova nella documentazione di Android:

  • Sviluppatore Android: Tag dell'applicazione

Vale la pena notare che per le API 28 e successive, non sarà più possibile utilizzare chiamate web esterne utilizzando "http", ma sarà necessario utilizzare "https", a meno che non si aggiunga il seguente tag:

android:usesCleartextTraffic="true"

Sociale

Questa sezione consente di attivare diverse funzioni sociali nel gioco Android. È possibile scegliere le seguenti opzioni:

  • Google - Abilita il salvataggio in cloud: Selezionando questa opzione si abilita il salvataggio in cloud di Google e viene richiesto di scaricare e installare il relativo extension dal Marketplace. Vedere la sezione sul salvataggio in cloud nella pagina Funzioni asincrone.
  • Google - ID servizi Google: Consente di inserire l'ID dei servizi Google della vostra applicazione.
