File batch / script del compilatore

Il compilatore GameMaker consente di inserire i file Batch (Windows) o Shell scripts (su macOS e Linux) nella directory del progetto e nella directory di ogni extension del progetto.

Posizionamento

Un file script può essere collocato nella radice della cartella del progetto, accanto al file .yyp del progetto.

Un file script può anche essere collocato nella directory di un file extension, che può avere il seguente aspetto: <project_directory>/extensions/<extension_folder>/.

Le versioni extension di un file script vengono eseguite per prime e la versione principale viene eseguita per ultima. L'ordine di esecuzione di extensions non può essere controllato, quindi lo sviluppatore di extension deve assicurarsi che scripts di un'estensione non faccia affidamento su scripts di un'altra estensione.

Esecuzione

I file di script con nomi specifici, collocati in una qualsiasi delle directory descritte sopra, vengono eseguiti in vari punti del processo di compilazione:

Nome del file batch di Windows macOS/Linux Shell Script Nome Descrizione
pre_run_step.bat pre_run_step.sh Viene eseguito prima dell'inizio della distribuzione del gioco.
post_run_step.bat post_run_step.sh Viene eseguito quando il gioco è preparato e pronto per essere eseguito; dopo l'esecuzione dello script, il gioco viene avviato.
pre_build_step.bat pre_build_step.sh Viene eseguito prima che venga avviato il compilatore asset per la creazione del gioco.
post_build_step.bat post_build_step.sh Viene eseguito dopo che il compilatore asset è stato avviato per la creazione del gioco.
pre_package_step.bat pre_package_step.sh Questa operazione viene eseguita prima della fase finale di impacchettamento, quando tutti i file sono pronti ma il file ZIP finale o il pacchetto di archiviazione sta per essere creato.
post_package_step.bat post_package_step.sh Questa operazione viene eseguita dopo il completamento della fase finale di confezionamento e la preparazione del file ZIP o del pacchetto di archiviazione finale.
post_textures.bat post_textures.sh Viene eseguito dopo che è stato generato il sito textures per il gioco. È possibile utilizzare questo passaggio per ottimizzare manualmente il gioco textures.

Questo script riceve una variabile d'ambiente chiamata TexturesDir, che memorizza il path sito generato textures.

Questi file script sono supportati durante la compilazione su tutte le piattaforme di destinazione.

Disabilitare Scripts per un'estensione

Se si utilizzano più extensions che eseguono scripts, ad esempio Steamworks e GDK, si dovrà disabilitare Steamworks quando si esporta in GDK e viceversa.

Per questo, vedere: Come disattivare Extensions?
