Preferenze del plugin

Le preferenze dei plugin sono usate per controllare qualsiasi plugin extra che è stato aggiunto a GameMaker IDE. Per impostazione predefinita questo ha una sezione dedicata al plugin Source Control per integrare i progetti GameMaker con Git.

Controllo delle fonti (Git)

Questa sezione si occupa delle preferenze che sono necessarie per far funzionare il plugin Git SCM con GameMaker.

  • Percorso dell'eseguibile Git: Questa è la posizione dell'eseguibile di Git che GameMaker usa per tutti i comandi di Source Control. Come mostrato nell'immagine qui sopra, questo si trova nella cartella cmd della tua installazione Git.

Sotto questo puoi impostare lo strumento Merge e lo strumento Diff per Source Control con queste impostazioni(opzionale):

  • Merge (Tool location): qui si specifica il file completo path per lo strumento di merge Git.
  • Unisci (Opzioni strumento): In questo campo puoi aggiungere qualsiasi comando extra da eseguire ogni volta che usi lo strumento di unione.
  • Diff (posizione dello strumento): Qui si specifica il file completo path per lo strumento Git diff.
  • Diff (Opzioni strumento): In questo campo puoi aggiungere qualsiasi comando extra da eseguire ogni volta che usi lo strumento diff.

Finally there are two more options that affect how Git is set up in new projects: 

  • Add skeleton .git defaults to new projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to new projects that you create. It is enabled by default.
  • Add skeleton .git defaults to imported projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to projects that you import from a .yyz file. It is enabled by default.
