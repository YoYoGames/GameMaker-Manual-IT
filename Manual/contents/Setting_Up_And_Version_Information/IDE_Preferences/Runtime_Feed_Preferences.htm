Preferenze di alimentazione runtime

Le preferenze di Runtime Feeds sono utilizzate per visualizzare e modificare il Runtime attualmente installato, nonché il runtime in uso corrente. GameMaker può avere diversi strumenti runtime installati in qualsiasi momento ed è possibile impostare quale utilizzare da questa preferenza. Idealmente si desidera utilizzare il più aggiornato runtime, ma a volte può avere un problema, o può essere richiesto di utilizzare uno più vecchio per la comparabilità, ecc... e quindi è possibile cambiare gli strumenti runtime che GameMaker utilizza per costruire il vostro gioco qui.

La sezione iniziale di questa preferenza è relativa alla provenienza degli strumenti di runtime e richiede l'inserimento di un URL di un feed RSS. Questo feed verrà poi scansionato per gli aggiornamenti degli strumenti di runtime che possono poi essere scaricati e installati. Questa sezione mostrerà l'URL del feed principale runtime per impostazione predefinita, ma è possibile fare clic sul pulsante Aggiungi per creare un nuovo feed Runtime dove si dà il nome e l'URL per il feed RSS, e quindi fare clic su Applica per dire a GameMaker di accedervi. Sarà necessario chiudere e riaprire la finestra delle Preferenze perché il nuovo feed venga mostrato.

Una volta che un feed Runtime è stato creato sarà elencato in questa sezione come sottocategoria, e si dovrebbe sempre avere almeno un feed elencato (di default questo sarebbe il feed Master ). Le preferenze di Runtime Feed sono usate per selezionare, installare e disinstallare i diversi strumenti di runtime. Nella parte superiore hai l'attuale strumento runtime in uso, la data in cui è stato creato e la posizione in cui è memorizzato sul disco rigido. Sotto questi dettagli puoi trovare una lista di tutti i runtimes disponibili, e puoi sceglierne uno qualsiasi da questa lista per farlo diventare il runtime "in esecuzione" usato per compilare i tuoi progetti.

Per selezionare un diverso runtime da usare, basta trovarlo nella lista e poi fare doppio clic su . Se il runtime non è stato scaricato e installato, vi verrà richiesto di farlo, e una volta installato vi verrà chiesto di chiudere e riavviare il IDE in modo che il nuovo runtime possa essere inizializzato.

È possibile aggiornare la lista runtime in qualsiasi momento dal dato feed RSS cliccando il pulsante di aggiornamento , e si può pulire il disco rigido di vecchi runtimes cliccando il pulsante "Clean Runtimes Now".
