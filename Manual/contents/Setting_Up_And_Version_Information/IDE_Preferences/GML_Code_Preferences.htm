Preferenze delcodice GML

Le preferenze del codice GML sono usate per definire come alcune delle caratteristiche del linguaggio per la codifica saranno visualizzate in IDE di GameMaker. Le opzioni per GML Code sono le seguenti:

  • Visualizzare i tipi di argomenti nella barra di stato: Questa opzione si riferisce a quando state usando il vostro scripts personalizzato e avete usato la notazione JSDoc per identificare i diversi argomenti, ecc... (vedi qui per maggiori informazioni). Quando questa opzione è selezionata, e hai specificato i tipi di argomenti in script, allora saranno mostrati nella barra di stato in fondo all'editor, racchiusi in {}. Questa opzione è disattivata per impostazione predefinita.
  • Mostra le descrizioni degli argomenti nella barra di stato: Se questa opzione è selezionata allora qualsiasi descrizione che avete dato al vostro scripts personalizzato usando la notazione JSDoc sarà mostrata nella barra di stato in fondo alla finestra dell'editor. Per impostazione predefinita questa opzione è disattivata.
  • Mostra l'ortografia USA per le funzioni di sistema: Per default GameMaker usa l'ortografia inglese GB per le funzioni, per esempio draw_text_colour(), tuttavia questo potrebbe non essere esattamente quello a cui siete abituati e quindi potete switch passare all'ortografia USA usando questa opzione, in modo che le funzioni diventino draw_text_color(), per esempio. Questa opzione è disattivata per impostazione predefinita.
  • Numero massimo di errori di sintassi da visualizzare: Questo valore imposta il numero massimo di errori di sintassi che saranno visualizzati in qualsiasi momento. Se trovate che IDE è in ritardo, potete provare a ridurre questo valore. Questo è impostato a 50 per impostazione predefinita.
  • Abilita gli avvisi sui riferimenti alle variabili: Quando abilitato, questo segnalerà qualsiasi potenziale problema con le variabili, come quando una è stata definita da qualche parte ma non è effettivamente usata nel codice. Quando è disabilitato, questi avvertimenti non saranno più mostrati. L'impostazione predefinita è che questo sia abilitato.
  • Abilita la piegatura delle dichiarazioni come regioni: Quando è abilitato, le parentesi nel vostro codice possono essere piegate come regioni. Questo è abilitato di default.
