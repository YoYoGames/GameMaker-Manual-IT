Preferenze del mercato

Le Preferenze del Mercato servono per impostare il certificato del Mercato per caricare o acquistare assets. Il modulo del certificato consiste nelle seguenti sezioni per quando si è già creato un certificato (vedere più in basso in questa pagina per i dettagli su come creare un certificato):

  • Certificate File: Questa sezione è dove si specifica il file *.pem che Marketplace utilizza per la certificazione di assets. Se si dispone di un certificato precedentemente creato, allora si può cliccare sul pulsante file browser e aggiungerlo al forum, ma se non hai un certificato e desideri crearne uno, allora dovresti omettere qualsiasi informazione qui e invece cliccare sul pulsante Create New Certificate (spiegato sotto).
  • Password del certificato: qui si deve inserire la password del certificato per il file selezionato sopra. La password viene creata quando si crea per la prima volta il file *.pem (vedere Creare un nuovo certificato, sotto) e sarà irreversibilmente legata al file del certificato creato.
  • Controllare: Una volta che hai riempito i campi di cui sopra dovresti cliccare su questo pulsante per assicurarti che la password e la selezione del file siano corrette e funzionino insieme.

Confermare le finestre di dialogo

Qui potete selezionare il comportamento dei dialoghi di conferma quando lavorate con una risorsa dal Browser di Asset. Le opzioni disponibili sono:

  • Risposta automatica alla cancellazione di contenuti offline: Quando cancelli un marketplace asset dalla tua libreria senza una connessione web, normalmente ricevi un avviso che ti chiede se sei sicuro di voler procedere. Impostando questa opzione su "OK" il messaggio sarà soppresso come se tu avessi cliccato il pulsante "OK" per procedere. Il valore predefinito è "Show Message".

Creare un nuovo certificato

Se non avete ancora creato un file di certificato per l'utilizzo di Marketplace, allora dovete cliccare sul pulsante etichettato Create New Certificate in fondo a questo modulo. Quando lo si fa, si apre una nuova finestra che richiede di inserire le seguenti informazioni:

  • Publisher Name: Questo è il nome che verrà utilizzato come nome del Publisher per asset. All'interno del dashboard di Marketplace è possibile impostare diversi publisher, se necessario, e ognuno dovrebbe avere il proprio corrispondente file *.pem corrispondente.
  • Password del certificato: qui dovete dare una password unica per il file del certificato da creare. Fate attenzione a ciò che scegliete perché non potete cambiarla in seguito e se non avete la password corretta per il file del certificato nella nelle sezioni precedenti non sarà possibile aggiornare alcun Marketplace assets esistente (né caricarne di nuovi).

Con questi dettagli si può quindi cliccare sul pulsante Create Certificate per generare il file *.pem. Ti verrà richiesto di dare una posizione di salvataggio e un nome al file, e dovresti prendere nota di questo per riferimento futuro. Una volta fatto questo, il nuovo file e la password saranno automaticamente aggiunto automaticamente nei campi del modulo elencati sopra e si può cliccare sul pulsante Controlla per verificare che il certificato sia stato generato correttamente.

NOTA: Si raccomanda di fare uno screenshot delle impostazioni attuali e di fare una copia del file *.pem . Entrambi dovrebbero essere salvati insieme in un luogo sicuro per riferimento futuro, o nel caso in cui sia necessario reinstallare GameMaker ad un certo punto. Non è possibile aggiornare assets che sono già stati caricati a meno che il pacchetto aggiornato sia firmato con lo stesso certificato con cui è stato creato l'originale.
