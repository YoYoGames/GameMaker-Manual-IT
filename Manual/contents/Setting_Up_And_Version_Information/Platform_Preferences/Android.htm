Preferenze Android

Le preferenze di Android sono divise in varie sezioni elencate di seguito:

Nella parte superiore delle preferenze è possibile impostare la dimensione massima del JDK Memory Heap (l'impostazione predefinita è 2GB). Poi devi impostare i vari percorsi in modo che GameMaker sappia dove cercare i vari strumenti necessari per costruire il pacchetto eseguibile finale e testare il tuo gioco. Questo è diviso in tre sezioni per Android SDK, Android NDK e Java JDK. Questi paths dovrebbero essere stati compilati automaticamente per te, ma in caso di problemi puoi cliccare sul pulsante a destra per aprire il file explorer e selezionare manualmente le posizioni per ogni elemento. Puoi trovare i dettagli di quali versioni sono richieste per ogni elemento SDK qui. Paths dovrebbe essere convalidato mentre lo applichi e un messaggio mostrerà "Found" o ti dirà di eventuali errori (nel qual caso dovresti rivedere tutte le informazioni per assicurarti che siano corrette).

Qui sotto avete le diverse opzioni di imballaggio che sono:

  • Esegui sempre l'installazione completa dell'APK: flaggando questo farà sì che GameMaker rimuova qualsiasi runner precedente del tuo gioco dal tuo dispositivo, così come tutti i file associati (per esempio, i file *.ini ) dal tuo dispositivo di prova prima di installare una nuova versione del gioco.
  • Installa su pacchetto: Quando questa opzione è contrassegnata, la compilazione di un pacchetto eseguibile finale *.apk lo installerà automaticamente anche sul dispositivo di destinazione attualmente selezionato.

Infine arriviamo al KeyStore. Questo è un file che verrà utilizzato per "firmare" tutte le vostre applicazioni Android quindi compilate tutti i dettagli correttamente e tenete a mente che questo file sarà necessario per creare e aggiornare tutte le vostre applicazioni Android in futuro, il che significa che dopo aver impostato it up si consiglia di fare uno screenshot delle impostazioni utilizzate e di fare un backup del file finale. Se perdete questo file non sarete in grado di aggiornare i giochi esistenti che sono stati caricati nel negozio.

Per creare un nuovo file keystore sono necessari i seguenti dettagli:

  • Filename: Questo è il nome del file keystore.

NOTA: Questo non è il vostro nome o il nome della società, ma il nome del file da generare! Se avete già un file keystore da progetti precedenti, potete cliccare sul pulsante del browser di file qui e indirizzare GameMaker al file keystore creato in precedenza (o cliccare sul pulsante Importa ). In questo caso, devi ancora compilare il resto delle informazioni richieste (esattamente come quando hai generato il file per la prima volta) ma non devi cliccare il pulsante Generate Keystore ! Questo genererà un nuovo file keystore, sovrascrivendo il precedente.

  • Nome comune: Questo sarebbe normalmente il tuo nome.
  • Password: la tua password di sicurezza per il file keystore che deve essere lunga almeno sei caratteri.
  • Alias: Questo è un altro nome che può essere usato per il file keystore e dovrebbe essere diverso dal nome inserito sopra.
  • Alias Password: la tua password di sicurezza alias per il file keystore che deve essere lunga almeno sei caratteri.

    NOTA Se la tua versione di Android Studio è 4.2 o superiore, devi usare la stessa password per i campi "Password" e "Alias Password" sopra. L'uso di una password e di una password alias diverse causerà il fallimento delle tue compilazioni.
  • Unità organizzativa: Il dipartimento all'interno dell'azienda di cui fai parte.
  • Organizzazione: Il nome della tua azienda.
  • Località: Il nome del paese o della città in cui si trova.
  • Paese: Il codice standard di due lettere per il paese in cui si trova.

Una volta che queste informazioni sono state compilate e ne siete soddisfatti, premete il pulsante " Generate Keystore for GameMaker " per creare il file keystore necessario. Questo può richiedere un po' di tempo, ma una volta finito, normalmente non avrà bisogno di essere cambiato di nuovo. Nota che puoi cliccare sul pulsante Import per importare un file keystore creato in precedenza (vedi la sezione Filename , sopra) e puoi anche recuperare l'hash SHA1 del keystore cliccando sul pulsante Generate Key Hash . Premendo questo genererà due speciali chiavi hash di sicurezza direttamente collegate al tuo file keystore che alcuni siti web richiedono per la corretta funzionalità (per esempio, Facebook, o Google Play per l'hash Sha1). Se ne hai bisogno, premi semplicemente il pulsante, poi copia e incolla le chiavi hash che vengono generate.
