Se il punto di collisione

Questa azione è usata per controllare e vedere se c'è una collisione con una o più istanze di un dato object in un punto specifico di room e valuterà a true se viene rilevata una collisione, o false altrimenti. Voi date l'oggetto da controllare e la posizione da controllare - che può essere una posizione assoluta in room o una posizione relativa all'istanza - e si può anche controllare il flag Not per verificare se non c'è una collisione nella posizione data, il che significa che l'azione valuterà solo come true se non ci sono collisioni, altrimenti valuterà come false. Puoi anche impostare l'opzione Exclude Self per escludere l'istanza chiamante dal controllo delle collisioni.

Poi puoi selezionare/deselezionare l'opzione per restituire la lista. Selezionando questo significa che l'azione creerà e popolerà un elenco struttura dati con tutte le istanze in collisione con il punto e imposta la variabile Target all'ID dell'elenco DS. Quando è deselezionato, imposterà la variabile Target ad un valore inferiore a 0 solo quando non viene rilevata alcuna collisione, o il valore ID unico di una delle istanze in collisione (nota che se più istanze sono in collisione allora non hai modo di sapere quale ID di istanza sarà restituito quando non usi una lista).

Se avete scelto di restituire una lista - e la variabile Target non è impostata su Temp - allora potete anche selezionare l'opzione Free Target. Quando è impostata su true, allora l'elenco contenuto nella variabile target sarà distrutto e un nuovo elenco creato (e restituito alla variabile target) ogni volta che si chiama l'azione. In questo modo è possibile riutilizzare la stessa variabile senza preoccuparsi di perdite di memoria, poiché, se non si seleziona l'opzione Free Target, si diventa responsabili di distruggere la lista prima di utilizzare nuovamente la variabile nell'azione. Si noti che, indipendentemente dal fatto che questa opzione sia selezionata o meno, se avete impostato l'azione su Return List, allora la variabile target conterrà sempre un ID di lista, e come tale, questo dovrà essere liberato quando l'istanza viene distrutta o il room finisce (si può usare l'evento Clean Up per questo, per esempio, insieme all'azione Free Data Structure ). Vale la pena notare che è possibile impostare l'opzione Free Target per utilizzare una variabile predefinita o un'espressione e quindi controllare quando la struttura dati dell'elenco viene liberata impostando la variabile altrove (o avendo diversi risultati per l'espressione) purché valuti true/false.

Infine, si fornisce la variabile Target che si vuole contenere il valore restituito o l'ID dell'elenco. Se si contrassegna la variabile Target per contenere il valore restituito come una variabile Temp (locale), allora l'azione creerà questa variabile per contenere il valore restituito solo fino alla fine dell'evento, in questo caso l'opzione Free Target non fa nulla e sarai responsabile della distruzione dell'elenco DS se questa opzione è stata selezionata. Se non viene fornita alcuna variabile di destinazione e l'opzione Return List è selezionata, non verrà creata alcuna lista.

IMPORTANTE! Le collisioni saranno registrate solo per quelle istanze che hanno una maschera di collisione valida, cioè: hanno un sprite assegnato al sprite_index, o un sprite assegnato al mask_index. Se le istanze del object che vengono controllata in questa azione non hanno una maschera di collisione, allora la collisione non sarà rilevata, indipendentemente da ciò che l'istanza sta disegnando in quel momento.
