File Azione Biblioteca

Le File Actions trattano due diversi tipi di file - Buffer file e file Ini - e hanno anche alcune azioni generiche per rinominare o copiare file esistenti.

I file buffer vengono creati salvando i dati da un buffer che avete precedentemente creato e questi dati possono poi essere caricati di nuovo in un buffer in qualsiasi momento in futuro. Potete trovare maggiori informazioni su buffers qui: Azioni Buffer.

Ifile Ini sono file piccoli e leggeri, compatibili con la maggior parte delle piattaforme. Sono ideali per memorizzare piccoli pezzi di informazioni, come le preferenze dell'interfaccia, i punteggi più alti locali, i dati dei livelli ecc... e sono molto facili da usare. I file Ini non devono essere stati creati in precedenza per utilizzare queste azioni, e si può leggere da un file Ini inesistente e si otterrà semplicemente un valore di ritorno default (che si specifica), tuttavia si consiglia di creare almeno un file "base" ini da aprire e modificare prima di utilizzare le azioni. Questo file ini di base può essere creato semplicemente chiamando l'azione Open Ini File seguita da Close Ini File, poiché la chiusura il file lo scriverà sul disco, oppure puoi includerne uno nei Included Files del Browser di Asset. Se stai usando un file incluso nel Browser Asset come Ini di base, dovresti anche leggere la sezione del manuale su come funziona il File System.

NOTA: Per i giochi con localizzazioni è molto importante che le lettere accentate possano essere lette da file esterni. Questo significa che dovreste prima creare il file ini in formato UTF8 e poi aggiungerlo in GameMaker come file incluso in modo che venga esportato all'esecuzione del gioco e utilizzato al posto del file ini di default in formato ANSI che viene creato dalle funzioni ini quando non ne è stato fornito nessuno in precedenza. In questo modo, è possibile leggere e scrivere correttamente con tutti gli accenti e le lettere non romane lettere non romane vengono mantenuti.
