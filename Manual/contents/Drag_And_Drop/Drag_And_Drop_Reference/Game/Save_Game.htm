Salvare il gioco

Questa azione farà un salvataggio di base del gioco in un dato file. Si dà un nome di file (come string e includendo il extension) e l'azione eseguirà un salvataggio limitato dello stato attuale del gioco. È importante capire che l'azione è progettata per prendere un'"istantanea" dello stato di gioco corrente ed è da usare più come un sistema di checkpoint che come un sistema di salvataggio universale, poiché non salva nessuna delle risorse dinamiche (come le strutture di dati o le particelle). Come tale l'azione dovrebbe essere usata con molta attenzione per non ottenere errori o perdite di memoria. Per un approccio più completo al salvataggio dei dati di gioco, vedi le Azioni File.

Sintassi dell'azione:

Argomenti:

Argomento Descrizione
Filename Il nome - come string e con il extension - da dare al file da salvare (se il file esiste già, sarà sovrascritto)

Esempio:

Il codice del blocco d'azione di cui sopra controllerà la presenza di una collisione con un'altra istanza e, se ne viene trovata una, controlla una variabile di istanza. Se la variabile restituisce true allora il gioco viene salvato e la variabile impostata su false.
