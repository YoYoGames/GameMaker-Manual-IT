Se l'istanza esiste

Questa azione può essere usata per controllare se un'istanza di uno specifico object esiste nel gioco room, o se un'istanza specifica stessa esiste. Si dà l'indirizzo object dall'esploratore asset, e se esiste un'istanza in room allora l'azione restituirà true altrimenti restituirà false. Si noti che è possibile utilizzare una variabile qui con l'ID unico per l'istanza specifica (per esempio come restituito dall'azione Create Object Instance) o anche l'ID dell'istanza restituito dall'editor room quando si è aggiunta l'istanza.

Se si contrassegna l'argomento "Not", allora l'azione controllerà per vedere se non esiste alcuna istanza e se non ne viene trovata nessuna restituirà true e se ne esiste anche solo una restituirà false.

Queste azioni saranno ora eseguite se il "if" valuta true, mentre qualsiasi azione abbandonata altrove sarà eseguita dopo il blocco "if".

Esempio:

Il codice del blocco d'azione di cui sopra controlla una collisione e restituisce il valore ID unico dell'istanza che è in collisione. Questo valore viene memorizzato in una variabile temporanea e poi controllato per vedere se il valore corrisponde a un'istanza che è attualmente all'interno di room (se non viene trovata nessuna collisione, la variabile conterrà la parola chiave "noone"), e se è così, allora viene impostata una variabile nell'istanza con cui si sta scontrando.
