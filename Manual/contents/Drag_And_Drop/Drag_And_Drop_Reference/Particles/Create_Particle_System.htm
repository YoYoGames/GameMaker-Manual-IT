Creare un sistema di particelle

Questa azione inizializzerà il sistema di particelle pronto per la creazione e l'utilizzo di tipi di particelle ed emettitori con esso. Dai il nome al livello come string - per esempio "Particle Layer" (che deve esistere nell'editor room ) - e poi contrassegna il sistema come persistente o no. Se il sistema è contrassegnato come Persistente allora rimarrà attivo in tutti i rooms dopo che è stato creato, e gli verrà assegnato un livello gestito se il livello iniziale non esiste nei successivi rooms. Se la persistenza non è contrassegnata, allora il sistema sarà automaticamente distrutto alla fine del room in cui è stato creato, così come tutti gli emettitori associati al sistema. Se si desidera rimuovere manualmente il sistema dalla memoria, è necessario utilizzare l'azione Destroy Particle System.

È inoltre necessario fornire una variabile di destinazione che conterrà il valore ID unico del sistema di particelle. Questo valore ID è quello che verrà utilizzato in altre funzioni per impostare le proprietà del sistema o per dire a GameMaker quale sistema utilizzare quando si creano le particelle. È possibile scegliere di contrassegnare questa variabile come una variabile locale temporanea, ma ciò significherà che verrà scartata alla fine dell'azione script o dell'evento, il che significa che il sistema esisterà ancora, ma vi lascerà senza alcun mezzo per accedervi, quindi bisogna fare attenzione deve essere fatta attenzione quando si usa questo (in generale i sistemi non avranno mai bisogno di essere memorizzati in una variabile locale temporanea, ma per certe cose, come disegnare particelle su una superficie, potrebbe essere necessario).

NOTA: Un sistema di particelle persistente, una volta creato, rimane residente nella memoria del sistema per tutta la durata del gioco, e quindi se avete bisogno di ricrearlo in qualsiasi momento (ad esempio, se si chiama l'azione Restart Game ), allora è necessario distruggere il sistema corrente prima altrimenti si ottiene una perdita di memoria. Una perdita di memoria avviene quando una risorsa viene aggiunta alla memoria e poi "de-referenziata", il che significa che GameMaker non ha più modo di interagire con quella risorsa. Questo porta la memoria ad essere intasata da risorse inaccessibili che a loro volta possono causare problemi al vostro gioco. Per distruggere un sistema di particelle persistente quando non è più necessario usate l'azione Destroy Particle System.

Sintassi dell'azione:

Argomenti:

Argomento Descrizione
Layer Il livello in cui saranno disegnate le particelle di questo sistema
Persistent Se il sistema di particelle deve rimanere attivo dopo che il corrente room è terminato
Target La variabile di destinazione per contenere il valore ID unico del sistema creato

Esempio:

Il blocco d'azione di cui sopra crea una variabile globale, quindi crea un nuovo sistema di particelle e assegna il suo valore ID unico alla variabile globale. Il sistema di particelle creato è contrassegnato come persistente, quindi rimarrà attivo anche dopo la fine dell'attuale room.

Indietro: Azioni delle particelle
Successivo: Distruggere il sistema di particelle
