Eseguire lo script

Questa azione può essere usata per chiamare una funzione script o una funzione definita dall'utente. In generale, questa azione verrebbe usata solo per il supporto legacy e non dovrebbe essere usata in progetti attuali o nuovi. Per usare l'azione si dà l'indirizzo script asset da chiamare - o il nome della funzione definita dall'utente da chiamare - e poi si forniscono tutti gli argomenti che richiede, cioè: qualsiasi valore (strings, interi, indici di risorse, ecc...) che si desidera inserire nello script o funzione in modo che possa utilizzarli. Puoi aggiungere altri argomenti come richiesto cliccando sull'icona più accanto all'azione.

Alcune scripts e funzioni definite dall'utente possono anche restituire un valore per un uso successivo nell'elenco dei blocchi d'azione, e quindi è possibile specificare la variabile di destinazione che dovrebbe memorizzare questo valore di ritorno. Questa può essere stata creata in precedenza o può essere una nuova variabile temporanea temporanea locale (se si seleziona la casella di controllo "Temp"). Sia il campo target che quello dell'argomento possono essere lasciati vuoti se non sono richiesti.

Notate che generalmente è meglio usare l'azione Function Call per chiamare funzioni definite dall'utente, e non dovreste generalmente aver bisogno di usare questa azione, o chiamare un script asset direttamente nel codice.

Sintassi dell'azione:

Argomenti:

Argomento Descrizione
Script Il nome della funzione script o definita dall'utente da chiamare.
Argument0 ... Argument3 I diversi argomenti (valori) che devono essere passati a script o alla funzione (gli argomenti inutilizzati possono essere lasciati vuoti)
Target Il nome della variabile che deve essere presa di mira per qualsiasi valore restituito (può essere lasciato vuoto)

Esempio:
