Se l'espressione

Questa azione è usata per controllare che un'espressione sia true o false, e potete controllare qui qualsiasi espressione logica, purché valuti uno o l'altro valore. Per esempio, si può usare una qualsiasi delle seguenti espressioni:

x < (room_width / 2)
a == b
++val >= 100

Se contrassegnate l'argomento "Not", allora le espressioni di cui sopra saranno negate, per esempio "==" diventa "not=". Notate anche che l'espressione non è limitata ai valori numerici e potete controllare strings, puntatori, ID di risorse o qualsiasi altra cosa che una funzione può restituire o usare (vedete la sezione sui tipi di dati per maggiori informazioni), ma non potete confrontare due diversi data types senza ottenere un errore - per esempio se provate a usare un string e un intero nell'espressione otterrete un errore. Per maggiori informazioni sulle espressioni, vedere qui.

NOTA IMPORTANTE: A causa di problemi di precisione in virgola mobile, il controllo per vedere se due valori sono esattamente uguali può restituire false, poiché uno può essere esattamente 1, mentre l'altro potrebbe essere 1.00000000000001. Questo può essere evitato usando l'azione Decimal to Integer prima di controllare o usando le espressioni "maggiore di (>)" o "minore di (<)".

Notate che per aggiungere azioni nel blocco "if", esse devono essere lasciate cadere a lato dell'azione, come mostrato nell'immagine qui sotto:

Queste azioni saranno ora eseguite se il "if" valuta true, mentre qualsiasi azione abbandonata altrove sarà eseguita dopo il blocco "if". Nota che puoi "impilare" le espressioni nella stessa azione cliccando l'icona più accanto a l'azione e dando un'altra espressione da controllare. Questo controllerà se tutte le espressioni date restituiscono true in un solo controllo ed eseguirà il codice dell'azione aggiunta solo se lo fanno (vedi l'esempio dato sotto).
