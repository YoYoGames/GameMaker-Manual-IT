Pacchetti locali Asset

Nel menu Strumenti sono disponibili due opzioni per la creazione e l'importazione di Pacchetti locali Asset . Un Pacchetto locale Asset è un gruppo di risorse salvate nel formato di file *.yymp che può essere facilmente installato in progetti esistenti. Ciò significa che è possibile creare una propria libreria di pacchetti per le risorse comunemente utilizzate, come le librerie script, le introduzioni per le schermate di avvio, i sistemi di menu e così via, senza dover utilizzare ilMarketplace GameMaker.

Creare un pacchetto Asset

Per creare un pacchetto asset, occorre prima aver creato le risorse che si desidera salvare in un progetto e poi, dal menu Strumenti, selezionare l'opzione Crea pacchetto locale. Si aprirà la seguente finestra:

In questa finestra è possibile fornire un nome di editore opzionale e un nome di visualizzazione e un ID pacchetto obbligatori. L'ID pacchetto verrà utilizzato per generare il nome del file con cui verrà salvato il pacchetto e verrà automaticamente aggiunto al file *.yymp extension al momento della creazione. È inoltre possibile impostare il numero di versione del pacchetto e scegliere se firmarlo o meno (l'impostazione predefinita è disattivata). La firma di un pacchetto richiede la creazione di un certificato di firma (si può fare dalle Preferenze del Marketplace) e significa che qualsiasi asset firmato non potrà essere importato se è stato manomesso dopo la creazione.

Una volta compilati i dettagli del pacchetto, si può procedere a selezionare il sito assets che si desidera includere nella sezione "Progetto" sulla destra (utilizzando il tasto sinistro del mouse o una combinazione di / e con il tasto sinistro del mouse). Le risorse selezionate possono essere spostate nella vista "Pacchetto di risorse", usando i pulsanti Aggiungi o Aggiungi tutto e possono essere rimosse usando i pulsanti Rimuovi e Rimuovi tutto.

Una volta soddisfatti della selezione delle risorse e dopo aver compilato i dettagli richiesti, facendo clic sul pulsante Ok si aprirà un file explorer in cui si potrà scegliere il percorso in cui salvare il pacchetto. Una volta selezionato, il pacchetto verrà creato e verrà visualizzata una finestra di conferma, con l'opzione di aprire il percorso del pacchetto nell'esploratore di file del sistema operativo.

Importare un pacchetto Asset

Esistono due modi per importare un pacchetto locale Asset:

  • la prima consiste semplicemente nel trascinare il file *.yymp da una finestra di explorer a GameMaker IDE
  • la seconda è selezionare l'opzione "Importa pacchetto locale" dal menu Strumenti.

Quando si usa il menu Strumenti, si apre una finestra di esplorazione e si può navigare fino al percorso del pacchetto e poi selezionarlo per l'importazione. In entrambi i casi si aprirà la seguente finestra in GameMaker:

A sinistra si trova il contenuto del pacchetto e a destra le parti che si desidera importare. È possibile selezionare una o più risorse dal pacchetto (utilizzando il tasto sinistro del mouse o una combinazione di / e insieme al tasto sinistro del mouse) e quindi utilizzare il tasto Aggiungi per aggiungerle all'importazione, oppure utilizzare il tasto Aggiungi tutto per aggiungere tutto all'importazione.

Quando tutto è pronto, si può fare clic sul pulsante Importa per aggiungere le risorse selezionate al progetto. Si noti che esiste anche l'opzione Importa tutte le risorse in un nuovo progetto, che creerà un nuovo progetto (e chiederà di indicare una posizione in cui salvarlo) e aggiungerà a questo le risorse del pacchetto.

Si noti che i pacchetti asset creati con una versione precedente a GameMaker.3 richiederanno la conversione, poiché il formato dei file di progetto è cambiato con l'aggiornamento alla versione 2.3. Ciò significa che potrebbe essere preferibile importare l'intero progetto e convertire tutto, quindi esportarlo come nuovo pacchetto in una nuova posizione se assets viene importato frequentemente in altri progetti.
