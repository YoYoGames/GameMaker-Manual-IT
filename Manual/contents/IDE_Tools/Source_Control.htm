Controllo delle fonti

Source Control Management (SCM) è il nome dato al metodo di lavorare con sotto-versioni o backup dei vostri progetti attraverso un repository e una fonte locale. Fondamentalmente, una soluzione SCM è un pacchetto software indipendente che controlla tutti gli aspetti del mantenimento, della modifica e del confronto delle versioni del vostro progetto mentre ci lavorate. Questo è particolarmente utile per coloro che lavorano in un team e hanno bisogno di essere in grado di controllare chi fa cosa e non preoccuparsi di perdere dati o fare modifiche che potrebbero dover essere annullate in un secondo momento, ma anche gli individui possono beneficiare di questo sistema potente e flessibile.

GameMaker fa uso di Git per il suo SCM, che deve essere installato e impostato separatamente prima che SCM possa essere usato all'interno di GameMaker. Vedere la documentazione di Git per informazioni più specifiche riguardo a Git.

GameMaker mostrerà un menu a discesa nella barra dei menu per Source Control, che sarà popolato con le seguenti opzioni una volta che Source Control è stato attivato per il vostro progetto (vedere la sezione Impostazione del plugin Git sopra per i dettagli):

Qui delineiamo ognuna delle opzioni disponibili (la maggior parte di esse sono spiegate in modo più approfondito nelle sezioni del tutorial di cui sopra):

  • Create Project Repository: Questa opzione ti permette di creare un repository locale nella stessa directory dei tuoi file di progetto.
  • Clona repository: Questa opzione vi permetterà di clonare un repository da una sorgente a una nuova destinazione.
  • Commit Changes: Con questa opzione puoi mettere in scena i file modificati nel tuo progetto e poi commetterli nel repository.
  • Spingere le modifiche: Dopo aver fatto un commit o un merge questa opzione è usata per spingere le modifiche al repository master.
  • Pull Changes: Con questa opzione puoi aggiornare il repository locale estraendo i file modificati dal repository master.
  • Visualizza cronologia: Questo aprirà la finestra della cronologia dove puoi vedere tutta la storia delle versioni del progetto e scegliere di rollbackare specifici file paths o intere revisioni.
  • Mostra conflitti: Questo aprirà la finestra Conflicts ed elencherà tutti i file in conflitto che possono esistere nel progetto, permettendovi di trattarli o attraverso GameMaker IDE o usando uno specifico Merge Tool.
