Clonare un repository

Quando il tuo progetto è memorizzato in un repository esterno, hai bisogno di ricordarti sia di spingere le tue modifiche che di estrarre quelle nuove. La ragione di questo è che quando fai il commit, memorizzi le tue modifiche localmente, il che ti permette di lavorare lontano da una connessione internet e di sincronizzarti quando sei pronto, e per sincronizzarti fai il push delle tue modifiche al repository remoto. Tirare le modifiche significa che vuoi tirare tutte le modifiche che altri hanno fatto dal repository remoto nel tuo repository locale. Tuttavia, questo significa che ci possono essere momenti in cui le persone possono modificare gli stessi file, il che a sua volta crea un conflitto di fusione. Hai bisogno di sapere come gestire questi conflitti, quindi ne creeremo deliberatamente uno ora e poi ti mostreremo come gestirlo.

NOTA: Condividere un repository locale del progetto (anche attraverso un servizio di sincronizzazione di file come Dropbox) non è raccomandato, poiché i commit sono generalmente scritti direttamente su di esso, e quindi si ha una protezione dai conflitti molto scarsa.

Per generare il nostro conflitto dobbiamo prima avere due repository locali, uno per il repository Master e uno per la nostra copia. Per creare questo abbiamo bisogno di clonare il repository, cosa che viene fatta con i seguenti passi:

  • Vai a Source Control > Clone Repository.
  • Nel campo URL, digita l'indirizzo path al progetto che abbiamo impostato nella guida Reverting Files qui sopra (per default questo sarà <DRIVE>:\Users\<Username>\Documents\GameMaker\<ProjectName>).
  • Abbiamo poi bisogno di impostare un path su cui clonare:
    Quando clicchiamo sul pulsante "Ok" otterremo il seguente avvertimento sul tentativo di clonare un repository "non-bare":
    Possiamo cliccare nuovamente su "Ok".
  • Ora ci verrà presentato un esploratore di directory per creare un repository proxy "nudo" (questo è un repository che si può tranquillamente memorizzare su Dropbox, per esempio). Sfogliare fino a dove vogliamo creare questo repository proxy e selezionare (o creare) la cartella da utilizzare.

Una volta che abbiamo fatto questo, GameMaker spingerà il progetto originale al proxy, e ne tirerà uno nuovo al path che abbiamo impostato nella finestra Clone Repository e poi aprirà automaticamente un browser di file in modo da poter aprire il progetto memorizzato. Se ora andiamo in Source Control > View History vedremo i nostri commit originali più un nuovo commit "Push For Clone".

Ora hai clonato il tuo repository e puoi continuare a lavorare sulla versione locale del progetto.
