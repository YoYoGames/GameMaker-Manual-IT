Conflitti

In questa sezione vedremo come risolvere i conflitti tra un repository locale clonato e un repository master. Supponendo che tu stia usando un repository locale, avremo bisogno di modificare un file e quindi causare un qualche tipo di conflitto, quindi apri il test script che abbiamo usato e cambia il messaggio "Hello World" in qualcosa come questo:

show_debug_message("Hello World, How Are You?");

Ora dovresti salvare il progetto e fare il commit nel repository locale. Ora, dato che siamo collegati ad un repository master remoto (anche se è uno che abbiamo clonato sullo stesso disco) abbiamo bisogno di fare un Push per portare le modifiche al master. Per questo, andate su Source Control > Push Changes, che ci darà un aggiornamento dello stato nella scheda Source Control Output:

Ora abbiamo bisogno di aprire il nostro progetto originale (non quello clonato), quindi cercate dove lo avete salvato sul disco - o semplicemente controllate i Progetti recenti nel menu File, perché dovrebbe essere direttamente sotto quello corrente - e caricatelo in GameMaker. Un modo semplice per controllare di avere il progetto giusto è semplicemente controllare il sito script che avete e se il messaggio è "Hello World" allora avete quello giusto, oppure potete aprire la finestra View History e controllare così.

Ora avete bisogno di modificare nuovamente il sito script, in modo da fargli leggere qualcosa di simile a questo:

show_debug_message("This will cause a conflict.");

Questo può essere salvato su disco, e le modifiche impegnate, ma se provate a spingere queste modifiche otterrete il seguente messaggio:

Qualcuno ha già inviato delle modifiche che tu non hai ancora! Questo significa che hai bisogno di fare un pull sul repository e vedere esattamente qual è il problema, quindi vai su Source Control > Pull Changes. Questo mostrerà due finestre, la prima è un avvertimento sui file modificati (come per le azioni precedenti, vuoi scegliere Reload qui e non Save), e l'altra è la finestra Conflicts :

Questa finestra mostrerà i file in conflitto in una lista a sinistra, e potrete selezionare i file da quella lista usando il tasto sinistro del mouse (o selezionare più file usando / o insieme a ). Avete quattro opzioni per trattare i file in conflitto, disponibili dai pulsanti sulla destra:

  • Use Theirs - Questo sovrascriverà le modifiche che hai fatto con quelle del repository remoto.
  • Use Mine - Questo sovrascriverà le modifiche dal repository remoto con quelle che hai appena fatto.
  • Unisci - Questo tenterà di aprire uno strumento di unione esterno per gestire il conflitto.
  • Scegliere Merge Tool - Questo ti permetterà di configurare il Merge Tool (vedi la sezione su External Merge/Diff Tools per maggiori dettagli).

Ora, potete usare i pulsanti per risolvere il conflitto, ma prima di farlo, apriamo il file in conflitto per vedere esattamente qual è il problema. Se aprite il sito script che è in conflitto, ora avrà questo aspetto:

Questo mostra che "This Will Cause A Conflict" è nell'HEAD (questo è ciò che abbiamo spinto nel repository locale) e che "Hello World, How Are You?" è stato estratto da master, il repository remoto. Ora puoi tornare alla finestra Conflicts e cliccare su Use Theirs per estrarre il file modificato e sovrascrivere il file HEAD corrente. Il script sarà ora segnalato come modificato e dovresti fare un nuovo commit e push prima di continuare.

Notate che potremmo effettivamente risolvere questo problema nel GameMaker IDE. Se rimuoviamo le linee 1, 2, 3 e 5 in modo che ci rimanga solo show_debug_message("Hello World, How Are You?") e lo salviamo, il conflitto sarà risolto, poiché Git assumerà che si sappia cosa si sta facendo con il proprio file. Tuttavia, questo è soggetto a errori, quindi si raccomanda di installare uno strumento di fusione, quindi configurarlo e usarlo invece di cercare manualmente di unire.
