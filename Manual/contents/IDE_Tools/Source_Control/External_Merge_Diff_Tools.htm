Strumento di fusione

Quando si lavora a un progetto con altre persone, c'è sempre la possibilità che si verifichi un conflitto, dato che più persone modificano lo stesso file e la maggior parte dei sistemi source control può fare un buon tentativo di unire qualsiasi file in conflitto, ma quando ci sono modifiche nello stesso punto, è necessario un po' di aiuto per sapere cosa fare.

Poiché ogni strumento di fusione è diverso dall'altro, esistono quattro macro che possono essere utilizzate per aiutare a specificare i file ad esso destinati e che possono essere impostate dalla finestra Conflitti facendo clic sul pulsante con la dicitura Scegli strumento di fusione:

Facendo clic su questo pulsante si aprirà la finestra delle preferenze del plugin Git:

Qui si fornisce un path allo strumento di unione scelto e quindi una macro (o più macro) Opzioni strumento di unione da utilizzare per decidere cosa fare, con le seguenti opzioni disponibili:

  • ${scm_mine}: path alla vostra versione
  • ${scm_theirs}: path alla loro versione
  • ${scm_base}: path alla versione precedente alle modifiche in conflitto.
  • ${scm_merged}: path dove lo strumento di fusione deve salvare

Come aiuto per l'impostazione, quando si assegna un path a determinati strumenti di fusione, GameMaker inserisce alcune impostazioni predefinite per iniziare, a seconda dello strumento scelto:

  • KDiff3: "${scm_theirs} "${scm_mine}" -o "${scm_merged}"
  • Fondere: "${scm_mine}" "${scm_merged}" "${scm_theirs}"
  • Notepad++: ${scm_merged}
  • TartarugaFusione: /base:"${scm_base}" /theirs:"${scm_theirs}" /mine:"${scm_mine}" /merged:"${scm_merged}"
  • DiffMerge: "${scm_mine}" "${scm_merged}" "${scm_theirs}"

Se non avete (o non volete usare) uno strumento di fusione, potete aprire il sito script in IDE e cercare di risolvere il problema da lì.

Quindi, torniamo al nostro esempio di conflitto, come indicato nella sezione sui conflitti... Abbiamo fatto un pull da master e uno dei file è in conflitto. Si apre la finestra Source Control Conflicts , dove si può fare clic sul file in conflitto script, e ora fare clic su Merge, dato che è stato configurato lo strumento di fusione. Facendo clic su Unisci si aprirà lo strumento in cui è possibile gestire il conflitto e quindi utilizzare le macro fornite per apportare le modifiche necessarie. Una volta terminato e chiuso lo strumento di fusione, GameMaker IDE mostrerà la seguente finestra di conferma:

Rispondete semplicemente "Sì" a questa richiesta, come se l'aveste appena fatto voi stessi usando lo strumento di unione, e poi dovete fare il commit di questa modifica per indicare la fine dell'operazione di unione(Controllo sorgente > Commit Changes). La finestra Commit ora mostrerà che la fusione è in corso e mostrerà anche alcuni file non sottoposti a commit:

Questi file non staccati sono il nostro file script con l'aggiunta di .base, .ours e .theirs. Possiamo eliminare tutti questi file in modo sicuro, quindi facciamo clic su in alto e poi su + in basso per selezionarli tutti. Quindi, facciamo clic con il pulsante destro del mouse sui file selezionati e scegliamo Elimina percorso per rimuoverli. C'è anche un messaggio di commit precompilato, che indica che si tratta di una fusione e che c'era un conflitto (si può aggiungere o modificare a piacere).

Ora possiamo fare clic su Impegna .

A causa del fatto che si possono avere problemi con altri file, significa che il flusso di lavoro quando si utilizzano repository remoti o condivisi dovrebbe essere il seguente:

Make Changes > Save > Commit > Pull > Merge > Push

Strumento Diff

È necessario impostare anche uno strumento Diff da usare quando si eseguono i commit di un repository. Si imposta dalle Preferenze, come per lo strumento Merge:

Lo strumento Diff è configurato in modo simile allo strumento Merge, ma utilizza solo le opzioni ${scm_base} e ${scm_theirs}; in questo caso, ${scm_base} rappresenta il file non modificato, mentre ${scm_theirs} rappresenta lo stato attuale del file. Le opzioni dello strumento dovrebbero essere precompilate con le opzioni predefinite per Meld, KDiff3 e TortoiseMerge, proprio come per lo strumento Merge. Si noti che GameMaker non supporta l'output di patch diff di singoli file, pertanto non esiste un'opzione predefinita per Notepad++.

Per visualizzare le modifiche tra le revisioni attuali e quelle precedenti utilizzando lo strumento Diff, è sufficiente aprire la finestra Commit e fare doppio clic su un file. Se lo strumento è stato configurato correttamente, verrà lanciato e sarà possibile vedere le modifiche tra i file.
