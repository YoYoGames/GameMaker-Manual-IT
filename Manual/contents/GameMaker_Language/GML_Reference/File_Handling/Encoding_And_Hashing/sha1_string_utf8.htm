sha1_stringa_utf8

In crittografia, SHA-1 è una funzione di hash crittografica progettata dalla United States National Security Agency ed è impiegata in diverse applicazioni e protocolli ampiamente utilizzati come il popolare Git dove è usato per controllare le modifiche ai file. Questa funzione prenderà un input UTF-8 string (che ha un numero variabile di byte per carattere) e restituisce un message digest a 160 bit in formato ASCII, per esempio:

sha1 input string = "The quick brown fox jumps over the lazy dog"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

In questo modo è possibile generare una chiave sicura che può essere memorizzata e utilizzata per controllare l'integrità delle informazioni inviate a (o ricevute da) un server esterno (per esempio).

NOTA: Ci sono due formati per la codifica sha1, UTF-8 e unicode. Entrambi sono forniti per facilitare la comunicazione con diverse impostazioni server, ma il più comune da usare è l'unicode.

NOTA: SHA-1 non è completamente sicuro e può essere rotto. Vedi questa pagina per maggiori informazioni.

Sintassi:

sha1_string_utf8(string)

ArgomentoTipo Descrizione
string The string to hash.

Ritorna:

Esempio:

var hash, str;
str = json_encode(hiscore_map);
hash = sha1_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

Il codice precedente codificherà una mappa DS in un JSON string. Un hash sha1 viene poi generato e memorizzato in un file ini in modo che questo possa essere usato in seguito per controllare l'integrità del JSON se le stesse informazioni dovessero essere ricevute in seguito da server. Il JSON viene poi inviato.
