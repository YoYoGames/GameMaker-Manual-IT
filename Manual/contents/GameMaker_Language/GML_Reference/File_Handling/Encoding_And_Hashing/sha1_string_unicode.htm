sha1_stringa_unicode

In crittografia, SHA-1 è una funzione di hash crittografica progettata dalla United States National Security Agency ed è impiegata in diverse applicazioni e protocolli ampiamente utilizzati come il popolare Git dove è usato per controllare le modifiche ai file. Questa funzione prende un input unicode string e restituisce un message digest a 160 bit in formato ASCII, per esempio:

sha1 input string = "The quick brown fox jumps over the lazy dog"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

In questo modo è possibile generare una chiave sicura che può essere memorizzata e utilizzata per controllare l'integrità delle informazioni inviate a (o ricevute da) un server esterno (per esempio).

NOTA: Ci sono due formati per la codifica sha1, UTF-8 e unicode. Entrambi sono forniti per facilitare la comunicazione con diverse impostazioni server, ma il più comune da usare è l'unicode.

NOTA: SHA-1 non è completamente sicuro e può essere rotto. Vedi questa pagina per maggiori informazioni.

Sintassi:

sha1_string_unicode(string)

ArgomentoTipo Descrizione
string The string to hash.

Ritorna:

Esempio:

var hash, str;
str = base64_encode(game_data);
hash = sha1_string_unicode(str);
http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?hash=" + hash); http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?data=" + str);

Il codice di cui sopra codifica in base64 un string e poi genera un hash sha1. Infine, sia l'hash che il string codificato sono inviati a un server.
