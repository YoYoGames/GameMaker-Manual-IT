md5_string_utf8

In crittografia, MD5 (Message-Digest algorithm 5) è una funzione di hash crittografica ampiamente utilizzata con un valore di hash a 128 bit ed è stata impiegata in un'ampia varietà di applicazioni di sicurezza. È anche comunemente usato per controllare l'integrità dei file e strings. Questa funzione prenderà un input UTF-8 string (che ha un numero variabile di byte per carattere) e restituirà l'hash MD5 esadecimale di 32 caratteri che è unico per quel string. In questo modo è possibile generare una chiave sicura che può essere memorizzata e utilizzata per controllare l'integrità delle informazioni inviate a (o ricevute da) un server esterno (per esempio).

NOTA: Ci sono due formati per la codifica MD5, UTF-8 e unicode. Entrambi sono forniti per facilitare la comunicazione con diverse impostazioni server, ma il più comune da usare è l'unicode.

NOTA: MD5 non è completamente sicuro e può essere rotto. Vedi questa pagina per maggiori informazioni.

Sintassi:

md5_string_utf8(string)

ArgomentoTipo Descrizione
string The string to hash.

Ritorna:

Esempio:

var hash, str;
str = json_encode(hiscore_map);
hash = md5_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

Il codice precedente codificherà una mappa DS in un JSON string. Un hash MD5 viene poi generato e memorizzato in un file ini in modo che questo possa essere usato in seguito per controllare l'integrità del JSON se le stesse informazioni dovessero essere ricevute in seguito da server. Il JSON viene poi inviato.
