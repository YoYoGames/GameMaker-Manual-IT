file_find_first

Questa funzione restituisce il nome del primo file che soddisfa la maschera e gli attributi. Se non esiste un tale file, allora viene restituito un string vuoto.

La maschera può contenere un path con caratteri jolly, per esempio C:\temp\*.doc.

Gli attributi danno i file aggiuntivi che volete vedere, quindi i file normali sono sempre restituiti quando soddisfano la maschera. Potete aggiungere le seguenti costanti per vedere il tipo di file che volete (usando |). Se non volete aggiungere alcun attributo, usate 0.

Costante di attributo del file
Costante Descrizione
fa_readonly File di sola lettura
fa_hidden File nascosti
fa_sysfile File di sistema
fa_volumeid File volume-id
fa_directory Elenchi
fa_archive File archiviati

Gli attributi possono essere usati solo su Windows. Per tutte le altre piattaforme, usate 0.

ATTENZIONE Questa funzione potrebbe non funzionare come vi aspettate a causa della sandbox di GameMaker! Si prega di vedere la sezione sul File System per maggiori informazioni.

NOTA Questa funzione non funzionerà affatto sul target HTML5.

Sintassi:

file_find_first(mask, attr);

Argomento Tipo Descrizione
mask Stringa The mask to use for searching.
attr Costante di attributo del file(s) The specific file attribute to look for.

Ritorna:

String

Esempio:

if directory_exists("\User Content")
{
    fileA = file_find_first("/User Content/*.doc", fa_readonly);
    fileB = file_find_next();
    fileC = file_find_next();
    file_find_close();
}

Questo codice controlla se la directory specificata esiste e, se esiste, ci va e restituisce il primo file "read only" .doc trovato. Poi cerca altri due file e chiude il cercatore di file.

Potete cercare qualsiasi numero di file usando un while ciclo:

var files = [];
var file_name = file_find_first("/User Content/*.doc", fa_readonly);

while (_file_name != "")
{
    array_push(_files, _file_name);

    _file_name = file_find_next();
}

file_find_close();

Il codice precedente crea un array vuoto per memorizzare tutti i nomi dei file che sono stati trovati, e inizia a cercare i file di sola lettura .doc. Se quel nome di file non è un string vuoto, lo spinge nell'array files, e poi cerca di trovare il file successivo. Il loop continuerà a ripetere fino a quando non viene trovato un string vuoto (il che significa che non ci sono più file corrispondenti), che è quando termina il loop e chiude il cercatore di file alla fine.
