Gestione dei file

Ad un certo punto, mentre create il vostro gioco, sarà necessario memorizzare informazioni esternamente ed essere in grado di recuperare queste informazioni. Per esempio, potreste voler memorizzare i dettagli di login, o i punteggi più alti, o forse la progressione del giocatore attraverso i vostri livelli. Qualunque sia la ragione, avrete bisogno di sapere come utilizzare le diverse funzioni di file che sono disponibili quando si lavora con GameMaker.

È molto importante rendersi conto che GameMaker è Sandboxed per impostazione predefinita, il che significa che ci sono una serie di limiti e complicazioni minori quando si utilizzano le funzioni di file system che sono disponibili, e vale la pena notare anche che il file system funzionerà in modo leggermente diverso a seconda della piattaforma a cui si punta. Per maggiori informazioni in merito, consultate la seguente sezione:

  • Il file system

È possibile disattivare il sandboxing sugli obiettivi del desktop (Windows, macOS e Ubuntu (Linux)) dalle Opzioni di gioco per la piattaforma scelta, il che significa che è possibile salvare e caricare da qualsiasi luogo, nei limiti dei permessi del sistema operativo.

Se vi è stato concesso il permesso da Steam di usare le loro API dedicate con il vostro gioco, scoprirete che c'è anche una nuova serie di funzioni di file a vostra disposizione. Queste sono spiegate nella sezione Steam API di questo manuale.

Infine, dovreste anche notare che ci sono alcune differenze distinte tra alcune piattaforme di destinazione che verrebbero prese in considerazione quando si usano le funzioni dei file. Queste sono:

  • iOS: Su iOS, se state caricando un file incluso da una cartella, cioè: con un path qualcosa come "Backgrounds/background1.png", non è necessario includere la cartella come parte del path e fornire semplicemente il nome del file è sufficiente. Su tutte le altre piattaforme è necessario fornire la cartella (o le cartelle) come parte di path al file.
  • macOS e Ubuntu (Linux): Su macOS e Ubuntu, i nomi dei file e paths sono sensibili alle maiuscole e alle minuscole su queste piattaforme, e quindi per prevenire qualsiasi potenziale problema, GameMaker farà automaticamente usare tutti i nomi/percorsi in minuscolo per default.
  • HTML5: Su HTML5, il salvataggio e il caricamento da e verso la memoria locale funzioneranno come spiegato nelle diverse pagine di funzioni, tuttavia se stai caricando un file da un server, devi usare solo la funzione buffer_load_async() poiché il caricamento sincrono è stato deprecato sulla maggior parte dei browser e alla fine sarà obsoleto. Notate anche che non potete creare o distruggere directory nella memoria locale del browser.
