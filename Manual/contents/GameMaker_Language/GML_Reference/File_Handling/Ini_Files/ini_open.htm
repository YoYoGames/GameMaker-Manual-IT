ini_open

Questo apre un file ini per la lettura e la scrittura. Se il file ini non esiste nella posizione che state controllando, GameMaker può crearne uno, ma solo se ci scrivete dei dati. Se avete solo letto informazioni dal file ini, allora i valori predefiniti per la funzione di lettura saranno restituiti, ma il file ini non sarà effettivamente creato.

Notate che potete avere solo un file ini aperto alla volta e ricordatevi di usare ini_close() una volta che avete finito di leggere/scrivere dal file .ini, poiché le informazioni non sono effettivamente memorizzate su disco fino ad allora (è anche conservata in memoria fino a quando il file viene chiuso).

ATTENZIONE! Questa funzione potrebbe non funzionare come vi aspettate a causa della sandbox di GameMaker! Vedere la sezione sul File System per maggiori informazioni.

Sintassi:

ini_open(name);

ArgomentoTipo Descrizione
name The filename for the .ini file.

Ritorna:

Esempio:

ini_open("Settings/savedata.ini");
score = ini_read_real("save1", "score", 0);
ini_close();

Questo aprirà 'savedata.ini' e leggerà il valore del punteggio sotto la sezione "save1" con la chiave "score" in esso, poi chiuderà nuovamente il .ini. Se non c'è nessun valore sotto "save1", "score" o non c'è nessun file "savedata.ini" presente, score sarà impostato a 0 (il valore predefinito). Nota che il file ini è stato messo nella sottocartella "Settings", che è la cartella che contiene il file ini nei file inclusi in Asset Browser.
