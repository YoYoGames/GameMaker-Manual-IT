chiusura del file_testo

Una volta che avete finito di lavorare con un dato file (sia in lettura che in scrittura), dovete chiudere nuovamente il file, altrimenti rischiate di perdere le informazioni contenute al suo interno. Questo previene anche le perdite di memoria e fa sì che non si superi mai il limite dei file avendo più di 32 file aperti.

Sintassi:

file_text_close(fileid);

ArgomentoTipo Descrizione
fileid The id of the file to close.

Ritorna:

Esempio:

var file = file_text_open_write(working_directory + "Game_Data.txt");
while (!file_text_eof(file))
{
    file_text_readln(file);
}
file_text_write_string(file, level_data);
file_text_close(file);

Il codice qui sopra apre un file per la scrittura poi loops attraverso le linee di testo già scritte nel file fino a raggiungere la fine. A questo punto scrive un string e poi chiude nuovamente il file.

Indietro: File di testo
Il prossimo: file_text_open_read
