file_testo_apri_scrittura

Questa funzione apre il file di testo con il nome del file indicato per la sola scrittura (se il file non esiste, viene creato), restituendo l'id univoco del file che deve essere memorizzato in una variabile, poiché verrà utilizzato per tutte le azioni successive relative a quel file.

Si noti che se il file non può essere creato (ad esempio a causa di un nome di file illegale) la funzione restituirà -1.

Se il file esiste già, l'uso di questa funzione ne causerà la sovrascrittura al momento del salvataggio dei dati. Se si desidera aggiungere dati a un file mantenendo intatto il suo contenuto esistente, utilizzare la funzione file_text_open_append().

NOTA: È possibile avere un massimo di 32 file aperti alla volta. È inoltre consigliabile chiudere sempre i file al termine, in quanto in questo modo si scrivono le informazioni e si libera la memoria associata al file.

ATTENZIONE! Questa funzione potrebbe non funzionare come ci si aspetta a causa del sandboxing di GameMaker! Per ulteriori informazioni, consultare la sezione sul File System.

Sintassi:

file_text_open_write(fname);

Argomento Tipo Descrizione
fname Stringa The name of the file to write to.

Restituzione:

Text File ID or -1

Esempio:

var file;
file = file_text_open_write(working_directory + "level.txt");
file_text_write_string(file, level_data);
file_text_close(file);

Il codice di cui sopra aprirà il file "level.txt" per la scrittura e quindi scriverà il string memorizzato nella variabile "level_data" prima di chiudere nuovamente il file.
