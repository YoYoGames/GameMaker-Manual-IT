File di testo

I file di testo sono un modo utile per memorizzare grandi quantità di dati esternamente al vostro gioco. Contrariamente ai file ini, potete avere fino a 32 file di testo aperti contemporaneamente, e potete anche leggere e scrivere grandi quantità di dati in essi. Tuttavia, sta a voi decidere come i dati debbano essere strutturati, dato che non c'è una struttura "sezione" o "chiave" come nei file*.ini .

Si prega di notare che per i giochi con localizzazioni è molto importante che le lettere accentate possano essere lette da file esterni. Questo significa che dovreste prima creare il file txt in formato UTF8 e poi aggiungerlo in GameMaker come file incluso in modo che venga esportato durante l'esecuzione del gioco e usato al posto del file txt di default in formato ANSI che viene creato dalle funzioni file di GameMaker quando nessun file viene precedentemente fornito. In questo modo, è possibile leggere e scrivere correttamente con tutti gli accenti e le lettere non romane mantenute.

Esistono le seguenti funzioni che si occupano di file:

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
