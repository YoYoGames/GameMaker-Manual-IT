controllare le autorizzazioni

Con questa funzione potete controllare se un permesso specifico è stato concesso al gioco dall'utente. Si fornisce il permesso da controllare come string usando il formato "android.permission.<permission>", quindi per controllare il permesso RECORD_AUDIO (per esempio) si chiama

os_check_permission("android.permission.RECORD_AUDIO");

La funzione restituirà una delle tre costanti - mostrate di seguito - per dire al gioco come lo stato del permesso. Per ulteriori informazioni sui permessi delle app, consultare la documentazione di Android.

IMPORTANTE! Questa funzione è solo per il target Android.

Sintassi:

os_check_permission(permission)

Argomento Tipo Descrizione
permission Stringa The permission to check (a string)

Ritorna:

Permission State Constant

Costante di stato del permesso
Costante Descrizione
os_permission_granted Questo indica che il permesso è stato concesso
os_permission_denied Questo indica che il permesso non è stato concesso
os_permission_denied_dont_request Questo indica che il permesso è stato bloccato dalle impostazioni del telefono, o che l'utente ha precedentemente negato la richiesta e selezionato "Don't ask again".

Esempio:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

Il codice di cui sopra controlla il tipo di sistema operativo e se è Android, esegue un controllo sui permessi e se il permesso "INTERNET" non è ancora stato concesso, lo richiede.
