os_get_info

Questa funzione restituisce un file DS Map con informazioni dettagliate sul sistema operativo su cui è in esecuzione il gioco. Le informazioni esatte restituite dipendono dal sistema operativo e dal dispositivo, quindi si consiglia di testare questa funzione su tutte le piattaforme a cui si vuole puntare e di restituire i valori restituiti usando una funzione come show_debug_message(). Si noti che la mappa DS non viene automaticamente cancellata dalla memoria e si dovrebbe usare la funzione ds_map_destroy() quando non è più necessaria.

Di seguito sono riportati alcuni dettagli sui dati aggiuntivi che saranno resi disponibili per la mappa DS specifici per le seguenti piattaforme; si noti che su tutte le piattaforme , tranne HTML5, la mappa avrà la chiave"is64bit", che sarà true o false a seconda che il runner sia in esecuzione in modalità a 64 bit o meno.

Windows / UWP

Su Windows Desktop e Windows UWP, si otterranno alcuni dati extra da DX11 con i seguenti tasti disponibili nella mappa DS (utili per extensions):

  • udid: Un identificatore unico per la macchina
  • video_d3d11_device: puntatore al dispositivo DX11
  • video_d3d11_context: puntatore al contesto DX11
  • video_d3d11_spawnchain: puntatore alla catena di swap DX11
  • video_adapter_vendorid: string con il vendor id dell'adattatore
  • video_adapter_deviceid: string con l'id del dispositivo dell'adattatore
  • video_adapter_subsysid: string con l'id del sottosistema dell'adattatore
  • video_adapter_revision string con il numero di revisione dell'adattatore.
  • video_adapter_description: string con la descrizione dell'adattatore video
  • video_adapter_dedicatedvideomemory: string con la dimensione della memoria video
  • video_adapter_dedicatedsystemmemory: string con la dimensione della memoria di sistema (utilizzata dall'adattatore DX11)
  • video_adapter_sharedsystemmemory: string con le dimensioni della memoria di sistema condivisa

macOS

La funzione restituisce alcune informazioni OpenGL limitate su macOS:

  • udid: Un identificatore unico per la macchina
  • gl_vendor_string: GL_VENDOR da OpenGL, esempio - "Intel Inc.".
  • gl_version_string: GL_VERSION da OpenGL, esempio - "2.1 INTEL-16.2.16"
  • gl_renderer_string GL_RENDERER da OpenGL, ad esempio "Intel(R) Iris(TM) Graphics 6100".

Ubuntu

Contiene le stesse informazioni contenute nel sito macOS.

Android

Sulla piattaforma Android si ottengono alcuni dati extra da diverse API con le seguenti chiavi disponibili nella mappa DS:

  • android_tv: questo restituirà true se il dispositivo è stato rilevato come dispositivo TV Android o false altrimenti
  • GL_VERSION: la versione di OpenGL come riportata dal driver
  • GL_VENDOR: il fornitore di OpenGL come riportato dal driver
  • GL_RENDERER: il renderer di OpenGL come riportato dal driver
  • GL_EXTENSIONS: il sito extensions di OpenGL disponibile, come riportato dal driver.
  • GL_SHADING_LANGUAGE_VERSION: la versione di GLSL supportata da questo driver OpenGL.
  • GL_MAX_TEXTURE_SIZE: la dimensione massima di texture supportata da questo driver OpenGL.
  • SDK_INT: valore di android.os.Build.VERSION.SDK_INT (vedi nota sotto)
  • RELEASE: valore di android.os.Build.VERSION.RELEASE
  • MODELValore di android.os.Build.MODEL
  • DEVICEValore di android.os.Build.DEVICE
  • MANUFACTURERValore di android.os.Build.MANUFACTURER
  • CPU_ABI: valore di android.os.Build.CPU_ABI
  • CPU_ABI2: valore di android.os.Build.CPU_ABI2
  • BOOTLOADER: valore di android.os.Build.BOOTLOADER
  • BOARD: valore di android.os.Build.BOARD
  • VERSION: valore di os.version dal metodo Android System.getProperty.
  • REGION: valori di user.region dal metodo Android System.getProperty
  • VERSION_NAME: valore del nome della versione di questo pacchetto - vedere qui per maggiori informazioni.
  • PHYSICAL_KEYBOARDSe pensiamo che sia disponibile una tastiera fisica allora string "TRUE" altrimenti "FALSE"

NOTA: Per informazioni sulle variabili android.os.Build, consultare la documentazione di Android Developer.

iOS e tvOS

Ecco le chiavi restituite su iOS e tvOS (per informazioni dettagliate, consultare la documentazione per sviluppatori Apple):

  • name: string come riportato da [[UIDevice currentDevice] name]
  • systemName: string come riportato da [[UIDevice currentDevice] systemName]
  • systemVersion: stringa riportata da [[UIDevice currentDevice] systemVersion]
  • model: stringa riportata da [[UIDevice currentDevice] model]
  • localisedModel: stringa riportata da [[UIDevice currentDevice] localizedModel]
  • uiIdiom: stringa riportata da [[UIDevice currentDevice] userInterfaceIdiom]
  • platform: hw.machine string raccolto da sysctlbyname
  • hwModel: hw.model string raccolto da sysctlbyname
  • freeDiskSpace: Lo spazio libero su disco
  • totalDiskSpace: Lo spazio totale su disco
  • cpuFrequency: La frequenza della CPU
  • busFrequency: La frequenza del bus
  • cpuCount: Il conteggio della CPU
  • totalMemory: La memoria totale del dispositivo
  • userMemory: La memoria utente
  • maxSocketBufferSize: La dimensione massima della presa buffer
  • Chiavi aggiuntive contenenti informazioni sulla grafica OpenGL

PlayStation 4

Per informazioni sulle funzioni qui menzionate, consultare la documentazione di PS4.

La funzione restituisce i seguenti tasti su una console PlayStation 4:

  • display_safe_area_ratio: Un valore in virgola mobile restituito da sceSystemServiceGetDisplaySafeAreaInfo().
  • is_neo_mode: Questo sarà 1 se il gioco gira su una Neo (PS4 Pro) e 0 se gira su una PS4 normale.
  • enter_button_assign: Un valore intero restituito da sceSystemServiceParamGetInt(SCE_SYSTEM_SERVICE_PARAM_ID_ENTER_BUTTON_ASSIGN, &enterassign); un valore di 0 significa che il pulsante Cerchio viene utilizzato come "assegnazione" e 1 significa che viene utilizzato il pulsante X.

PlayStation 5

La funzione restituisce i seguenti tasti su una console PlayStation 5:

  • display_safe_area_ratio: Come per la PS4 (vedi sopra).
  • display_resolution: "HD", "4K" o "sconosciuto".
  • display_dynamic_range: "HDR", "SDR" o "sconosciuto".
  • display_refresh_rate: "120Hz", "60Hz" o "Sconosciuto".

Xbox One

Le informazioni restituite su Xbox One sono per lo più le stesse di Windows, tuttavia i tasti video_adapter_* e udid sono 0 (tranne video_adapter_description che è un string "" vuoto). Inoltre non è presente il tasto video_d3d11_swapchain.

Questa piattaforma restituisce anche le seguenti informazioni:

  • is_xbox_one_x: Questo sarà 1 se il gioco viene eseguito su una Xbox One X e 0 se viene eseguito su una Xbox One di base.

Xbox Serie X/S

Le informazioni riportate sulle console Xbox Series X/S sono per lo più le stesse di Xbox One, tuttavia i tasti video_d3d11_* sono stati sostituiti da video_d3d12_*.

Ecco alcune chiavi che questa piattaforma restituisce in aggiunta a quelle elencate alla voce Xbox One:

  • video_d3d12_cmdqueue: Punta alla coda dei comandi DX12
  • video_d3d12_cmdlist: Punta all'elenco dei comandi DX12
  • video_d3d12_currentrt: Punta al target di rendering DX12 corrente

Sintassi:

os_get_info()

Restituzione:

DS Map ID

Esempio:

if (os_type == os_android)
{
    var _info = os_get_info();
    
    if (_info[? "android_tv"])
    {
        global.android_tv = true;
    }
}

Il codice precedente controlla se il sistema operativo corrente è Android, ottiene le informazioni sul sistema operativo e legge la mappa restituita per verificare se il gioco è in esecuzione su un televisore Android; in tal caso imposta la variabile personalizzata global.android_tv su true.
