esterno_definire

Questa funzione può essere usata per definire una chiamata di funzione esterna a una specifica dll (per Windows) o dylib (per Mac). Questo file può essere un file incluso o parte di un extension.

Si fornisce il nome (e path) del file, poi il nome della funzione che si desidera definire. Poi dovete definire la convenzione di chiamata da usare (vedete l'elenco delle costanti qui sotto) e il tipo di risultato che ci si aspetta (anch'esso una costante, come elencato qui sotto).

Infine dovete dare il numero di argomenti che la funzione può prendere (da 0 a 15) e per ognuno degli argomenti dovete specificare anche il suo tipo. Notate che per le funzioni con 4 o più argomenti, tutti devono essere di tipo ty_real.

NOTA: Questo è solo per dll o dylib che sono state aggiunte come "Included Files" a GameMaker IDE. Non funzionerà con quei file aggiunti come extensions, poiché questi richiedono che si definiscano le funzioni nel pacchetto extension stesso.

Costanti di tipo di chiamata esterna
Costante Descrizione
dll_cdecl Questa è la chiamata predefinita C, C++
dll_stdcall Questa è la chiamata standard WinAPI (soloWindows dll )
Costanti esterne Data Type
Costante Descrizione
ty_real Un argomento numero reale
ty_string Un argomento null-terminato string

Sintassi:

external_define(dll, name, calltype, restype, argnumb, argtype[0], argtype[1], ...argtype[10]);

Argomento Tipo Descrizione
dll Stringa The name of the DLL file (string)
name Stringa The name of the function (string)
calltype Costante di tipo di chiamata esterna (dll_*) The calling convention used
restype Costante esterna Data Type (ty_*) The type of the result to expect
argnumb Real The number of arguments (0 - 10)
argtype[0 ... 10] Costante esterna Data Type (ty_*) The types of the arguments being used

Ritorna:

External Function

Esempio:

my_funcion = external_define("MyDLL.dll", "MyMin", dll_cdecl, ty_real, 2, ty_real, ty_real);

Il codice dell'esempio precedente definirà una funzione esterna chiamata "MyMin" con due argomenti.
