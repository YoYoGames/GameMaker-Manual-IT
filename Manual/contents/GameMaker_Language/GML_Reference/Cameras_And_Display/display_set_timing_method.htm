metodo di temporizzazione

Questa funzione può essere utilizzata per impostare il metodo di temporizzazione da utilizzare per il rendering del gioco. Il metodo può essere una delle seguenti costanti:

Costante Descrizione
tm_sleep Utilizzare il valore del margine di sonno come metodo di temporizzazione principale
tm_countvsyncs Usa il timing vsync come metodo di timing principale (predefinito per tutte le piattaforme supportate)

Il metodo di temporizzazione vsync utilizza il supporto della piattaforma di destinazione per la sincronizzazione verticale per fornire un'ancora per i calcoli di temporizzazione del rendering del gioco, mentre l'impostazione della modalità su sleep margin cercherà semplicemente di garantire che ogni fotogramma duri per la giusta quantità di tempo (diciamo 1/30 o 1/60 di secondo) aspettando o dormendo. In generale la temporizzazione vsync di default darà i risultati più fluidi, ma si noti che quando si utilizza il metodo vsync, i valori di margine di sonno sono ancora rilevanti, anche se avranno un impatto ridotto e si consiglia di tenerlo al valore di default quando si utilizza questo metodo.

Per impostazione predefinita su tutte le piattaforme tranne PS4, Ubuntu e HTML5, GameMaker utilizzerà il metodo di temporizzazione vsync, mentre sulle piattaforme non supportate, può essere utilizzato solo il temporizzazione sleep margin.

Se volete impostare il margine di sonno, potete farlo usando la funzione display_set_sleep_margin() e potete trovare quale metodo di temporizzazione è attualmente utilizzato con la funzione display_get_timing_method().

Sintassi:

display_set_timing_method(method);

Argomento Tipo Descrizione
method Metodo di temporizzazione del display Costante The timing method to use (see the list of constants, above)

Ritorna:

N/A

Esempio:

if display_get_timing_method() != tm_sleep
{
    display_set_timing_method(tm_sleep);
    if display_get_sleep_margin() != 20
    {
        display_set_sleep_margin(20);
    }
}

Il codice di cui sopra controllerà il metodo di temporizzazione e poi se non è impostato su tm_sleep sarà impostato su quello e il margine di sonno impostato su 20.
