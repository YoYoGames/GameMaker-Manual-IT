creazione di un giunto fisico

Il giunto a carrucola è usato per collegare due istanze all'interno del mondo fisico in modo tale che muovendone una si influenzi direttamente il movimento dell'altra. Questi giunti sono prima ancorati nello spazio del mondo in due punti, ognuno collegato a un'istanza al suo centro di massa. Questo può essere cambiato impostando le coordinate locali di ancoraggio x/y relative all'origine dell'istanza, il che significa che la connessione effettiva dall'istanza al punto di ancoraggio del mondo fisico può essere spostata da qualche parte diversa dalla sua origine. L'immagine seguente illustra questo:

Se guardate inst2 potete vedere che il suo punto di ancoraggio locale è stato spostato di 20 pixel a "sinistra" della sua origine (Ricordate! I punti di ancoraggio del mondo sono definiti usando le coordinate room, mentre i punti di ancoraggio locali sono definiti relativamente all'origine dell'istanza). Potete anche specificare un rapporto per il giunto della puleggia, che dice a un lato o all'altro di muoversi più velocemente, con un valore inferiore a 1 che fa muovere più velocemente la prima istanza (ad esempio 0,5 la farà muovere due volte più velocemente) e un valore superiore a 1 che fa muovere più velocemente la seconda istanza (ad esempio 2 la farà muovere due volte più velocemente).

Come per tutti i giunti, se impostate il valore "col" su true allora le due istanze possono interagire e collidere tra loro ma solo se hanno eventi di collisione, invece se è impostato su false, non collideranno in nessun caso.

Sintassi:

physics_joint_pulley_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, l_anchor1_x, l_anchor1_y, l_anchor2_x, l_anchor2_y, ratio, col)

Argomento Tipo Descrizione
inst1 ID istanza The first instance to connect with the joint
inst2 ID istanza The second instance to connect with the joint
w_anchor1_x Real The x coordinate where the first part of joint is anchored, within the game world
w_anchor1_y Real The y coordinate where the first part of the joint is anchored, within the game world
w_anchor2_x Real The x coordinate where the second part of the joint is anchored, within the game world
w_anchor2_y Real The y coordinate where the second part of the joint is anchored, within the game world
l_anchor1_x Real The local x coordinate where joint 1 is anchored to the first instance
l_anchor1_y Real The local y coordinate where joint 1 is anchored to the first instance
l_anchor2_x Real The local x coordinate where joint 2 is anchored to the second instance
l_anchor2_y Real The local y coordinate where joint 2 is anchored to the second instance
ratio Real Set the velocity ratio between two instances
col Booleano Whether the two instances can collide (true) or not (false)

Ritorna:

Physics Joint ID

Esempio:

var t_fix, inst1, inst2;
t_fix = physics_fixture_create();
physics_fixture_set_circle_shape(t_fix, 20);
inst1 = instance_create_layer(150, room_height - 90, "Instances", obj_Block);
inst2 = instance_create_layer(300, room_height - 90, "Instances", obj_block);
physics_fixture_bind(t_fix, inst1);
physics_fixture_bind(t_fix, inst2);
physics_joint_pulley_create(inst1, inst2, 150, room_height - 140, 300, room_height - 140, 0, 0, 0, 0, 2, 0);
physics_fixture_delete(t_fix);

Il codice qui sopra crea e definisce una nuova fixture, crea due istanze di "obj_Block", e poi lega questa fixture a loro. Continua poi a definire un giunto di puleggia tra queste istanze, senza offset per i giunti, un rapporto di 2:1 (il che significa che inst2 si muoverà più velocemente). Nessuna collisione avviene tra le due istanze nella puleggia.
