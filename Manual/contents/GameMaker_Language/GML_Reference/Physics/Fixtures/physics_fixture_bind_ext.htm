physics_fixture_bind_ext

Una volta che abbiamo definito la nostra fixture deve essere legata ad un'istanza. Questo significa che le sue proprietà sono trasferite all'istanza selezionata, non alla fixture stessa, così che una fixture può essere legata a più istanze se tutte devono avere le stesse proprietà. Potete specificare un indice object per il target e tutte le istanze presenti in room in quel momento riceveranno le proprietà di quella fixture (ma non qualsiasi nuova istanza di object creata successivamente), oppure potete usare le parole chiave speciali other e all. Potete anche specificare un genitore object e tutte le istanze figlie con quel genitore riceveranno anche la fixture. Una volta che la fixture è stata legata a tutte le istanze di cui avete bisogno, può essere cancellata se non è più necessaria e le istanze con le proprietà di quella fixture non saranno interessate e manterranno quelle proprietà.

Normalmente, l'attrezzatura sarà legata all'istanza con il centro di massa posizionato all'origine dell'istanza, tuttavia questo non è sempre ciò che si richiede e quindi questa funzione permette anche di spostare la posizione x e y in cui l'attrezzatura è legata di una data quantità (se non avete bisogno di questo allora usate physics_fixture_bind() invece). È importante notare che una fixture può supportare solo un singolo offset, poiché l'aggiunta di più offset a una singola fixture non è supportata da Box2D.

La funzione restituirà anche un valore "id" unico per la fixture legata(non la fixture stessa) che può poi essere usato per rimuovere ("un-bind") le proprietà fisiche dall'istanza usando la funzione physics_remove_fixture(). Questo permette di aggiungere e rimuovere le proprietà fisiche da un'istanza senza distruggere e ricreare objects.

NOTA: Le fixture dovrebbero essere cancellate quando non sono più necessarie, poiché la mancata cancellazione può causare una perdita di memoria che rallenterà ed eventualmente farà crashare il gioco.

Sintassi:

physics_fixture_bind_ext(fixture, target, xoffset, yoffset)

Argomento Tipo Descrizione
fixture ID apparecchio fisico The fixture that is to be bound
target ID dell'istanza o dell'oggetto Asset The target instance that is to receive the fixture (can be an instance id, an object id, other, or all)
xoffset Real The offset along the x-axis
yoffset Real The offset along the y-axis

Ritorna:

Real

Esempio:

var fix, inst;
fix = physics_fixture_create();
physics_fixture_set_circle_shape(fix, 16);
physics_fixture_set_density(fix, 1.0);
inst = instance_create_layer(x, y, "Instances", genericBodyObject);
my_fix = physics_fixture_bind_ext(fix, inst, sprite_width / 2, -(sprite_height / 2));
physics_fixture_delete(fix);

Il codice qui sopra creerà una fixture e assegnerà il suo indice alla variabile "fix". Poi definisce la forma e la densità della fixture prima di legarla all'istanza con un offset basato sulla larghezza e l'altezza di sprite. L'indice per la fixture legata è memorizzato nella variabile "my_fix". Infine, la fixture stessa viene cancellata per evitare perdite di memoria quando non è più necessaria.
