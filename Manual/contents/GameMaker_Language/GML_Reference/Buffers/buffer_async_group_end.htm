buffer_async_group_end

Questa funzione termina la definizione di un gruppo buffer async. Dovete aver precedentemente chiamato la funzione buffer_async_group_begin() per avviare il gruppo, quindi chiamare la funzione buffer_save_async() per ogni file che volete salvare (o buffer_load_async() per caricare buffers). Infine chiamate questa funzione, che avvierà il salvataggio dei file. La funzione restituirà un valore ID unico per il salvataggio, che può essere utilizzato nell'evento Asynchronous Save / Load per analizzare i risultati dalla async_load mappa DS.

Sintassi:

buffer_async_group_end();

Ritorna:

Async Request ID

Esempio esteso:

La funzione buffer_async_group_end() può essere chiamata da qualsiasi evento, e poiché è asincrona il callback può essere quasi istantaneo o potrebbe richiedere diversi secondi. Chiamare la funzione è semplice e sarebbe qualcosa di simile a questo:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
save_id = buffer_async_group_end();

Il codice qui sopra avvia un gruppo buffer poi lo imposta per salvare 4 file in modo asincrono. La definizione del gruppo viene poi terminata (a quel punto inizierà il salvataggio), memorizzando l'ID della chiamata alla funzione nella variabile "save_id". Quando il salvataggio è completo, l'evento asincrono Save/Load verrà attivato e sarà possibile analizzare la mappa async_load per l'ID corretto della funzione, in questo modo:

if ds_map_find_value(async_load, "id") == saveid
{
    if (ds_map_find_value(async_load, "status") == false)
    {
        show_debug_message("Save failed!");
    }
}

Il codice qui sopra controllerà prima l'id della mappa DS che è stata creata, poi controllerà lo stato del callback, inviando un messaggio di debug se ci sono stati problemi.
