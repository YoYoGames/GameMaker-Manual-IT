carico_del_buffer

Questa funzione è usata per caricare un buffer che è stato precedentemente salvato usando le funzioni buffer_save() così come qualsiasi file incluso o caricato esternamente. Restituirà un nuovo ID del buffer che viene creato dalla funzione come "grow" buffer (vedi qui) con un allineamento di byte di 1. Questo ID dovrebbe essere memorizzato in una variabile e utilizzato in tutte le ulteriori chiamate di funzioni a questo buffer. Se il caricamento fallisce per qualsiasi motivo, la funzione restituisce -1 e viene mostrato un messaggio nella finestra di output del compilatore che dice che il caricamento è fallito.

NOTA È importante rimuovere qualsiasi risorsa creata dinamicamente come questa dalla memoria quando non ne hai più bisogno per evitare perdite di memoria, quindi quando hai finito con il buffer che hai creato dovresti liberarlo di nuovo usando buffer_delete().

Note specifiche per la piattaforma

  • Su HTML5, qualsiasi buffers caricato dalla memoria locale sarà decodificato usando base64 (poiché sono salvati come base64 strings), tuttavia, usando questa funzione su Included Files o qualsiasi file esterno non userà la decodifica base64, poiché non ci si aspetta che siano in un formato base64.
  • Su HTML5, tutti i file inclusi sono memorizzati come testo codificato UTF-8, il che può comportare un aumento della dimensione del buffer caricato rispetto al file originale, poiché la versione codificata UTF-8 del file può avere una dimensione diversa da quella originale (a causa del cambiamento di formato).

Sintassi:

buffer_load(filename);

ArgomentoTipo Descrizione
filename The name of the file to load from.

Ritorna:

Buffer ID

Esempio:

player_buffer = buffer_load("Player_Save.sav");

Il codice qui sopra caricherà in memoria un buffer precedentemente salvato, creando un nuovo buffer. L'indice di questo nuovo buffer è memorizzato nella variabile "player_buffer".
