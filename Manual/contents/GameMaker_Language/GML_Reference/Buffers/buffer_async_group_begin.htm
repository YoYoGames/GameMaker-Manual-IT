buffer_async_group_begin

Questa funzione viene chiamata quando si vuole iniziare il salvataggio di più buffers in più file (o il caricamento di più buffers). Il nome del gruppo è un string e sarà usato come nome di directory per dove i file sono salvati, e dovrebbe essere usato come parte del file path quando si caricano di nuovo i file in IDE successivamente (usando una qualsiasi delle buffer_load() funzioni). Questa funzione è da usare solo con le funzioni buffer_save_async() e buffer_load_async() e dovete anche terminare il gruppo asincrono chiamando buffer_async_group_end() altrimenti i file non saranno salvati/caricati.

Notate che per le piattaforme console (come PS4 per esempio), il nome del gruppo sarà usato come descrizione dello slot di salvataggio, e usare questa funzione può aiutarvi ad evitare di avere l'UI che mostra ogni file che viene salvato. Notate anche che quando si usa UWP si può fare il roaming dei giochi salvati tra le finestre native di Xbox e Microsoft store creando un gruppo che è lo stesso di un gruppo ERA Xbox, per esempio:

if os_type == os_uwp
{
    buffer_async_group_begin("root/yourgroupname");
}
else
{
    buffer_async_group_begin("yourgroupname");
}

Sintassi:

buffer_async_group_begin(groupname);

Argomento Tipo Descrizione
groupname Stringa The name of the group (as a string).

Ritorna:

N/A

Esempio:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

Il codice qui sopra inizia un gruppo buffer poi lo imposta per salvare 4 file in modo asincrono. La definizione del gruppo viene poi terminata (a quel punto inizierà il salvataggio).
