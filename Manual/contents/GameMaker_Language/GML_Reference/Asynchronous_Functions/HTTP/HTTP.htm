HTTP

Questa sezione elenca tutte le diverse funzioni HTTP asincrone disponibili in GameMaker. Queste funzioni genereranno un evento HTTP asincrono in tutte le istanze che lo hanno:

  • http_request
  • http_get
  • http_get_file
  • http_post_string

Si prega di notare che le funzioni di cui sopra http_* potrebbero non funzionare come previsto a causa di problemi di sicurezza tra domini. Questo significa che le richieste al tuo server o i tentativi di caricare risorse da altri domini sono bloccati e possono sembrare restituire risultati vuoti o errori 404. Uno dei modi in cui puoi aggirare questo problema è di avere del PHP lato server che permetta a certi domini di accedere al tuo server (questo è anche un modo per proteggere le tue risorse e bloccare servers che non sono incluse nella lista PHP allow).

Il seguente è un esempio di codice PHP che puoi usare per questo:

$http_origin = $_SERVER['HTTP_ORIGIN'];
if ($http_origin == "http://127.0.0.1:51268")
{
    header("Access-Control-Allow-Origin: *");
}

Per le richieste di caricamento di immagini in cui è importante determinare o impostare il loro tipo di origine incrociata, esistono le seguenti funzioni:

  • http_get_request_crossorigin
  • http_set_request_crossorigin

Nella maggior parte dei casi queste funzioni non dovrebbero essere utilizzate, ma se il gioco è memorizzato su un server protetto - dove alcuni assets possono richiedere l'autenticazione di base per essere accessibili e stanno generando errori di sicurezza durante il caricamento - l'impostazione del tipo cross-origin per le richieste di immagini a "use-credentials" potrebbe essere necessario (in contrasto con l'impostazione predefinita "anonymous").
