http_set_request_crossorigin

Con questa funzione è possibile impostare il tipo di origine incrociata da utilizzare quando si caricano immagini da un file. La funzione è esclusivamente per la piattaforma HTML5 ed è utile solo quando si carica sprites da un file usando la funzione sprite_add(). Si noti che quando è impostato su "use-credentials" non è più necessario dare un path assoluto al sprite che viene caricato, ma invece si dovrebbe dare un path relativo (come mostrato nell'esempio qui sotto). Per impostazione predefinita il tipo di origine incrociata è impostato su "anonymous".

Sintassi:

http_set_request_crossorigin(origin_type);

ArgomentoTipo Descrizione
origin_type The cross origin type to use (a string)

Ritorna:

Esempio:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
    http_set_request_crossorigin("use-credentials");
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

Il codice di cui sopra controlla prima il tipo di origine incrociata attualmente impostato e se non è "use-credentials" allora viene impostato su "use-credentials" e poi viene aggiunto un sprite da un file.
