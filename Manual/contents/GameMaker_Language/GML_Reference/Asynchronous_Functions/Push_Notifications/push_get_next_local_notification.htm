push_get_next_local_notification

Questa funzione popolerà una mappa DS pre-fatta con una serie di coppie chiave/valore per la prossima notifica push locale in queue da mostrare. Restituisce -1 se non ci sono altre notifiche in coda, o un valore reale altrimenti che rappresenta l'ID della notifica. Questo ID può quindi essere utilizzato per cancellare la notifica utilizzando la funzione push_cancel_local_notification(). Dovreste chiamare la funzione push_get_first_local_notification() per ottenere la prima notifica in queue e poi usare questa funzione per continuare attraverso di essa.

NOTA: Questa funzione è limitata ai moduli target iOS e Android.

a mappa DS conterrà le seguenti coppie chiave/valore:

  • "title" - Il titolo della notifica (questo sarà disponibile solo sulla piattaforma Android poiché iOS non memorizza le informazioni sul titolo)
  • "messaggio" - Il testo del corpo del messaggio
  • "data" - La stringa del pacchetto di dati

Sintassi:

push_get_next_local_notification(map);

ArgomentoTipo Descrizione
map The DS Map ID da usare.

Ritorna:

Push Notification ID or -1

Esempio :

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
    {
        push_cancel_local_notification(ntf);
    }
    ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

Il codice di cui sopra crea un DS Map e poi richiede i dati per la prima notifica disponibile nel queue. Se esiste una notifica, la mappa viene popolata e il codice controlla poi la chiave "dati" per uno specifico string. Se la string viene trovata, la notifica viene cancellata e si passa a controllare la notifica successiva fino a quando tutte le queue sono state controllate.
