push_cancel_local_notification

Questa funzione annulla la notifica push data. È possibile ottenere l'ID della notifica da cancellare utilizzando le funzioni push_get_first_local_notification() e push_get_next_local_notification() e la funzione restituirà true in caso di successo o false altrimenti (per esempio se la notifica non esiste o è già stata attivata).

NOTA: Questa funzione è limitata ai moduli target iOS e Android.

Sintassi:

push_cancel_local_notification(ID);

Argomento Tipo Descrizione
ID ID della notifica di spinta The ID of the notification to cancel.

Ritorna:

Boolean

Esempio :

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
    {
        push_cancel_local_notification(ntf);
    }
    ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

Il codice di cui sopra crea un DS Map e poi richiede i dati per la prima notifica disponibile nel queue. Se esiste una notifica, la mappa viene popolata e il codice controlla poi la chiave "dati" per uno specifico string. Se la string viene trovata, la notifica viene annullata e si passa a controllare la notifica successiva fino a quando tutte le queue sono state controllate.
