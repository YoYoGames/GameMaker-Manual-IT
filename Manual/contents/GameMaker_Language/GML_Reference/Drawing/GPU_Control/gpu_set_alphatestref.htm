gpu_set_alphatestref

Questa funzione imposterà il valore di riferimento per il test alfa (quando è stato abilitato usando il tasto gpu_set_alphatestenable()). Questa è la soglia di "cut-off" alla quale i pixel con alfa non saranno disegnati.

Fondamentalmente, se avete questo valore impostato a 0 e avete l'interpolazione lineare attivata (sia nelle Opzioni di gioco che usando gpu_set_texfilter()), si possono ancora avere errori visivi ai bordi delle sovrapposizioni objects. Questo perché si possono avere aree quasi completamente trasparenti che bloccano le cose disegnate dietro di loro, ma se si imposta questo valore di riferimento, per esempio, a 254 (cioè uno sotto il massimo), questo garantisce che solo il 100% dei pixel opachi saranno disegnati. Potete anche disattivare l'interpolazione texture, il che vi permetterebbe normalmente di usare i valori di default senza problemi e senza bisogno di usare questa funzione, ma molto dipende dalla grafica di partenza.

NOTA: Questa è una funzione di disegno avanzata, quindi se non siete sicuri di averne bisogno o meno, o non sapete esattamente cosa fa, probabilmente fareste meglio a non usarla perché può seriamente corrompere il modo in cui il vostro gioco è disegnato.

Sintassi:

gpu_set_alphatestref(val);

ArgomentoTipo Descrizione
val Sets the reference value for alpha testing from 0 to 255 (default value is 0)

Ritorna:

Esempio:

if !gpu_get_alphatestenable()
{
    gpu_set_alphatestenable(true);
    gpu_set_alphatestref(128);
}

Il codice qui sopra controllerà per vedere se l'alpha testing è abilitato e se non lo è switch su alpha testing e imposterà la soglia di test a 128 (solo i pixel con un alpha superiore a 0.5 saranno disegnati).
