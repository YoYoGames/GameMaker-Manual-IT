vertex_format_add_custom

Questa funzione permette di utilizzare un data type personalizzato per specifici attributi del formato dei vertici come parte del nuovo formato dei vertici che viene creato. I valori disponibili da utilizzare sono definiti dalla costante data type che si sceglie, elencata di seguito:

Vertice Data Type Costante
Costante Descrizione
vertex_type_float1 Un singolo valore in virgola mobile
vertex_type_float2 Due valori in virgola mobile
vertex_type_float3 Tre valori in virgola mobile
vertex_type_float4 Quattro valori in virgola mobile
vertex_type_colour Valori di quattro componenti (r, g, b, a)
vertex_type_ubyte4 Quattro componenti valori di byte senza segno (da 0 a 255)


Anche l'uso che verrà fatto di queste costanti deve essere definito in modo che i valori possano essere "legati" correttamente all'interno di shader che viene creato. Questo è necessario a causa del fatto che DX e OpenGL hanno requisiti diversi, quindi se non li si lega correttamente, non verranno visualizzati correttamente in shader. Le costanti d'uso disponibili che si possono scegliere sono elencate di seguito e quelle che si utilizzano dipendono dalle specifiche di shader che si sta creando:

Tipo di utilizzo dei vertici Costante
Costante Descrizione
vertex_usage_position valori di posizione (x, y, z)
vertex_usage_colour valori di colore (r, g, b, a)
vertex_usage_normal valori normali dei vertici (nx, ny, nz)
vertex_usage_textcoord Coordinate UV (u, v)
vertex_usage_blendweight il blendweight della matrice di input (per l'animazione scheletrica, per esempio)
vertex_usage_blendindices gli indici delle matrici da usare (per l'animazione scheletrica, per esempio)
vertex_usage_depth profondità dei vertici buffer valore
vertex_usage_tangent valori tangenti
vertex_usage_binormal valori binormali
vertex_usage_fog valori di nebbia
vertex_usage_sample indice del campionatore


Ci sono alcune cose importanti da notare quando si usano formati personalizzati come questi:

  • La funzione vertex_format_add_custom() supporta solo vertex_usage_position, vertex_usage_colour, vertex_usage_normal e vertex_usage_textcoord quando si usa GLSL shaders. Questi mappano rispettivamente agli attributi shader in_Position , in_Colour[0 - ...], in_Normal (tutto ciò che non è uno di questi tre attributi - ad esempio: texture coordinate - può essere mappato a qualsiasi attributo definito).
  • In generale dovreste usare vertex_usage_textcoord per tutti i parametri extra dove possibile, poiché tipi come vertex_usage_blendweight e vertex_usage_tangent sono quasi deprecati nella maggior parte dei linguaggi shader, e probabilmente non saranno convertiti correttamente. Invece usate vec, vec2, vec3 o vec4 tipi vertex_usage_textcoord e tutto dovrebbe funzionare bene.
  • GLSL ES non supporta gli attributi interi, quindi passare in ivec4's non funziona (questo tipo è di solito usato quando si passa in vertex_usage_blendindices). Quello che dovete fare è passare in texture coordinate e poi nel shader, convertirle in ivec4 come questo:

attribute vec3 in_Position;
attribute vec4 in_BlendIndices;
attribute vec4 in_BlendWeights;

varying vec4 v_vColour;
varying mat4 v_mat;

void main()
{
    gl_Position = gm_Matrices[MATRIX_WORLD_VIEW_PROJECTION] * vec4( in_Position.xyz, 1.0);
    v_vColour = in_BlendWeights;
     ivec4 t = ivec4(in_BlendIndices);
     v_mat = gm_Matrices[ t.x ];
}

  • I pesi delle miscele sono di solito memorizzati in una matrice e poi vi si accede usando gli indici delle miscele, ma potete vedere qui che invece di definire in_BlendIndices come un attributo ivec4, è un vec4, poi castato in un ivec4 nel codice. Questo viene poi usato per indicizzare l'array creato usando il gm_Matrix (si può accedere ad un array solo usando un valore INT - non un float).

Sintassi:

vertex_format_add_custom(type, usage);

Argomento Tipo Descrizione
type Vertice Data Type Costante The data type that this custom vertex data will hold (see the type constants listed above).
usage Tipo di utilizzo dei vertici Costante The use that the data will get(see the usage constants listed above).

Ritorna:

N/A

Esempio:

vertex_format_begin();
vertex_format_add_textcoord();
vertex_format_add_custom(vertex_type_float3, vertex_usage_position);
my_format = vertex_format_end();

Il codice qui sopra creerà un nuovo formato di vertice con solo texture e 3 valori in virgola mobile personalizzati per la posizione. Viene poi memorizzato l'id del formato nella variabile "my_format".
