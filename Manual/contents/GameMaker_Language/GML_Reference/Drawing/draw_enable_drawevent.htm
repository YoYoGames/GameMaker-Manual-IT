disegna_abilita_dravviamento

Con questa funzione potete scegliere di abilitare (true) o disabilitare (false) l'evento draw per tutte le istanze del gioco, dandovi così il controllo su come e quando le cose vengono disegnate, il che è utile se volete implementare una tecnica di "salto di frame". Notate che questo non solo impedisce alle istanze di disegnare sullo schermo, ma sopprime completamente l'evento draw, il che significa che bisogna fare attenzione perché qualsiasi logica di gioco che è presente in quell'evento non sarà nemmeno eseguita.

Una cosa importante da capire su questa funzione è che se la chiamate proprio all'inizio del gioco, prima che il frame iniziale venga renderizzato (cioè: l'evento Create del primo object nel primo room del gioco), allora la finestra di gioco non verrà renderizzata. Questo potrebbe essere utile per progetti multiplayer che richiedono un server dedicato che non ha bisogno di renderizzare nulla, tuttavia tenete a mente che questo non lo rende headless in quanto GameMaker non lo supporta -- quindi non sarete in grado di eseguirlo su un server che non ha un'interfaccia grafica.

Sintassi:

draw_enable_drawevent(enable);

ArgomentoTipo Descrizione
Enable Set to true o false.

Ritorna:

Esempio:

frame_skip ++;
if (frame_skip mod 5) == 0
{
    draw_enable_drawevent(true);
}
else
{
    draw_enable_drawevent(false);
}

Il codice qui sopra controlla una variabile e se ritorna true, abilita l'evento draw, altrimenti l'evento draw è disabilitato.
