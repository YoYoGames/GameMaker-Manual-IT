gc_get_stats

Con questa funzione potete recuperare informazioni sullo stato attuale del garbage collector. La funzione restituirà una struct che avrà le seguenti variabili membro (si noti che "oggetti" qui si riferisce a qualsiasi cosa che può essere oggetto di garbage collector e non a istanze generali di object come definito nel browser Asset ):

Strutture GC Stats
Variabile Tipo Descrizione
objects_touched Real Questo è il numero di objects attivi che il garbage collector ha trovato nel frame precedente. Questo varia a seconda di quale generazione è stata raccolta.
objects_collected Real Il numero di objects che il garbage collector ha determinato che non erano attivi nel frame precedente, e che quindi possono essere cancellati.
traversal_time Real Questo è il tempo in microsecondi (sul thread principale) che il garbage collector ha impiegato per capire quali objects erano attivi.
collection_time Real Questo è il tempo in microsecondi (su un thread separato) che il garbage collector ha impiegato per ripulire objects ritenuto inattivo.
gc_frame Real Questo è un contatore che viene incrementato ogni volta che si verifica un passaggio di garbage collection. Se la garbage collection è disabilitata, questo non aumenta.
generation_collected Real Questo è l'indice della generazione che è stata raccolta per ultima. 0 è la generazione più giovane e 3 è attualmente la più vecchia.
num_generations Real Questo è il numero totale di generazioni di garbage collection.
num_objects_in_generation Array di Reales Questo è un array (di dimensione num_generations) che contiene il numero di objects in ogni generazione.

NOTA: sulla piattaforma di destinazione HTML5 la garbage collection è gestita dal motore JavaScript e quindi le variabili membro nel precedente struct restituiranno tutte 0 se questa funzione è usata su quella piattaforma.

Quando si usa questa funzione, si prega di notare che le informazioni mostrate per objects sono aggiornate solo quando un'intera generazione ha finito di essere elaborata, il che può richiedere diversi fotogrammi a seconda dell'impostazione del tempo di frame (vedere qui per maggiori informazioni sul tempo di frame).

Sintassi:

gc_get_stats();

Ritorna:

GC Stats Struct

Esempio:

if (global.debug == true)
{
    var _s = gc_get_stats();
    var _t = _s.traversal_time;
    var _c = _s.collection_time;
    show_debug_message("Traversal time = " + string(_t))
    show_debug_message("Collection time = " + string(_c))
}

Il codice qui sopra controlla una variabile globale e se è true ottiene informazioni dal garabge collector e le invia alla console come messaggi di debug.
