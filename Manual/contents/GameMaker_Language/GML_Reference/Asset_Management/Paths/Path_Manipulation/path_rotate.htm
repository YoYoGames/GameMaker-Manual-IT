percorso_rotate

Potete usare questa funzione per ruotare un dato path intorno al suo centro. Ricordate che in GameMaker (a meno che non stiate usando la fisica) gli angoli sono calcolati in senso antiorario, quindi ruotando il path di 90 gradi ruoterebbe il path a sinistra. Questa funzione cambia l'attuale risorsa path , e quindi influenzerà permanentemente il modo in cui path è usato da tutte le istanze del gioco dal momento in cui la funzione viene usata fino alla fine del gioco. Se questo non è ciò di cui avete bisogno, allora dovreste usare una funzione come path_duplicate() per creare prima una copia di path, poi chiamare questa funzione sul duplicato asset (non dimenticate di chiamare path_delete() su asset quando non è più necessario).

NOTA: Una volta che un path è stato assegnato a un'istanza usando la path_start() si può usare la variabile path_orientation per cambiare anche la rotazione di path.

Sintassi:

path_rotate(index, angle);

Argomento Tipo Descrizione
index Percorso patrimoniale The index of the path to flip.
angle Real The angle of rotation degrees.

Ritorna:

N/A

Esempio:

path_rotate(pth_Patrol, 90);

Questo ruoterebbe il dato di 90 gradi in senso antiorario.

Indietro: Manipolazione del percorso
Il prossimo: path_rescale
