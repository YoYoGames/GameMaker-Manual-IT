spostamento del percorso

Con questa funzione potete spostare un path lungo l'asse orizzontale e verticale. Si prega di notare che questo non avrà alcun effetto visibile nel gioco se l'istanza che segue il path che viene spostato lo segue relativamente (piuttosto che assolutamente) poiché la posizione del percorso del percorso non viene presa in considerazione. Questa funzione cambia l'effettivo asset path , e quindi influenzerà permanentemente il modo in cui path è usato da tutte le istanze nel gioco dal momento in cui la funzione viene usata fino alla fine del gioco. Se questo non è non è ciò di cui avete bisogno, allora dovreste usare una funzione come path_duplicate() per creare prima una copia di path, poi chiamare questa funzione sul duplicato asset (non dimenticate di chiamare path_delete() su asset quando non è più necessario).

Sintassi:

path_shift(index, xshift, yshift);

ArgomentoTipo Descrizione
index The index of the path to change.
xshift The number of horizontal pixels to shift the path. Negative=left, positive=right.
yshift The number of vertical pixels to shift the path. Negative=up, positive=down.

Ritorna:

Esempio:

my_path = path_duplicate(pth_AI);
path_shift(my_path, 50, 50);
path_start(my_path, 4, path_action_reverse, true);

Il codice precedente duplica un path asset e memorizza il suo valore ID in una variabile. Questa variabile viene poi usata per spostare la posizione del duplicato path e poi l'istanza inizia il path.

Indietro: Manipolazione del percorso
Il prossimo: path_index
