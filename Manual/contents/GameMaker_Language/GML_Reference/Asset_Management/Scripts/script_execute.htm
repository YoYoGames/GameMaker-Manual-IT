script_execute

Questa funzione è stata originariamente progettata per richiamare le risorse di script in modo dinamico, ma con l'introduzione delle funzioni definite dall'utente può ora essere utilizzata in due modi:

  • Il primo modo è quello di supportare scripts, dove ogni script era una singola funzione e il titolo di script era anche il nome della funzione. Usato in questo modo, funzionerà come prima ed eseguirà un script asset creato dall'utente e gli passerà gli argomenti dati. Si noti che in questo caso non viene chiamato lo script, ma la funzione dello script con lo stesso nome all'interno di script.
  • Il secondo modo consiste nel chiamare una funzione da un sito script asset che contiene più funzioni. In questo caso, si fornirà il nome della funzione e gli argomenti che richiede, anziché il nome script asset. Si noti che questo funziona solo per le funzioni di script e non per i metodi, poiché solo le funzioni di script hanno un ID script assegnato.

Sebbene questa funzione sia stata progettata principalmente per il supporto legacy, non è deprecata, in quanto ha ancora il suo posto nelle situazioni di rete e di chiamata di procedura remota. È bene notare che questa funzione è lenta e il suo uso frequente può influire negativamente sulle prestazioni.

NOTA Questa funzione non può essere utilizzata con le funzioni integrate.

Sintassi:

script_execute(scr, arg0, arg1, arg2..., etc...);

Argomento Tipo Descrizione
scr Attivo dello script o Funzione script The function/script that you want to call
arg0, arg1, arg2..., etc... Variabile The different arguments that you want to pass through to the function/script

Restituzione:

Variable (Will depend on the return value from the script/function being called)

Esempio:

script_execute(choose(move_up, move_down, move_left, move_right), irandom(5));

L'esempio di codice sopra riportato utilizza script_execute per scegliere una delle quattro funzioni definite dall'utente (in questo caso relative al movimento) e le passa un intero casuale come primo argomento.
