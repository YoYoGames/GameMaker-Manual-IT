audio_listener_set_velocity

Questa funzione può essere usata per dare all'ascoltatore dato effetti Doppler e simulare il movimento audio basato sul vettore che si risolve dalle posizioni x, y e z relative date. L'indice predefinito dell'ascoltatore è 0, ma è possibile utilizzare la funzione audio_get_listener_info() per ottenere i diversi indici disponibili per la piattaforma di destinazione.

Se l'ascoltatore dato non si muoverà mai, o il movimento non è un movimento costante, normalmente non è necessario impostare questi valori, ma, per esempio, se si sta facendo un gioco a scorrimento in cui il giocatore ha un movimento costante dal basso verso l'alto e i nemici un movimento costante dall'alto verso il basso, si dovrebbe impostare le velocità dell'ascoltatore e degli emettitori (per gli emettitori si dovrebbe usare audio_emitter_velocity()) ai vettori appropriati per simulare il corretto effetto Doppler mentre si muovono oltre l'istanza del giocatore.

L'immagine qui sotto mostra come questo potrebbe essere impostato per il gioco di esempio dato sopra:

Sintassi:

audio_listener_set_velocity(index, x, y, z);

Argomento Tipo Descrizione
index ID ascoltatorereale o audio The listener to change the velocity of (default 0).
x Real The x velocity of the listener (default 0).
y Real The y velocity of the listener (default 0).
z Real The z velocity of the listener (default 0).

Ritorna:

N/A

Esempio:

var num = audio_get_listener_count();
for(var i = 0; i < num; ++i;)
{
    var info = audio_get_listener_info(i);
    var data = audio_listener_get_data(info[? "index"]);
    if data[? "vx"] != 0
    {
        audio_listener_set_velocity(info[? "index"], 0, 0, 0);
    }
    ds_map_destroy(info);
    ds_map_destroy(data);
}

Il codice di cui sopra controlla il numero di ascoltatori disponibili poi loops attraverso di loro e se la loro velocità x non è 0, imposta i loro valori di velocità a 0, 0, 0.
