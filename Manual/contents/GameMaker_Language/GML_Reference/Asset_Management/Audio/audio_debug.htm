audio_debug

switch Questa funzione può essere usata per mostrare informazioni di debug sul sistema audio, con true che la accende e false che la spegne. Quando state debugging audio in questo modo, la sovrapposizione mostrerà le seguenti informazioni:

  • Handle - L'handle della sorgente OpenAL (unico per ogni suono riprodotto)
  • Data Handle - L'handle OpenAL buffer (mostra lo stesso valore per più suoni con lo stesso handle di origine)
  • Processed Buffers - quanti buffers sono stati giocati
  • Queued Buffers - quanti buffers sono stati messi in coda (e sono ancora in attesa di giocare)
  • Guadagno - Il valore di guadagno per l'audio (0 - 65536)
  • Play Offset - la posizione del campione (posizione corrente di riproduzione) nel buffer corrente
  • Code Buffers Bars - un piccolo diagramma a riquadro dell'elaborato/incollato buffers con l'offset di riproduzione (posizione attuale) disegnato come una linea blu

Per i gruppi sincronizzati di audio, avete anche la funzione audio_sync_group_debug().

NOTA Questa funzione non funziona sulla piattaforma di destinazione HTML5.

Sintassi:

audio_debug(enable);

ArgomentoTipo Descrizione
enable Enable (set to true) o disabilitare (false) l'audio debugging.

Ritorna:

Esempio:

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

Il codice qui sopra switch accenderà o spegnerà la sovrapposizione del debug audio a seconda che il gioco sia in modalità debug o meno.
