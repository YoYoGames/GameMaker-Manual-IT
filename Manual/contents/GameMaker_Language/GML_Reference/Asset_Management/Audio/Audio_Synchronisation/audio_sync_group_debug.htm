audio_sync_group_debug

uesta funzione può essere utilizzata per visualizzare informazioni di debug su qualsiasi gruppo di sincronizzazione dato (l'indice del gruppo è il valore restituito quando il gruppo è stato creato utilizzando la funzione audio_create_sync_group()). Potete chiamare la funzione con un valore di -1 per switch disattivare la sovrapposizione.

Quando siete debugging i gruppi di sincronizzazione audio, la sovrapposizione verrà mostrata:

  • Quale gruppo di sincronizzazione sta eseguendo il debug
  • Quanti canali ci sono dentro
  • Quante volte il gruppo di sincronizzazione è fallito (questo significa che ha cercato di fare troppo lavoro)
  • Una lista dei canali
    • Rosso significa che questo canale non sta attualmente facendo alcun lavoro
    • Giallo significa che il canale è in stato iniziale (non ancora in decodifica o riproduzione)
    • Verde significa che il canale sta attualmente cercando/decodificando
    • Viola significa che il canale ha esaurito i dati
  • Una barra di profiling che mostra il lavoro che il thread di decodifica sta facendo
    • blu significa che un file viene letto per la prima volta
    • rosso significa che il thread sta leggendo un file per scoprire da dove leggere
    • verde significa che il thread sta decodificando i dati
    • viola significa che il thread sta dormendo finché non ha bisogno di lavorare

Notate che per il generale debugging dell'audio, avete anche la funzione audio_debug().

NOTA: Questa funzionalità non è disponibile per la piattaforma di destinazione HTML5.

Sintassi:

audio_sync_group_debug(group_index);

Argomento Tipo Descrizione
group_index ID del gruppo di sincronizzazione audio The group index to debug, or -1 to switch off the debugging.

Ritorna:

N/A

Esempio:

if (debug_mode)
{
    audio_sync_group_debug(sg);
}
else
{
    audio_sync_group_debug(-1);
}

Il codice qui sopra switch accenderà o spegnerà la sovrapposizione del debug del gruppo di sincronizzazione audio mostrando il gruppo di sincronizzazione indicizzato nella variabile "sg" a seconda che il gioco sia eseguito in modalità debug o meno.
