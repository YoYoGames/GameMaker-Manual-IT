oggetto_è_ancestatore

Questa funzione può essere usata per controllare un object e vedere se è un antenato o meno di un altro object. Quindi, entrambi gli argomenti devono essere gli indici di object e non gli id di istanza, e il primo è sempre il object che si vuole scoprire se è un figlio del secondo argomento, che è sempre il object che si vuole controllare come antenato (genitore).

Sintassi:

object_is_ancestor(obj, par);

Argomento Tipo Descrizione
obj Oggetto Attività The object that is being checked as the child.
par Oggetto Attività The object that is being checked as the ancestor (parent).

Ritorna:

Boolean

Esempio:

if object_is_ancestor(object_index, obj_Enemy)
{
    instance_destroy();
}

Il codice di cui sopra controlla se l'istanza che esegue il codice è un figlio di object "obj_Enemy", e se è così viene distrutto.
