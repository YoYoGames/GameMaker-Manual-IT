ereditato dall'evento

Questa funzione chiamerà l'evento corrente del genitore object dell'istanza. Normalmente, quando un'istanza ha un genitore object, automaticamente inherits tutti gli stessi eventi del genitore, ma se (per esempio) il vostro genitore object ha un evento di creazione e ne aggiungete uno al vostro figlio object, tutte le istanze del figlio object eseguiranno il nuovo evento di creazione che avete aggiunto e non quello che è nel genitore object. Se hai bisogno di usare sia l'evento genitore object che l'evento figlio object dello stesso tipo, dovresti usare questa funzione perché eseguirà l'evento genitore object prima di continuare con il resto del codice o delle azioni che l'evento figlio contiene.

Sintassi:

event_inherited();

Ritorna:

Esempio:

event_inherited();
switch (sprite_index)
{
    case spr_Enemy_1: dmg += 2; break;
    case spr_Enemy_4: dmg -= 1; break;
    case spr_Enemy_10: dmg +=10; break;
}

Il codice di cui sopra chiama l'evento genitore ereditato (in cui inizializziamo la variabile "dmg") e poi va a modificare la variabile "dmg". Se non c'è un genitore specificato per l'istanza che esegue questo codice, otterremmo un errore "variabile sconosciuta" poiché dmg non è stato definito.
