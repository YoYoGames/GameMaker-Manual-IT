Oggetti

In GameMaker avete objects e avete le istanze. Objects sono essenzialmente il modello di base per un'istanza, e come tali non sono mai presenti in un room direttamente... solo le istanze del object sono inserite nel room. Questo significa che se volete cambiare qualcosa per tutte le istanze che state per creare, allora potete farlo cambiando l'oggetto.

IMPORTANTE! Cambiare qualcosa su un object non cambierà le istanze attualmente presenti in room, solo quelle che vengono create dopo la modifica.

Si può anche accedere alle informazioni su una base object e usarle per dettare comportamenti o cambiamenti nel gioco. GameMaker vi dà una serie di funzioni che vi permettono di ottenere vari dettagli su un object. Notate, un object non è un'istanza (un'entità di gioco), è puramente una risorsa da cui tutte le istanze sono create. Questo significa che i valori di ritorno di queste funzioni possono essere diversi dai valori reali mostrati dalle istanze in room, poiché una volta che un'istanza è creata può essere modificata attraverso il codice e le azioni GML Visual.

  • object_index
  • object_exists
  • object_get_name
  • object_get_mask
  • object_get_parent
  • object_get_persistent
  • object_get_solid
  • object_get_sprite
  • object_get_visible
  • object_get_physics
  • object_is_ancestor

Ci sono anche un certo numero di funzioni che permettono di impostare le proprietà di un object. Va notato che qualsiasi istanza di questo object che già esiste in room può non essere interessata da queste funzioni, ma tutte le nuove istanze di questo oggetto creato in room lo saranno, quindi si raccomanda di non cambiare mai le proprietà di un objects quando istanze di quel object sono presenti nel corrente room.

  • object_set_mask
  • object_set_persistent
  • object_set_solid
  • object_set_sprite
  • object_set_visible
