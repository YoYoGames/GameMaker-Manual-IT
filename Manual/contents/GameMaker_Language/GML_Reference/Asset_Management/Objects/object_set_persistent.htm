oggetto_set_persistente

Con questa funzione potete impostare la persistenza di uno specifico object. Un object persistente è "riportato" da room a room e mantiene le sue proprietà in ogni momento (per maggiori informazioni sulla persistenza vedere la sezione sull'Object Editor). Impostare la persistenza di object significa che tutte le istanze di questo object che sono create dopo che la persistenza è stata cambiata saranno create persistenti, mentre le istanze che sono già in room potrebbero non essere interessate.

Quando si crea un'istanza con persistenza abilitata in un room, le viene assegnato un livello o una profondità (a seconda di quale funzione si è usata per creare l'istanza del object). Quando room viene cambiato, e se il seguente room non ha un livello con lo stesso nome o profondità di quello assegnato, allora verrà creato un nuovo livello per l'istanza che sta persistendo attraverso rooms. Se hai dato un nome al livello quando l'istanza è stata creata, allora anche il nuovo livello avrà lo stesso nome del livello originale, mentre se hai assegnato una profondità all'istanza allora il nuovo livello sarà chiamato "_layer_XXX", dove "XXX" è un valore hex usato per dare un nome unico al livello. Questi livelli saranno rimossi da room su Room End. Un'altra cosa da notare sui livelli e le istanze persistenti è che se hai assegnato all'istanza un livello con un nome alla creazione, e c'è un altro livello nel seguente rooms con lo stesso nome, allora l'istanza persistente sarà assegnata al livello con lo stesso nome indipendentemente dalla profondità del livello. Infine, se un'istanza persistente si sposta su un room con un livello alla stessa profondità in cui l'istanza è stata creata, non verrà assegnata a questo livello, ma verrà invece creato un nuovo livello alla stessa profondità (seguendo la convenzione di denominazione spiegata in precedenza).

Si prega di notare che questa non è una funzione di istanza! Potete impostare il flag persistente delle singole istanze usando la variabile persistent e quindi avere dieci istanze tutte contrassegnate come persistenti anche se il object in questione ha il flag impostato a false nel IDE, e anche se cambiate il flag persistente del object a true usando questa funzione, tutte le istanze che attualmente nel room rimarranno come erano, e solo le istanze create dopo aver chiamato la funzione inizieranno con persistent contrassegnato come true.

Sintassi:

object_set_persistent(index, pers);

Argomento Tipo Descrizione
index Oggetto Attività The index of the object to change.
pers Booleano Whether the object should be flagged as persistent (true) o no (false).

Ritorna:

N/A

Esempio:

if (room == rm_final)
{
    object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

Il codice qui sopra controlla per vedere in quale room l'istanza si trova attualmente e se è nello specificato "rm_final" allora imposta la persistenza di object "obj_Player" a false e poi crea l'istanza.
