font_add_sprite_ext

Con questa funzione potete usare una "striscia di sprite" (il sprite stesso deve essere un sprite asset dal Browser Asset, o un sprite che avete aggiunto alle risorse di gioco usando sprite_add) per creare un nuovo font asset, dove ogni sotto-immagine sarebbe un singolo simbolo o lettera. A differenza del normale font_add_sprite() che ha un ordine specifico per le sotto-immagini di sprite, questa funzione mapperà le sotto-immagini di sprite in base all'argomento "string_map" della funzione. Questo argomento è un string che potete usare per dire a GameMaker in che ordine sono le sotto-immagini di sprite font e le mapperà automaticamente quando si scrive il testo. Così, per esempio, se avete una string-map di "AaBbCcDdEeFfGgHh", il vostro sprite font deve avere le sotto-immagini ordinate in questo modo. Puoi definire "spazio" come qualsiasi carattere tu voglia, per esempio una singola linea della dimensione che vuoi che lo spazio sia, e tutti gli spazi nel testo saranno resi a quella larghezza (ma l'immagine scelta non sarà mai resa), o se non fornisci un sprite per lo spazio allora sarà usata invece la larghezza del carattere più largo in sprite font.

Puoi anche impostare la spaziatura per il font per essere proporzionale (true) o no (false), dove un font proporzionale è spaziato in base alla larghezza effettiva delle lettere (così la lettera "i" prende meno room della lettera "w", per esempio) mentre un font non proporzionale sarà spaziato in base alla larghezza della sub-immagine, come un font monospaziato. Infine, si può definire lo spazio da lasciare tra ogni lettera quando si scrive, e questo può essere qualsiasi valore intero, con 0 che non è uno spazio (le lettere si "toccheranno" se proporzionale).

La funzione restituisce un valore di indice che dovrebbe essere memorizzato in una variabile, poiché questo sarà necessario in tutti gli altri codici che fanno riferimento a questo font, oppure restituisce -1 se la funzione fallisce per qualsiasi motivo.

Vale la pena notare che per far funzionare le funzioni di allineamento di font (come draw_set_halign()) allora font sprite dovrebbe avere l'origine impostata sull'angolo in alto a sinistra. Se usate altri valori, allora dovrete prendere in considerazione l'offset dell'origine quando disegnate il testo usando font.

NOTA Quando caricate un font in GameMaker dovete ricordarvi di rimuoverlo (con font_delete()) quando non è più necessario, altrimenti c'è il rischio di una perdita di memoria che rallenterà ed eventualmente bloccherà il gioco.

Sintassi:

font_add_sprite_ext(spr, string_map, prop, sep);

Argomento Tipo Descrizione
spr Sprite Asset The sprite to add a font based on.
string_map Stringa String from which sprite sub-image order is taken.
prop Booleano Set as proportional font or not.
sep Real The space to leave between each letter.

Ritorna:

Font Asset

Esempio:

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

Il codice di cui sopra creerà un nuovo font asset dal sprite indicizzato nella variabile "spr_CalcFont" e memorizzerà l'indice del nuovo font nella variabile "global.Font". Le sotto-immagini del sprite font saranno mappate sul string specificato.
