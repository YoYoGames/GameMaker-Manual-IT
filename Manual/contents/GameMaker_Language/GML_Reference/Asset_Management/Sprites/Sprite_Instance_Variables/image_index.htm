indice_immagine

Un sprite è composto da una o più sotto-immagini che possono far apparire il sprite animato come switch da una all'altra, o possono essere scambiate nel codice per dare diversi "stati", proprio come un pulsante in Windows. Se il sprite è animato, allora potete ottenere il fotogramma corrente dell'animazione controllando la variabile image_index, o se volete cambiare lo stato di un sprite statico, potete selezionare una nuova sotto-immagine impostando questa variabile alla sotto-immagine desiderata del sprite. Notate che per i cambiamenti in questa variabile per essere visibili, l'istanza dovrebbe avere o nessun evento di disegno (e quindi GameMaker disegnerà di default il sprite) o essere disegnata usando una delle funzioni di disegno come draw_self() o draw_sprite_ext() (fornendo l'image_index nell'argomento appropriato).

Si noti che l'impostazione di questa variabile direttamente su un fotogramma non attiverà alcun messaggio di trasmissione che può essere presente su quel fotogramma.

Notate anche che mentre usate l'animazione scheletrica sprites, potete ancora ottenere e impostare i valori di image_index - vedete la funzione skeleton_animation_get_duration() per esempi di come farlo.

Sintassi:

image_index;

Ritorna:

(single precision floating point value)

Esempio:

if (image_speed > 0)
{
    if (image_index >= image_number - 1)
    {
        instance_destroy();
    }
}

Il codice di cui sopra controlla per vedere se sprite si sta animando, e se lo fa allora controlla per vedere se l'image_index corrente è all'ultimo frame e in quel caso distrugge l'istanza.
