miscela_immagine

Questa variabile controlla la "tinta" dell'istanza sprite e il valore predefinito è -1 (ma può anche essere c_white). Qualsiasi altro valore (incluse le costanti di colore interne come c_red, o c_aqua) mescolerà il colore specificato con quello originale sprite. Si prega di notare che per i cambiamenti in questa variabile per essere visibili, l'istanza dovrebbe avere o nessun evento di disegno (e quindi GameMaker disegnerà di default il sprite) o essere disegnata utilizzando una delle funzioni di disegno esteso come draw_self() o draw_sprite_ext().

Si prega di notare che si dovrebbe cercare di limitare il blending sulla piattaforma HTML5 (a meno che non si usi WebGL), poiché ogni sprite mescolato deve essere memorizzato nella cache separatamente e quindi avere molti sprites mescolati può influire negativamente sulle prestazioni (si può anche impostare la dimensione di cache usando la funzione sprite_set_cache_size()).

Sintassi:

image_blend;

Ritorna:

Colour Constant

Esempio:

image_blend = make_colour_hsv(255, 255, random(255));

Il codice qui sopra imposterà l'image_blend di sprite definito per l'istanza ad un colore casuale.
