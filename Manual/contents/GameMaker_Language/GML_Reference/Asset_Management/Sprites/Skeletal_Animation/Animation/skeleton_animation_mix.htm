scheletro_animazione_mix

È possibile switch set di animazioni facilmente usando la funzione skeleton_animation_set() ma questo può causare un salto o uno stutter quando un'animazione viene scambiata con un'altra. Per evitare questo, è possibile impostare il valore di mix tra due set di animazione e sprite interpolerà tra di loro. Normalmente si vorrebbe fare questo nell'evento Create dell'istanza con l'animazione dello scheletro in quanto deve essere impostato solo una volta, e GameMaker interpolerà tutte le ulteriori modifiche a sprite usando i set di animazione in quell'istanza. Notate che il valore di durata va da 0 a 1, dove un valore di 0.5 avrebbe un'interpolazione "metà e metà" da un set all'altro.

NOTA Spine 4.0 cambia come funziona l'animazione rotazionale, poiché non interpola più la rotazione usando il percorso più breve possibile, ma usa i valori assoluti nel tuo keyframes per l'interpolazione. Questo può causare un comportamento inaspettato se stai aggiornando la tua animazione Spine da una versione precedente, e potrebbero essere necessari dei cambiamenti all'interno del tuo progetto Spine per far sì che le rotazioni si comportino come richiesto.

Sintassi:

skeleton_animation_mix(animfrom, animto, duration);

Argomento Tipo Descrizione
animfrom Stringa The name (a string) of the first animation set to interpolate from.
animto Stringa The name (a string) of the second animation set to interpolate to.
duration Real The duration of the interpolation (from 0 to 1)

Ritorna:

N/A

Esempio:

skeleton_animation_set("walk");
skeleton_animation_mix("walk", "jump", 0.2);
skeleton_animation_mix("jump", "walk", 0.4);

Il codice di cui sopra andrebbe nell'evento Create di un'istanza con un'animazione scheletrica sprite e imposta la durata del mix di animazione per interpolare tra le due animazioni impostate "walk" e "jump".
