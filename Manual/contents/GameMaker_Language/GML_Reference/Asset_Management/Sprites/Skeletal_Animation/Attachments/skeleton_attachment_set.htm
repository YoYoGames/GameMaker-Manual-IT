set di attacchi per scheletro

Un'animazione scheletrica sprite può avere altre immagini aggiunte come allegati, con queste immagini che vengono aggiunte a uno slot nominato (il nome è dato quando si crea lo slot dell'allegato nel programma di animazione) e saranno disegnate insieme all'animazione del corrente sprite. Con questa funzione è possibile impostare un allegato a un dato slot, dove si richiede di dare i nomi (come strings) dello slot e dell'allegato. Questi nomi sono definiti dal programma di animazione utilizzato o (nel caso dell'allegato) quando si chiama skeleton_attachment_create().

Notate che potete anche passare un sprite_index come allegato, e quel sprite sarà usato, oppure potete usare -1 per rimuovere l'allegato dallo slot. Quando si passa un indice sprite come argomento, si creerà uno slot di allegato usando il nome del sprite come string per nominare lo slot (così usando spr_sword, per esempio, si creerà uno slot di allegato "spr_sword"), e lo slot userà il primo indice di immagine (0) del sprite, così come il suo offset di origine x e y, con una scala di (1,1) e una rotazione di 0.

Sintassi:

skeleton_attachment_set(slot, attachment);

Argomento Tipo Descrizione
slot Stringa The slot name (a string) to get the attachment of.
attachment Stringa o Sprite Asset The name (as a string or a sprite_index) of the attachment image.

Ritorna:

String

Esempio:

if skeleton_attachment_get("slot_leftHand") == ""
{
    skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife"));
}

Il codice di cui sopra controllerebbe il nome dell'allegato attualmente assegnato per lo slot chiamato "slot_leftHand" e se viene restituito un string vuoto, viene allegato un nuovo sprite.
