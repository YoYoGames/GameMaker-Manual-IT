creare un attacco scheletrico

Quando si crea l'animazione scheletrica sprite, è possibile assegnare slot di attacco e allegati da inserire in essi. Queste sono semplicemente immagini che sono a parte dell'animazione, ma quando sono attaccate si muovono insieme all'osso a cui sono attaccate. Normalmente assegnereste questi allegati nel vostro programma di animazione (Spine), ma con questa funzione potete creare i vostri in fase di esecuzione usando un sprite asset dal vostro gioco. La funzione richiede che diate un nome all'allegato (come string) e che impostiate lo sprite_index e l'image_index da usare, così come l'origine x e y (che può essere diversa da quella definita da sprite nelle proprietà di sprite ), e potete anche impostare qualsiasi trasformazione che volete sia applicata all'immagine quando è attaccata. Se l'allegato è stato creato con successo la funzione restituirà 1 e se non lo è stato (avete fornito un indice sprite non valido, o la base sprite non è un Spine sprite) allora restituirà -1.

Sintassi:

skeleton_attachment_create(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot);

Argomento Tipo Descrizione
name Stringa The name (as a string) of the attachment to create.
sprite Sprite Asset The sprite_index to get the attachment image from.
ind Real The image_index to get the attachment image from.
xorigin Real The x origin for the image being used.
yorigin Real The y origin for the image being used.
xscale Real The horizontal scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
yscale Real The vertical scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
rot Real The rotation of the image. 0=normal, 90=turned 90 degrees counter-clockwise etc.

Ritorna:

Real (1 if successful, -1 if not)

Esempio:

skeleton_attachment_create("sword", spr_Weapons, 0, 0, 80, 1, 1, 90);
skeleton_attachment_create("knife", spr_Weapons, 1, 0, 45, 1, 1, 90);
skeleton_attachment_create("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

Il codice di cui sopra controllerebbe il nome dell'allegato attualmente assegnato per lo slot chiamato "slot_leftHand" e se viene restituito un string vuoto, viene allegato un nuovo sprite.
