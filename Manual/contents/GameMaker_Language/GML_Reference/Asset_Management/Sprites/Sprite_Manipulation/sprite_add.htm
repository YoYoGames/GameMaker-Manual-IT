sprite_add

Con questa funzione è possibile aggiungere un'immagine come sprite, caricandola da una fonte esterna dove il file immagine da caricare dovrebbe essere sempre in formato *.png, *.gif, *.jpg/jpeg o *.json ( i file*json sono usati per caricare sprites di animazione scheletrica fatti con Spine). La funzione restituisce il nuovo indice sprite che deve poi essere utilizzato in tutto il codice successivo che si riferisce al sprite. Se si utilizza questa funzione con HTML5 o si ottiene un'immagine da un URL, questa funzione genererà anche un evento asincrono Image Loaded. Vedere questa pagina per maggiori informazioni.

Tutte le immagini che devono essere trasformate in sprites animate - eccetto *.gif (vedi sotto) - devono avere un formato "striscia" (vedi l'immagine qui sotto) e saranno divise nel numero di sotto-immagini specificato seguendo la regola sprite width = strip width / sub images.

Come potete vedere nell'immagine qui sopra, il sprite è stato posizionato su uno sfondo viola scuro, e questo può essere rimosso impostando l'argomento "removeback" su true. Questo funziona controllando il pixel in basso a sinistra del sprite per il colore lì e poi usa quello come colore da rimuovere. Per esempio, nell'immagine sopra, se avessimo avuto il colore del pixel in basso a sinistra come verde, tutte le parti verdi di sprite sarebbero state rimosse e il resto dello sfondo viola ignorato.

Se scegliete l'opzione "removeback", potreste anche volere che GameMaker smussi i bordi di sprite impostando l'argomento "smooth" a true. Tutto ciò che fa è creare un bordo semi-trasparente intorno ai bordi di sprite dopo che il suo sfondo è stato rimosso.

Infine potete anche specificare l'origine x e y per sprite. Questo è il punto in cui sprite è "fissato" sull'istanza che lo usa, ed è sempre calcolato come relativo all'angolo 0,0 in alto a sinistra di una sottoimmagine sprite. Così, per esempio, un sprite che è 32 x 32 pixel con questi valori impostati a (16,16) avrà la sua origine al centro.

Se stai aggiungendo un'immagine in formato *.gif, allora hai solo bisogno di specificare il nome del file e il numero dell'immagine (che dovrebbe essere 1 per il primo fotogramma) e puoi lasciare tutti gli altri argomenti come 0, notando che solo la prima immagine della GIF sarà caricata. Per *.json sprites, il numero dell'immagine dovrebbe essere 1, e le impostazioni "removeback" e "smooth" saranno ignorate.

Per impostazione predefinita tutti i nuovi sprites hanno i loro riquadri di delimitazione calcolati automaticamente (l'esatto riquadro di delimitazione dipenderà dalla dimensione e dalla trasparenza del sprite), tuttavia si potrebbe desiderare di personalizzarlo, nel qual caso si dovrebbe usare anche la funzione sprite_collision_mask().

Va notato che la memoria usata quando si aggiunge un sprite in questo modo sarà più grande di quanto ci si possa aspettare. Questo perché GameMaker memorizzerà il sprite come una pagina texture e sarà anche memorizzato nella memoria della GPU, quindi la memoria totale sarà più grande di quella che ci si aspetterebbe per un file immagine dello stesso sprite.

NOTA: Quando caricate un sprite in GameMaker dovete ricordarvi di rimuoverlo (con sprite_delete()) quando non è più necessario, altrimenti c'è il rischio di una perdita di memoria che rallenterà ed eventualmente bloccherà il vostro gioco.

NOTA: Su iOS, se state caricando un file incluso da una cartella, cioè: con un path qualcosa come "Backgrounds/background1.png", non è necessario includere la cartella come parte di path e fornire semplicemente il nome del file è sufficiente. Su tutte le altre piattaforme è necessario fornire la cartella (o le cartelle) come parte di path al file.

NOTA: su HTML5, se state caricando un sprite da un sito sicuro server potreste aver bisogno di impostare il tipo cross-origin e usare un path relativo invece di un path assoluto. Vedere http_set_request_crossorigin() per maggiori dettagli.

Sintassi:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

Argomento Tipo Descrizione
fname Stringa The name (a string file path) of the file to add.
imgnum Real Use to indicate the number of sub-images (1 for a single image or for a *.gif).
removeback Booleano Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Booleano Indicates whether to smooth the edges if transparent.
xorig Real Indicate the x position of the origin in the sprite.
yorig Real Indicate the y position of the origin in the sprite.

Ritorna:

Sprite Asset

Esempio:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

Il codice qui sopra carica un sprite nel gioco e memorizza il suo indice nella variabile "spr".
