maschera di collisione sprite

Con questa funzione è possibile impostare le proprietà della maschera di collisione che un sprite dovrebbe avere. Se si seleziona o automatico (0) o immagine completa (1) come modalità del rettangolo di selezione, allora i valori del rettangolo di selezione individuale possono essere impostati a 0. Tuttavia per una maschera definita dall'utente (2) si dovranno impostare questi valori. I diversi valori del rettangolo di selezione sono sempre relativi all'angolo in alto a sinistra di sprite (indipendentemente dall'origine x e y) che è considerata la posizione (0, 0).

L'impostazione del tipo di maschera imposta la forma generale della maschera stessa, ma si noti che qualsiasi cosa diversa da una maschera rettangolare richiederà più potenza di elaborazione quando si risolvono le collisioni, con un conseguente calo delle prestazioni. In generale, dovreste usare tipi di maschera diversi da quella rettangolare solo se assolutamente necessario.

NOTA: Questa funzione non permette il tipo di maschera di collisione a rettangolo ruotato.

Il tipo di maschera che può essere impostato sarà una delle seguenti costanti:

Tipo di scatola di delimitazione (forma) Costante
Costante Descrizione
bboxkind_rectangular Una forma di maschera di collisione rettangolare (non rotante)
bboxkind_ellipse Una maschera di collisione di forma ellittica
bboxkind_diamond Una maschera di collisione a forma di diamante
bboxkind_precise Una maschera di collisione precisa, dove la maschera si conformerà ai pixel non trasparenti di sprite, in base al valore di tolleranza dato (vedi sotto))
bboxkind_spine Apart from rectangle, this is the only valid option for Spine sprites. It enables more precise collision checking against the mesh in the sprite.


Infine, la tolleranza è usata per definire quanto è precisa la maschera (quando viene usata con una maschera "full image", questo non avrà effetto), con una tolleranza di 0 significa che la maschera seguirà ogni singolo pixel che ha una trasparenza superiore a 0, mentre altri valori sposteranno il perimetro della maschera di collisione a seconda della trasparenza dei pixel.

NOTA: Questa funzione è utile solo per bitmap sprites e non funziona con SWF o JSON (Spine) sprites.

NOTA: Questa funzione funziona solo su sprites aggiunto o duplicato sprites e non direttamente su risorse pre-fatte. Puoi duplicare sprites usando la funzione sprite_duplicate().

Sintassi:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Argomento Tipo Descrizione
ind Sprite Asset The index of the sprite to set the bounding box of.
sepmasks Booleano Whether to create collision masks for each sub-image of the sprite (true), o una maschera per tutti (false).
bboxmode Real What kind of bounding box to use. 0 = automatic, 1 = full image, 2 = user defined.
bbleft Real The maximum left position of the bounding box.
bbtop Real The maximum top position of the bounding box.
bbright Real The maximum right position of the bounding box.
bbbottom Real The maximum bottom position of the bounding box.
kind Tipo di scatola di delimitazione (forma) Costante The kind of mask, a constant (see the table in the description).
tolerance Real Indicates the tolerance in the transparency value (0=no tolerance, 255=full tolerance).

Restituisce

N/A

Esempio:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

Il codice qui sopra carica un sprite da una fonte esterna e memorizza il nuovo indice nella variabile "spr". Il codice poi imposta il nuovo sprite per avere una precisa maschera di collisione per ciascuna delle sue sotto-immagini.
