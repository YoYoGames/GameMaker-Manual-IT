sprite_prefetch_multi

Questa funzione può essere usata per prefetchare (mettere in texture memoria) un certo numero di pagine texture che contengono il sprites dato. Si fornisce un array popolato con gli indici sprite (come definito quando si è creato il sprite asset) e le pagine texture su cui si trovano saranno caricate in memoria. Si noti che la funzione restituirà -1 se il prefetch non è supportato per la risorsa scelta o la piattaforma di destinazione è HTML5, oppure restituirà 0 se tutto funziona correttamente.

NOTA: C'è un impatto sulle prestazioni in quanto texture viene caricato nella memoria texture sulla maggior parte dei dispositivi, quindi si raccomanda di chiamare sprite_prefetch_multi() su qualsiasi grafica richiesta all'inizio di un livello per evitare qualsiasi stallo.

Sintassi:

sprite_prefetch_multi

Argomento Tipo Descrizione
array Matrice di risorse di sprite Array with the sprite indices to fetch

Ritorna:

Real (-1 or 0)

Esempio:

spr_a[0] = spr_Player_Aura1;
spr_a[1] = spr_Player_Aura2;
spr_a[2] = spr_Player_Aura3;
spr_a[3] = spr_Player_Aura4;
sprite_prefetch_multi(spr_a);

Il codice di cui sopra crea un array dove ogni elemento contiene un indice sprite. Questo array viene poi usato per mettere questi sprite textures in memoria.
