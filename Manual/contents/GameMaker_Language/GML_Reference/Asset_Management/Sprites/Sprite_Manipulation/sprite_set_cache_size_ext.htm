sprite_set_cache_size_ext

Quando si esegue la fusione delle immagini, HTML5 non può farlo dinamicamente nello stesso modo in cui potrebbe essere eseguito un eseguibile. Pertanto GameMaker deve salvare temporaneamente una copia miscelata dell'immagine e caricarla. Questa funzione imposta quante copie miscelate del dato sprite possono essere messe in cache prima che le vecchie vengano sovrascritte. Il valore predefinito è 4. Questo viene applicato a una singola sotto-immagine data di sprite.

Sintassi:

sprite_set_cache_size_ext(ind, index, max);

Argomento Tipo Descrizione
ind Sprite Asset The index of the sprite to change the cache size of.
index Real The sub-image of ind to change the cache size of.
max Real The maximum number of cached copies of the sprite that can be stored.

Restituisce

N/A

Esempio:

sprite_set_cache_size_ext(sprite0, 0, 2);

Questo imposterà il sprite cache della prima sotto-immagine dello sprite0 a 2 copie.
