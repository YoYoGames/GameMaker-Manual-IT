sprite_set_cache_size

Quando si esegue la fusione delle immagini, HTML5 non può farlo dinamicamente nello stesso modo in cui potrebbe essere eseguito un eseguibile. Pertanto GameMaker deve salvare temporaneamente una copia miscelata dell'immagine e caricarla. Questa funzione imposta quante copie miscelate del dato sprite possono essere memorizzate nella cache prima che le vecchie vengano sovrascritte. Il valore predefinito è 4. Questo viene applicato a tutte le sotto-immagini di sprite.

Sintassi:

sprite_set_cache_size(ind, max);

ArgomentoTipo Descrizione
ind The index of the sprite to change the cache size of.
max The maximum number of cached copies of the sprite that can be stored.

Restituisce

Esempio:

sprite_set_cache_size(sprite0, 2);

Questo imposterà il sprite cache di tutte le sotto-immagini dello sprite0 a 2 copie.
