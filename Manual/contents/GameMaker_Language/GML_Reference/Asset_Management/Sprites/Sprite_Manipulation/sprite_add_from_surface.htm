sprite_add_from_surface

Questa funzione funziona esattamente allo stesso modo di sprite_create_from_surface() solo che invece di creare un nuovo sprite dall'area della superficie indicizzata che si seleziona, aggiunge l'area definita della superficie come una nuova sotto-immagine a un sprite precedentemente creato (questo significa che non si può aggiungere direttamente a un sprite dall'albero delle risorse, ma solo a uno precedentemente creato da una superficie, o uno che è stato duplicato da una risorsa sprite usando sprite_duplicate()).

Va notato che la memoria usata quando si aggiunge un sprite in questo modo sarà più grande di quanto ci si possa aspettare. Questo perché GameMaker memorizzerà il sprite come una pagina texture e sarà anche memorizzato nella memoria della GPU, quindi la memoria totale sarà più grande di quella che ci si aspetterebbe per un file immagine dello stesso sprite.

Sintassi:

sprite_add_from_surface(index, surface, x, y, w, h, removeback, smooth);

Argomento Tipo Descrizione
index Sprite Asset The index of the sprite to add the new image to.
surface ID superficie The index of the surface from which we get the image.
x Real The x position to copy from.
y Real The y position to copy from.
w Real The width of the area to be copied (from the x position).
h Real The height of the area to be copied (from the y position).
removeback Booleano Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Booleano Indicates whether to smooth the edges.

Ritorna:

N/A

Esempio:

spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
var i;
for (i = 1; i < 8; i += 1)
{
    sprite_add_from_surface(spr_custom, surf, i, 0, 32, 32, false, false);
}

Il codice qui sopra crea un sprite dalla superficie indicizzata nella variabile "surf", assegnando il suo indice alla variabile "spr_Custom", e poi usa un for loop per muoversi attraverso la superficie e catturare varie sezioni che vengono aggiunte nel sprite come sotto-immagini.
