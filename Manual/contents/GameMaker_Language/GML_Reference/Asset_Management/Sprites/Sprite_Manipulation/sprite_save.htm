sprite_save

Questa funzione può essere usata per salvare qualsiasi sotto-immagine di un sprite su disco, dandogli il nome di file specificato. Il sprite deve essere stato aggiunto a runtime (non si può salvare sprites aggiunto attraverso il IDE) e il file deve essere salvato con un *.png extension.

NOTA: A seconda della piattaforma di destinazione scelta, siete limitati su dove potete salvare e caricare i file. Vedi Gestione dei file per maggiori informazioni.

Sintassi:

sprite_save(ind, subimg, fname);

Argomento Tipo Descrizione
ind Sprite Asset The index of the sprite to save.
subimg Real The sub-image of the sprite to save.
fname Stringa The filename for the saved sprite.

Restituisce

N/A

Esempio:

var surf, spr_custom;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);
sprite_save(spr_custom, 0, "Player_Custom_Sprite.png");
sprite_delete(spr_Custom);

Il codice qui sopra crea una superficie e memorizza il suo indice nella variabile locale "surf". Poi punta a quella superficie, la cancella e disegna diversi sprites l'uno sull'altro. Crea un nuovo sprite dall'immagine composita disegnata sulla superficie e assegna il suo indice alla variabile locale "spr_Custom" prima di liberare la memoria utilizzata dalla superficie. Infine, il sprite risultante viene salvato in un file e poi rimosso anche dalla memoria.
