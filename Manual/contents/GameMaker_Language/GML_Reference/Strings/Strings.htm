Corde

Prima o poi, durante la realizzazione di un gioco, sarà necessario utilizzare del testo. Il testo nei giochi viene trattato utilizzando le funzioni di stringa (un string è solo un altro modo di dire una riga di testo) e GameMaker ha una serie completa di funzioni che permettono di manipolare strings in molti modi, tra cui l'inserimento di un string in un altro, la copia di strings e la possibilità di analizzare strings per le cifre o le lettere che contengono. In generale, un string testo può essere creato solo aggiungendolo all'interno di doppi apici " " e non sono accettati strings apici singoli, né è possibile dividere il string testo su più righe e aspettarsi che GameMaker lo visualizzi come se le interruzioni di riga fossero dei newline (a meno che non si utilizzi un identificatore @ letterale di stringa, come spiegato di seguito).

Vale la pena di notare che ci sono alcune convenzioni che si possono usare quando si crea strings, soprattutto per quanto riguarda l'uso dei caratteri di escape. Si tratta di caratteri preceduti dal simbolo "\". Quindi, per esempio, se si volessero mettere le virgolette all'interno di un string si otterrebbe qualcosa di simile:

str = "Hello\"World\"";

GameMaker dispone anche di un supporto completo per i caratteri Unicode a quattro byte, che consente di decodificare e codificare i caratteri Unicode nei limiti superiori dello standard (compresi, ma non solo, gli emoji). Per lavorare con i caratteri Unicode, è possibile utilizzare "\" per precedere qualsiasi letterale Unicode - cifre esadecimali precedute da "u", ad esempio "\u00e2" per "á" - dove le cifre sono il numero del carattere Unicode. Quando si lavora con Unicode in questo modo, bisogna essere consapevoli del fatto che GameMaker interpreterà tutte le cifre che seguono il "u", quindi se si volesse scrivere "áa", ad esempio, si dovrebbe usare:

"\u00e2\a"

o

"\u00e2\u61"

o

"\u00e2" + "a"

poiché l'uso di "\u00e2a" darebbe luogo al carattere Unicode "ส" (che diventerebbe essenzialmente "\ue2a").

GameMaker può anche gestire qualsiasi letterale esadecimale, normalmente scritto come cifre esadecimali dopo "0x", ad esempio "0xff", dove le cifre sono il numero del carattere da utilizzare. In GameMaker si scrive "\x" e poi il valore esadecimale. Questi e altri caratteri di escape predefiniti sono elencati nella tabella seguente:

Costante Descrizione
\n Nuova linea
\r Ritorno a carrello
\b Backspace (0x08)
\f Avanzamento modulo (0x0c)
\t Scheda orizzontale (0x09)
\v Scheda verticale (0x0b)
\\ Backslash stesso (0x5c)
\a Allarme (0x07)
\u[Hex Digits] Inserire un carattere Unicode
\x[Hex Digits] Inserire un carattere letterale esadecimale
\[Octal Digits] Inserire un carattere Unicode ottale

NOTA: Strings supporta l'avanzamento modulo, la tabulazione verticale, ecc... ma questo non significa che lo faccia anche il rendering, e quando si disegna strings questi caratteri possono essere ignorati.

È anche possibile creare letterali di stringa precedendo l'intero string con il carattere @:

var test = @"
Line breaks
over multiple
lines
";

Il codice sopra riportato renderà string su più righe come se fosse incluso un carattere di escape di interruzione di riga. Un letterale verbatim string è simile ai precedenti letterali GameMaker versione string, ma anch'essi usano virgolette doppie o singole e devono essere preceduti da un simbolo @, possono essere spezzati su più righe nel file di codice e NON supportano i caratteri di escape, cioè @"Hello\World" non cercherà di sfuggire alla W di World e sarà memorizzato verbatim. Si noti però che quando si usano i letterali string come questo, è necessario spezzare string se si vogliono includere le virgolette come parte di string, cioè:

var test = @"Hello " + "\"" + @"World" + "\""

Un'altra cosa da notare è che il carattere Unicode 9647 (▯) viene usato per sostituire qualsiasi glifo mancante che potrebbe essere presente nel vostro font designato quando lo rendete nell'evento draw. Quindi, se il vostro font non ha, per esempio, il simbolo °, la scrittura di 90° produrrà effettivamente 90▯.
