Ingresso da tastiera

Quando avete a che fare con la tastiera in GameMaker avete una varietà di funzioni che possono essere utilizzate per riconoscere diversi stati della tastiera come premuto o rilasciato. Ci sono anche alcune che memorizzano tutte le pressioni dei tasti come string o che possono dirvi quale sia stato l'ultimo tasto premuto, così come altre che vi permettono di cancellare completamente lo stato della tastiera.

NOTA Queste funzioni sono progettate solo per le piattaforme desktop Windows/Mac/Ubuntu. Potreste trovare alcune delle variabili e costanti incorporate non valide su altre piattaforme e molte delle funzioni non funzioneranno sui cellulari.

Ogni carattere in ingresso da una chiave (o più chiavi) è definito dal suo codice UTF8, che è un valore numerico. Questo valore può essere recuperato per qualsiasi carattere usando la funzione ord() ma, GameMaker ha anche una serie di constants per i tasti speciali della tastiera più usati e una funzione speciale. In genere si usa una combinazione di ord() con le funzioni keyboard_check*(), qualcosa del genere:

if keyboard_check(ord("A"))
{
    hspeed = -5;
}

Quindi, quanto sopra controllerà il tasto "A" e se è stato premuto allora imposterà la velocità orizzontale di object a -5. Notate che usare ord() in questo modo funzionerà correttamente solo se l'input string è lungo un solo carattere ed è un numero da 0 a 9 o un carattere romano maiuscolo da A a Z. La funzione ord() restituirà un valore UTF8 completo, ma le funzioni keyboard_check*() rileveranno solo A - Z e 0 - 9. Ma cosa succede se volete usare i tasti freccia? O se volete modificare un'azione usando il tasto "shift"? Bene, per questo GameMaker ha una serie di costanti vk_ (vk_ sta per chiave virtuale) che potete usare al posto di ord():

Le seguenti costanti possono essere usate solo con keyboard_check_direct():

Costante chiave virtuale (vk_*)
Costante Descrizione
vk_lshift tasto shift sinistro
vk_lcontrol tasto di controllo sinistro
vk_lalt tasto alt sinistro
vk_rshift tasto shift destro
vk_rcontrol tasto di controllo destro
vk_ralt tasto alt destro

Quello che segue è un piccolo esempio di come usare le vk_ costanti:

if keyboard_check_pressed(vk_tab)
{
    instance_create_layer(x, y, "Controllers", obj_Menu);
}

Il codice qui sopra rileverà se il tasto "Tab" è premuto e creerà un'istanza di object "obj_Menu" se lo è.

Se avete bisogno di controllare un carattere chiave che non sia 0 - 9, A - Z o una delle costanti VK, allora dovreste controllare una delle variabili keyboard_*, come keyboard_lastchar per esempio:

var _key = keyboard_lastchar;
if ord(_key) == ord("ç")
{
    show_debug_message("ç key pressed");
}

Le seguenti funzioni sono disponibili per l'uso quando si lavora con una tastiera:

NOTA: Queste funzioni non funzionano quando si usa una tastiera virtuale sullo schermo.

  • io_clear
  • keyboard_check
  • keyboard_check_pressed
  • keyboard_check_released
  • keyboard_check_direct
  • keyboard_clear
  • keyboard_set_map
  • keyboard_get_map
  • keyboard_unset_map
  • keyboard_set_numlock
  • keyboard_get_numlock

Ci sono anche alcune funzioni che possono essere utilizzate per simulare la pressione dei tasti:

  • keyboard_key_press
  • keyboard_key_release

Infine, le seguenti variabili sono integrate in GameMaker e vi si può accedere per ottenere informazioni sullo stato della tastiera e sull'input:

NOTA: Quando si usa la tastiera virtuale, solo la variabile keyboard_string sarà aggiornata con l'input della tastiera.

  • keyboard_key
  • keyboard_lastkey
  • keyboard_lastchar
  • keyboard_string
