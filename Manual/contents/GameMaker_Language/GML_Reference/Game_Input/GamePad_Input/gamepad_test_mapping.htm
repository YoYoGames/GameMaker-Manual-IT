gamepad_test_mapping

Questa funzione può essere usata per impostare la mappatura del gamepad su quegli obiettivi che lo permettono. Si fornisce l'indice "slot" per il gamepad da impostare, e poi la mappa string, che dovrebbe essere stata creata utilizzando il formato SDL con i seguenti campi:

<guid>,<description>,platform:<platform-name>,<bindings>

I dettagli che devono essere inclusi in ognuno di questi campi sono:

  • guid - un string di cifre e lettere che identifica univocamente un tipo di dispositivo (questo può essere recuperato utilizzando la gamepad_get_guid() funzione)
  • description - una descrizione Ascii del dispositivo (questa può essere recuperata utilizzando la gamepad_get_description() funzione)
  • platform-name - sarà "platform:android", "platform:windows", "linux", o "mac" (si noti che questo sarà aggiunto automaticamente a string quando si chiama questa funzione, e quindi non è necessario fornirlo quando si costruisce la propria mappatura strings)
  • bindings - un insieme di voci separate da un ',' per legare l'input attuale a specifiche costanti GML

La lista di voci separate da virgole segue un formato specifico per ogni voce:

<abstract-gp-name>:<value-binding>

Qui <value-binding> (cioè: l'input diretto del gamepad) può essere uno qualsiasi dei seguenti:

  • a<N> - Questo rappresenta un asse del gamepad, dove il <N> è uno degli assi da 0 a gamepad_axis_count() -1
  • b<N> - Questo rappresenta un pulsante del gamepad, dove il <N> è uno dei pulsanti da 0 a gamepad_button_count() -1
  • h<N>.<mask> - Questo rappresenta un cappello del gamepad, dove il <N> rappresenta un cappello da 0 a gamepad_hat_count() -1, e il <mask> sarà 1, 2, 4 o 8.

Ogni valore di input dovrebbe essere legato a un nome SDL, che è espresso come <>abstract-gp-name>, che a sua volta è legato a una costante GML. La tabella seguente mostra l'equivalenza tra il nome SDL e la costante GameMaker:

Quando avete costruito la vostra mappatura string dovrebbe avere un aspetto simile a questo:

"050000005e040000fd020000ffff3f00,Xbox Wireless Controller,a:b0,b:b1,start:b4,dpdown:h0.4,dpleft:h0.8,dpright:h0.2,dpup:h0.1,guide:b6,leftshoulder:b9,leftstick:b7,lefttrigger:a4,leftx:a0,lefty:a1,rightshoulder:b10,rightstick:b8,righttrigger:a5,rightx:a2,righty:a3,x:b2,y:b3,platform:android"

Sintassi:

gamepad_test_mapping(index, mapping_string);

Argomento Tipo Descrizione
index Real Which gamepad index "slot" to set.
mapping_string Stringa The map string to use (see the description for more information).

Ritorna:

N/A

Esempio:

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length(global.PadInstances);
for (i = 0; i < len; i += 2)
{
    var left = global.PadInstances[i];
    var right = global.PadInstances[i+1];
    mapping += "," + left.sdlLabel + ":" + right.binding;
}
gamepad_test_mapping(global.padIndex, mapping);

Il codice di cui sopra si collega a loop attraverso un certo numero di istanze e usa i valori delle diverse variabili che contengono per creare una mappatura string, che viene poi impostata per l'uso sul gamepad nell'indice dello slot dato.
