rete_invia_udp_raw

Con questa funzione potete inviare dati attraverso la rete usando UDP a un server. La funzione prende l'id del socket a cui connettersi, l'URL a cui connettersi e la porta da usare. Dovete poi fornire l'id del buffer che contiene i dati da inviare (per maggiori informazioni su buffers vedere Reference - Buffers) e infine la dimensione (in byte) dei dati. UDP è "connectionless" nel senso che non si fa una connessione, si invia solo un pacchetto direttamente a un IP, e server riceve i dati in arrivo da un indirizzo IP e deve trattarli "così come sono". I dati inviati non sono formattati da GameMaker in alcun modo e i dispositivi riceventi riceveranno i dati come un flusso, il che significa che dovrete gestirli voi stessi. La funzione restituirà il numero di byte di dati inviati, o un numero inferiore a 0 se l'invio è fallito.

NOTA Questa funzione non funzionerà se usata in un progetto in esecuzione sul target HTML5, e nemmeno i progetti HTML5 saranno in grado di ricevere UDP.

Sintassi:

network_send_udp_raw(socket, url, port, bufferid, size);

ArgomentoTipo Descrizione
socket The id of the socket to use.
url The url or IP to connect to (a string).
port The port to connect to.
bufferid The id of the buffer to get the data from.
size The size (in bytes) of the data.

Ritorna:

Real

Esempio:

network_send_udp_raw(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

Il codice qui sopra invierà un pacchetto UDP grezzo a server definito dall'URL sulla porta 6510. I dati sono presi da buffer indicizzati nella variabile "buff".
