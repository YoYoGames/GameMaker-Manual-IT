rete_invio_diffusione

Con questa funzione è possibile trasmettere localmente i dati di un buffer a un intervallo di indirizzi IP (per ulteriori informazioni su buffers vedere Riferimento - Buffer). L'intervallo è limitato a quello del dispositivo che esegue server, per cui se il dispositivo ha un IP di 92.168.11.130, i dati saranno trasmessi nell'intervallo 92.168.11.*. La funzione restituisce il numero di byte di dati inviati o un numero inferiore a 0 se l'invio non è riuscito.

Questa funzione funziona solo se usata con UDP: il vostro server deve essere TCP e il vostro client deve avere un socket client UDP creato con network_create_socket_ext() per poter ricevere qualsiasi trasmissione inviata da server.

Questa funzione non funziona se utilizzata in un progetto che gira sul target HTML5.

Sintassi:

network_send_broadcast(socket, port, bufferid, size);

Argomento Tipo Descrizione
socket ID della presa di rete The id of the socket to use.
port Reale The port that the server will use.
bufferid ID buffer The id of the buffer to get the data from.
size Reale The size (in bytes) of the data.

Restituzione:

Real

Esempio:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.server_name);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

Il codice precedente scrive il nome string dell'attuale server (memorizzato in "global.ServerName"), quindi lo scrive in un buffer binario con l'id "broadcast_buffer". Questi dati vengono quindi trasmessi localmente a un intervallo di IP (l'IP del dispositivo è attualmente implicito come intervallo di base del broadcast) alla porta 6511.
