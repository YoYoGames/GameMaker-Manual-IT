Collegamento in rete

Le funzioni di rete di GameMaker si basano sui socket, che sono uno standard industriale su tutte le piattaforme. Grazie ai socket, è possibile creare sia client che servers utilizzando un semplice GML che consente di creare anche un gioco a giocatore singolo utilizzando il modello client/server (cosa che consentirà di aggiungere facilmente il multiplayer in seguito).

In pratica, un socket è un object che può inviare, ricevere, connettersi e ascoltare le porte della rete. Queste funzioni riguardano solo le connessioni TCP/IP, su cui si basa Internet (IPv4 per essere più precisi). In sostanza, consente di collegare un socket con un indirizzo IP (ad esempio 192.168.1.10) a un altro socket su un altro indirizzo IP (ad esempio 192.168.1.11). Una volta connessi, questi due socket possono inviare dati avanti e indietro usando le funzioni network_send_* (come network_send_packet()) e l'evento asincrono dedicato che viene attivato da esse. Questo evento asincrono genererà una mappa DS contenente i dati ricevuti e altri dettagli; i dettagli completi si trovano nella sezione sull'evento asincrono di rete.

Invece di dover leggere e gestire ogni singolo bit di traffico di rete che entra in una macchina, gli indirizzi IP si occupano anche delle porte, che sono numerate da 0 a 65535. In questo modo, ogni programma riceve solo i pacchetti di dati che gli interessano e non tutto ciò che è stato inviato da ogni programma. In questo modo si risparmia anche un notevole tempo di CPU, poiché si riduce notevolmente l'elaborazione dei dati, quindi con GameMaker è possibile anche "ascoltare" le porte. Quindi, server creerà un socket e lo legherà a una porta specifica da ascoltare. Riceverà quindi una notifica di connessioni, disconnessioni e trasferimenti di dati da quel socket.

I "pacchetti" di dati stessi sono costituiti da dati binari presi da buffer. Questo buffer può essere creato e manipolato utilizzando le funzioni di GameMaker Buffer.

Vale la pena notare che se si sta creando un progetto per il target HTML5, non si potranno usare le funzioni di server create a causa delle restrizioni del browser e si dovrà scrivere il proprio server per questa piattaforma usando qualcosa come Electron o NodeJS. Si noti inoltre che se il gioco è cross-platform e si prevede di connettersi a un'istanza HTML5 del gioco da un server dispositivo mobile o desktop, allora il proprio server può comunicare con l'istanza HTML5 solo utilizzando il protocollo Web Socket (questo viene fatto utilizzando le costanti network_socket_ws o network_socket_wss - vedere network_create_server() per maggiori informazioni). Ciò significa che il vostro server potrebbe dover creare due socket su porte diverse, uno per connettersi agli obiettivi non HTML5 (Android, Windows, ecc...) e uno per connettersi all'obiettivo HTML5.

Le seguenti funzioni possono essere utilizzate per il collegamento in rete:

  • network_create_server
  • rete_creare_server_raw
  • network_create_socket
  • network_create_socket_ext
  • network_connect
  • network_connect_async
  • network_connect_raw
  • rete_connessione_async_raw
  • network_resolve
  • network_set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
  • network_destroy
