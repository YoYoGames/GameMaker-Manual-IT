rete_invio_udp

Con questa funzione è possibile inviare dati attraverso la rete utilizzando UDP a un sito server. La funzione prende l'id del socket attraverso cui connettersi, l'URL a cui connettersi e la porta da utilizzare. Occorre poi fornire l'id del buffer che contiene i dati da inviare (per maggiori informazioni su buffers si veda Riferimento - Buffer) e infine la dimensione (in byte) dei dati. UDP è "senza connessione", in quanto non si effettua una vera e propria connessione, ma si invia un pacchetto direttamente a un IP e server riceve i dati in arrivo da un indirizzo IP e deve gestirli "così come sono". La funzione restituisce il numero di byte di dati inviati, o un numero inferiore a 0 se l'invio è fallito. Va notato che la dimensione finale dei dati inviati restituita da questa funzione includerà anche le informazioni dell'intestazione GameMaker, che sono 12 byte aggiuntivi.

Questa funzione non funzionerà se utilizzata in un progetto in esecuzione sul target HTML5, né i progetti HTML5 saranno in grado di ricevere UDP.

Sintassi:

network_send_udp(socket, url, port, bufferid, size);

Argomento Tipo Descrizione
socket ID della presa di rete The id of the socket to use.
url Stringa The url or IP to connect to (a string).
port Reale The port to connect to.
bufferid ID buffer The id of the buffer to get the data from.
size Reale The size (in bytes) of the data.

Restituzione:

Real

Esempio:

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

Il codice precedente invia un pacchetto UDP all'indirizzo server definito dall'URL sulla porta 6510. I dati vengono prelevati dal sito buffer indicizzati nella variabile "buff".
