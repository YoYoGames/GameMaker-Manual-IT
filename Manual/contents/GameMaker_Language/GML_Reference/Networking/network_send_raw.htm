rete_send_raw

Con questa funzione potete inviare un pacchetto di dati "grezzi" attraverso la rete. La funzione prende l'id del socket a cui connettersi e poi si deve fornire l'id del buffer che contiene i dati da inviare (per maggiori informazioni su buffers vedere Reference - Buffers) e infine la dimensione (in byte) del pacchetto di dati. I dati inviati non sono formattati da GameMaker in alcun modo e i dispositivi riceventi riceveranno i dati come un flusso, il che significa che dovrete gestirli voi stessi. La funzione restituirà il numero di byte di dati inviati, o un numero inferiore a 0 se l'invio è fallito.

Opzioni Argomento

L'ultimo argomento è opzionale ed è usato solo con i WebSockets. Permette di scegliere tra l'invio di dati binari o di testo. Entrambe le costanti possono essere specificate in questo argomento:

Costante del tipo di invio della rete
Costante Descrizione
network_send_binary Inviare un messaggio binario
network_send_text Inviare un messaggio di testo

Le API di alcune piattaforme accettano solo messaggi di testo quando si usano i WebSockets (ad esempio Twitch), quindi la costante network_send_text può essere usata in questi casi. Se questo argomento non è specificato, vengono inviati dati binari per impostazione predefinita.

Sintassi:

network_send_raw(socket, bufferid, size, [options]);

Argomento Tipo Descrizione
socket ID della presa di rete The id of the socket to use.
bufferid ID del buffer The id of the buffer to get the data from.
size Real The size (in bytes) of the data.
options Costante del tipo di invio della rete OPTIONAL Usato per i WebSockets per scegliere tra testo e dati binari; se non specificato, vengono inviati dati binari.

Ritorna:

Real

Esempio:

buff = buffer_load("player_save.dat");
network_send_raw(sock, buff, buffer_get_size(buff));

Le informazioni di cui sopra caricano in memoria un dato precedentemente salvato buffer e restituiscono l'id di buffer da memorizzare nella variabile "buff". Questo buffer completo viene poi inviato come un pacchetto di dati grezzi sulla rete utilizzando il socket identificato dalla variabile "sock".
