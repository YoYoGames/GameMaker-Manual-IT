sorgente_tempo_creata

Questa funzione crea una nuova sorgente di tempo. Per una panoramica sulle sorgenti temporali, leggere Sorgenti temporali.

La creazione di una nuova sorgente di tempo non la fa partire automaticamente; è necessario chiamare time_source_start() per attivarla.

Una nuova origine temporale viene eseguita una sola volta, a meno che non venga specificato l'argomento ripetizioni (si veda sotto).

È necessario distruggere una Sorgente di tempo utilizzando time_source_destroy() quando non ne avete più bisogno.

Argomenti

Ecco la ripartizione degli argomenti della funzione:

Genitore Fonte di tempo

È la sorgente temporale madre che controlla la nuova sorgente temporale. Può essere time_source_global, time_source_game o una sorgente temporale personalizzata già esistente.

Vedere: Sorgenti temporali integrate

Periodo

Si tratta di un periodo di tempo che può essere espresso in secondi o in fotogrammi, a seconda dell'unità specificata nell'argomento successivo.

Quando si utilizzano i frame come unità di misura, il periodo deve essere un numero intero. I valori non interi vengono arrotondati per difetto (floored), con l'eccezione dei valori inferiori a 1, che vengono arrotondati per eccesso.

Unità

È l'unità in cui è espresso il periodo e può essere time_source_units_seconds o time_source_units_frames.

Vedere: Unità di origine del tempo

È possibile utilizzare un valore di battiti al minuto utilizzando time_bpm_to_seconds().

Callback e argomenti

È necessario specificare un "callback" method che viene richiamata quando la sorgente temporale scade.

È possibile specificare facoltativamente un array contenente gli argomenti da passare al metodo quando viene chiamato.

L'array stesso non verrà passato nel metodo, ma ogni elemento dell'array verrà passato come argomento separato.

Ad esempio, se la funzione si aspetta gli argomenti (x, y, object), l'array di argomenti può essere simile a questo: [30, 600, obj_player].

Ripetizioni

È possibile specificare quante volte la sorgente temporale deve ripetersi. Un valore pari a 1 significa che viene eseguito una sola volta, che è il valore predefinito.

È possibile specificare il numero totale di volte in cui deve essere ripetuto, oppure -1 per farlo ripetere all'infinito.

Ad esempio, se si imposta questo valore a 3 e il periodo della sorgente temporale è "4 secondi", la sorgente temporale sarà attiva per un totale di 12 secondi e chiamerà il metodo di callback ogni 4 secondi.

Tipo di scadenza

Può essere time_source_expire_nearest o time_source_expire_after.

Vedere: Tipi di scadenza delle fonti di tempo

Sintassi:

time_source_create(parent, period, units, callback, [args, repetitions, expiry_type]);

Argomento Tipo Descrizione
parent ID della fonte temporale The parent Time Source che controlla la nuova sorgente temporale
period Reale The period that the Time Source per, nelle unità date
units Sorgente di tempo Unità Costante The units that the given period is in
callback Metodo o Funzione Script The method or script function to call when the Time Source Scade
args Array OPTIONAL Un array contenente gli argomenti da passare al metodo
repetitions Reale OPTIONAL Il numero di volte in cui la sorgente temporale deve ripetersi, o -1 per una ripetizione indefinita.
expiry_type Sorgente di tempo Costante di scadenza OPTIONAL Se la sorgente temporale scade nel fotogramma più vicino alla sua scadenza o nel fotogramma successivo.

Restituzione:

Time Source ID

Esempio 1:

var _my_method = function()
{
    instance_destroy();
}

var _time_source = time_source_create(time_source_game, 300, time_source_units_frames, _my_method);

time_source_start(_time_source);

In questo esempio, vogliamo che l'istanza si distrugga 300 fotogrammi dopo.

Il codice crea prima un metodo che chiama semplicemente instance_destroy().

Crea quindi una sorgente di tempo, che eredita dalla sorgente di tempo del gioco. Imposta il suo periodo a 300 fotogrammi.

Infine, avvia la Sorgente tempo.

Esempio 2:

var _my_method = function()
{
    show_debug_message("A second has passed!");
}

global.time_per_second = time_source_create(time_source_game, 1, time_source_units_seconds, _my_method, [], -1, time_source_expire_after);

time_source_start(global.time_per_second);

In questo esempio, stiamo creando una sorgente temporale globale che scade una volta al secondo. Questo codice andrebbe inserito nella radice di un file script.

Il codice crea innanzitutto un metodo che stampa un messaggio nel registro di output, dicendo "È passato un secondo!".

Crea quindi una nuova sorgente di tempo, che eredita dalla sorgente di tempo del gioco. Imposta il suo periodo a 1 secondo.

Il metodo viene passato nella sorgente temporale, in modo che possa essere richiamato ogni volta che scade. Per gli argomenti viene fornito un array vuoto.

Il conteggio delle ripetizioni è impostato a -1, in modo che la sorgente temporale non si fermi mai e continui a ripetersi per sempre. Il tipo di scadenza è impostato in modo che il callback venga eseguito sul primo fotogramma dopo la scadenza.

Infine, viene avviata la Sorgente tempo.
