debug_evento

Questa funzione genera un evento di debug personalizzato che verrà mostrato nella Vista grafica di debugger quando il gioco viene eseguito in modalità Debug. Se si desidera che i messaggi vengano visualizzati quando non si è in modalità di debug, usare show_debug_message().

La funzione accetta anche due strings riservati per aiutare a debugging utilizzare applicazioni esterne come Visual Studio. Questi strings sono:

  • "OutputDebugOn" - abilita una chiamata a OutputDebugString per il target Windows, il che significa che tutto l'output di debug - tutto ciò che si vede nella finestra Output - può essere visualizzato da Visual Studio o da applicazioni di terze parti.
  • "OutputDebugOff" - Disabilita il comportamento descritto sopra.
  • "BreakOnError" - Questa opzione è solo per le build YYC di Windows e significa che i progetti non visualizzeranno la solita schermata di errore del codice se runtime rileva un errore, ma andranno avanti e si bloccheranno. Questo permette di vedere lo stacktrace all'interno di Visual Studio se debugging.
  • "ResourceCounts" - Elenca tutte le risorse attualmente attive, come Strutture dati, Sorgenti di tempo, Superfici, ecc.
  • "DumpMemory" - Fornisce informazioni sull'utilizzo attuale della memoria.

Sintassi:

debug_event(string)

Argomento Tipo Descrizione
string Stringa The custom debug event string to use.

Restituzione:

N/A

Esempio:

if !surface_exists(global.EffectsSurface)
{
    debug_event("Recreating Effects Surface");
    global.EffectsSurface = surface_create(room_width, room_height);
}

Il codice precedente controlla se una superficie esiste e se non esiste, viene attivato un evento di debug nella vista grafica di debugger (il gioco deve essere stato eseguito in modalità Debug per essere visibile) e la superficie viene ricreata.
