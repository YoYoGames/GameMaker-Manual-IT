debug_get_callstack

Questa funzione genera un array di strings come "callstack" dove l'attuale script è elencato per primo, e poi tutti gli altri scripts che sono stati eseguiti in modo che l'attuale script sia eseguito. Il formato esatto di string varia a seconda della piattaforma di destinazione scelta, ma per lo più avrà il nome dello script/evento, poi i due punti : e il numero di linea, simile a questo:

"gml_Script_script2:1"
"gml_Script_script1:32"
"gml_Script_script0:64"
"gml_Object_object0_Create_0:1"

La funzione permette di passare un argomento opzionale, che è la profondità massima del callstack restituito. Questo valore è il numero di scripts che sono inclusi nell'array restituito a partire dal corrente script. Se questo argomento non è specificato, verrà restituito il callstack completo.

Si noti che l'array restituito conterrà sempre 0 nella sua ultima posizione, dopo aver elencato il callstack scripts.

Sintassi:

debug_get_callstack([maxdepth])

Argomento Tipo Descrizione
[maxdepth] Real OPTIONAL La profondità massima del callstack che viene restituita

Ritorna:

Array

Esempio:

if debug_mode
{
    if (keyboard_check(vk_escape))
    {
        var _a = debug_get_callstack(4);
        for (var i = 0; i < array_length_id(_a); ++i;)
        {
            show_debug_message(_a[i]);
        }
    }
}

Il codice qui sopra controlla se la modalità di debug è abilitata e, se lo è, controlla se un tasto viene tenuto premuto. In tal caso, emette lo stack di chiamate corrente sulla console, con una profondità massima di 4.
