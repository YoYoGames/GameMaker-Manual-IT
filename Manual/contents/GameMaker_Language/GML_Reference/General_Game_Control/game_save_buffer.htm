gioco_salva_buffer

Questa è una variante della funzione game_save(), quindi leggete prima la sua pagina perché contiene informazioni importanti relative al suo uso e a quello di questa funzione.

Con questa funzione è possibile salvare lo stato attuale del gioco in un "grow" buffer creato in precedenza (vedi Buffers) che può poi essere caricato di nuovo in qualsiasi momento usando game_load_buffer() la funzione. Questa funzione è progettata per l'uso in un solo room alla volta (cioè: non è consigliabile fare un salvataggio buffer in un room, poi provare a caricare il buffer da un altro) e dovrebbe idealmente essere usato solo per checkpoint o riavvii di livello.

NOTA: Questa funzione è molto limitata ed è progettata per i principianti per ottenere un sistema di checkpoint attivo e funzionante velocemente, ma gli utenti più avanzati potrebbero preferire codificare il proprio sistema usando le funzioni File, a causa del fatto che il gioco non salverà nessuna delle risorse dinamiche come strutture dati, superfici, aggiunte sprites ecc.

Sintassi:

game_save_buffer(buffer);

ArgomentoTipo Descrizione
buffer The buffer id to save to.

Ritorna:

Esempio:

if keyboard_check_pressed(ord("S"))
{
    global.Checkpoint = true;
    game_save_buffer(save_buff);
}

Il codice qui sopra imposterà una variabile globale su true e poi salverà lo stato attuale del gioco su buffer indicizzato nella variabile "save_buff" quando il tasto "S" viene premuto.
