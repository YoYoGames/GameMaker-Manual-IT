giocatore_locale

Quando si usa rollback_define_player() per definire un giocatore object, il sistema crea automaticamente un'istanza di quel object per ogni giocatore. Ogni istanza di object riceve la variabile player_local, che indica se l'istanza del giocatore è locale o meno.

Se non si usa rollback_define_player(), questa variabile non verrà creata per nessun objects.

È possibile utilizzare questa variabile per mostrare quale giocatore è locale. Non è consigliabile utilizzare questa variabile in modo da influenzare la logica di gioco, che deve essere la stessa per i giocatori locali e remoti.

Sintassi:

player_local;

Restituzione:

Boolean

Esempio:

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

Il codice qui sopra viene eseguito nell'evento Draw e controlla se l'istanza del giocatore è locale. In tal caso, disegna una freccia sprite sopra l'istanza, indicando al giocatore quale istanza sta controllando.
