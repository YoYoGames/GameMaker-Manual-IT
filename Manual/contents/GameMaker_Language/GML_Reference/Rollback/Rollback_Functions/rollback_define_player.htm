rollback_define_player

Questa funzione definisce il sito object che deve essere usato per i giocatori in multiplayer. È opzionale e, se utilizzata, il sistema creerà automaticamente istanze per i giocatori che si connettono e distruggerà le istanze per i giocatori che si disconnettono.

Per impostazione predefinita, crea le istanze dei giocatori nel livello "Instances", che deve esistere nel gioco room. Se si desidera utilizzare un altro livello, si deve passare il suo nome come secondo argomento.

Per informazioni sulla creazione di una partita con o senza questa funzione, vedere Definizione di un oggetto giocatore. Se utilizzata, questa funzione deve essere eseguita prima della funzione join/create.

Ogni istanza creata con questa funzione riceve le variabili di istanza elencate in questa pagina.

Sintassi:

rollback_define_player(object, [layer_name]);

Argomento Tipo Descrizione
object Oggetto Attività The object that will be used for creating player instances.
layer_name Stringa OPTIONAL Il nome del livello in cui verranno create le istanze del giocatore. "Istanze" per impostazione predefinita.

Restituzione:

N/A

Esempio:

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

Il codice qui sopra definisce un giocatore object, e poi tenta di unirsi a una partita. Se non viene raggiunto, inizia una nuova partita, con un massimo di 2 giocatori e il test di sincronizzazione disabilitato.
