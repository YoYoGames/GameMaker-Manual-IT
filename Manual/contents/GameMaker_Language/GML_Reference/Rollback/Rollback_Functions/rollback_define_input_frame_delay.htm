rollback_define_input_frame_delay

Questa funzione definisce il ritardo fisso dei fotogrammi per gli ingressi del lettore locale.

Per impostazione predefinita, il gioco utilizzerà un ritardo di ingresso adattivo, che si basa sulla latenza tra i giocatori. Questo funziona per quasi tutti i giochi in circolazione, ma alcuni tipi specifici di giochi (ad esempio i giochi di combattimento) potrebbero preferire l'uso di un ritardo di ingresso fisso, invece di un ritardo adattivo che può cambiare in qualsiasi momento.

Non è consigliabile modificare questa funzione a meno che non si sappia esattamente cosa si sta facendo. È possibile richiamare questa funzione in qualsiasi momento prima e durante una partita multigiocatore.

Il valore del ritardo specificato deve essere in numero di fotogrammi. Se si specifica -1, il gioco tornerà a utilizzare un ritardo adattivo. Il ritardo massimo impostabile è di 10 fotogrammi.

Sintassi:

rollback_define_input_frame_delay(frames);

Argomento Tipo Descrizione
frames Reale The number of frames to set as the fixed input delay, or -1 per ripristinare le impostazioni predefinite.

Restituzione:

N/A

Esempio:

// 'Options' script
global.frame_delay = 5;

// obj_game: Create event
rollback_define_input_frame_delay(global.frame_delay);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

È possibile definire il valore del ritardo dei fotogrammi come variabile globale e usarlo quando si chiama rollback_define_input_frame_delay() prima che venga avviata una partita di rollback.
