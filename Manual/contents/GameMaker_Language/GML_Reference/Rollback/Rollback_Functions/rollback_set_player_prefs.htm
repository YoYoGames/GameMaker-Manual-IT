rollback_set_player_prefs

Dopo aver abilitato le preferenze, questa funzione viene utilizzata per impostare le preferenze del giocatore locale. Ciò significa che la funzione deve basarsi sull'input locale e non su quello di Rollback.

Le preferenze possono essere di qualsiasi tipo. Se sono state impostate preferenze predefinite, assicurarsi di utilizzare lo stesso data type e la stessa struttura.

Quando un giocatore imposta le sue preferenze, tutti i giocatori ricevono un Evento Rollback.

Se utilizzata, questa funzione deve essere eseguita prima della funzione di unione/creazione.

Si prega di leggere: Preferenze del giocatore

Sintassi:

rollback_set_player_prefs(preferences);

Argomento Tipo Descrizione
preferences Qualsiasi The preferences to set for the local player.

Restituzione:

N/A

Esempio:

rollback_set_player_prefs({
personaggio: obj_customizer.character_selected,
colore: obj_customizer.color_selected
});

Il codice precedente imposta le preferenze del giocatore, leggendo i valori da un "customizer" object, che sarebbe non gestito (locale).
