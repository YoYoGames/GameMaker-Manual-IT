rollback_define_mock_input

Questa funzione può essere utilizzata durante il Sync Test per assegnare un input temporaneo "finto" al lettore specificato. In questo modo si sovrascrivono i controlli definiti per il giocatore e qualsiasi input casuale proveniente da Sync Test.

Per informazioni sul suo utilizzo, vedere Finto ingresso.

Sintassi:

rollback_define_mock_input(player_id, input_struct);

Argomento Tipo Descrizione
player_id Reale The ID of the player to assign mock input to.
input_struct Struttura A struct containing input names as variables, and their assigned inputs as values.

Restituzione:

N/A

Esempio:

rollback_define_input({
    fire: mb_left,
    interact: vk_space,
    left: ord("A"),
    right: ord("D")
});

rollback_define_mock_input(1, {
    fire: vk_control,
    interact: vk_shift,
    left: ord("J"),
    right: ord("L")
});

Il codice qui sopra definisce innanzitutto gli input per i giocatori. Quindi definisce un finto input per il giocatore 1, mantenendo gli stessi nomi di input ma assegnando input diversi. In questo modo due persone possono giocare in locale utilizzando la tastiera.
