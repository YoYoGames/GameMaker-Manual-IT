Code DS

Un DS queue è in qualche modo simile a uno stack DS, ma funziona su una base first-in first-out (FIFO). Il valore che viene messo per primo in queue è anche il primo ad essere rimosso da esso. Potete pensarlo come un queue in un negozio per pagare, dove la persona che si trova per prima nel queue paga per prima e poi lascia il negozio. Queues sono tipicamente usati per memorizzare azioni o comportamenti (nella forma di scripts, per esempio) che devono ancora essere fatti da un'istanza ma ci sono molti altri usi.

NOTA: Come tutte le risorse dinamiche, le strutture dati occupano memoria e quindi dovrebbero sempre essere distrutte quando non sono più necessarie per evitare perdite di memoria che rallentano e alla fine fanno crashare il gioco.

NOTA: se avete bisogno di controllare se una struttura di dati esiste, potete usare la funzione ds_exists() funzione.

Esistono le seguenti funzioni che si occupano delle code DS:

  • ds_queue_create
  • ds_queue_destroy
  • ds_queue_clear
  • ds_queue_empty
  • ds_queue_size
  • ds_queue_dequeue
  • ds_queue_enqueue
  • ds_queue_head
  • ds_queue_tail
  • ds_queue_copy
  • ds_queue_read
  • ds_queue_write
