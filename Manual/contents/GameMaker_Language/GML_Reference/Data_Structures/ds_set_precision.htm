ds_set_precision

Quando si confrontano i valori, per esempio quando si cerca in una mappa o si ordina una lista, GameMaker deve decidere quando due valori sono uguali. Per strings e valori interi questo è chiaro, ma per i numeri reali, a causa di errori di arrotondamento in virgola mobile, numeri apparentemente uguali possono facilmente diventare disuguali. Per esempio, è possibile che (5 / 3) * 3 non sia uguale a 5! Per evitare questo, un valore di precisione è usato su tutte le funzioni di numeri reali, e quando la differenza tra due numeri è più piccola di questa precisione sono considerati uguali. La precisione predefinita di 0,0000001 è usata per tutte le funzioni di strutture di dati, a meno che non venga cambiata da questa funzione.

NOTA: questa precisione è usata in tutte le strutture di dati ma non in altri confronti in GML!

Sintassi:

ds_set_precision(prec);

Argomento Tipo Descrizione
prec Real The precision value (default 0.0000001)

Ritorna:

N/A

Esempio:

ds_set_precision(0.0001);

Il codice qui sopra cambierà l'impostazione di precisione predefinita per tutte le funzioni della struttura dei dati.

Indietro: Strutture di dati
Dopo: ds_exists
