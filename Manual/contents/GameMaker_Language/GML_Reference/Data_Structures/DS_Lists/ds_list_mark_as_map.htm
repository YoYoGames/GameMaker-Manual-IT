ds_list_mark_as_map

Questa funzione "marcherà" (o "flag") una data posizione all'interno di una lista DS precedentemente creata come contenente una mappa DS. Questa funzionalità è necessaria quando si codifica JSON strings (che si può creare usando json_encode()), ma può anche essere utile quando si annidano le strutture di dati, poiché gli elementi marcati in questo modo saranno automaticamente smaltiti (distrutti) quando la lista DS madre viene distrutta. Questo significa che non devi passare manualmente attraverso il contenuto della lista e distruggere le strutture dati marcate individualmente prima di distruggere la lista "madre". Tuttavia, se si elimina la posizione dell'elenco individualmente, la struttura dati che conteneva non verrà eliminata e si dovrebbe chiamare la funzione appropriata di distruzione della mappa DS prima di eliminare la posizione dell'elenco padre. Nota anche che se chiami la funzione ds_list_clear() su una lista, qualsiasi elemento contrassegnato come mappa verrà distrutto anch'esso quando la lista verrà cancellata.

Vale anche la pena notare che se la posizione che è stata segnata cambia all'interno della lista, il "marchio" si sposterà con essa, quindi se avete segnato la posizione 3 nella lista (per esempio) e poi inserite altri 2 elementi sotto di essa in modo che si sposti fino alla posizione 5, sarà ancora segnato come una mappa.

Sintassi:

ds_list_mark_as_map(id, pos);

ArgomentoTipo Descrizione
id The id of the list to mark.
pos The position within the list to mark.

Ritorna:

DS Map ID or -1 (if it fails)

Esempio:

var sub_map = ds_map_create();
ds_map_add(sub_map, "player", player_array);
ds_map_add(sub_map, "enemy", enemy_array);
ds_list_add(j_list, sub_map);
ds_list_mark_as_map(j_list, 0);

Il codice di cui sopra crea una mappa DS e poi la popola con due chiavi, ciascuna contenente un array di valori. Questa mappa viene poi aggiunta nella lista DS data, e la posizione "segnata" come tale in modo che possa essere codificata correttamente in seguito.
