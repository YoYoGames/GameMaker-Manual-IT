ds_priority_read

Con questa funzione potete ricreare una priorità DS salvata (una che è stata precedentemente scritta come string usando ds_priority_write()). Dovete prima creare una nuova priorità DS in cui leggere il string, e se la priorità DS esiste già e ha informazioni memorizzate, allora queste saranno cancellate prima della lettura. Questa funzione è di vitale importanza quando si creano meccanismi di salvataggio/caricamento per il gioco.

Notate che se il DS specificato string è stato scritto da GameMaker: Studio 1.2.x runtime (o più vecchio), dovreste specificare l'argomento opzionale "legacy", impostandolo su true poiché il formato string è cambiato dopo.

Sintassi:

ds_priority_read(id, str, [legacy]);

Argomento Tipo Descrizione
id ID di priorità DS The id of the data structure to write.
str Stringa The string to read from.
legacy Booleano OPTIONAL Può essere true o false o omesso completamente.

Ritorna:

N/A

Esempio:

p_queue = ds_priority_create();
ini_open("save.ini");
var str = ini_read_string("P_Queues", "0", "");
if (str != "")
{
    ds_priority_read(p_queue, str);
}
ini_close();

Il codice qui sopra crea una priorità queue e memorizza l'indice nella variabile "p_queue". Poi apre un file ini e legge un string da quello, controllando per assicurarsi che il string non venga prima restituito come vuoto. Questo string viene poi letto nella priorità DS appena creata.
