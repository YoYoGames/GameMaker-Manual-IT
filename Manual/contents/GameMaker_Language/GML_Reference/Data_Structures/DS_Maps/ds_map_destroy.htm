ds_map_destroy

Le mappe DS occupano spazio in memoria, che viene assegnata loro quando vengono create. Ciò significa che è anche necessario liberare questa memoria quando la mappa DS non è necessaria per evitare errori, perdite di memoria e perdita di prestazioni durante l'esecuzione del gioco. Questa funzione fa proprio questo. Notate che la distruzione di una mappa de-referenzia qualsiasi struttura dati memorizzata nella mappa, causando una perdita di memoria, quindi dovreste passare attraverso la mappa e distruggere tutti gli elementi della struttura dati manualmente prima di distruggerla per evitare questo. Il unica volta che questo non è necessario è quando hai contrassegnato qualsiasi elemento nella mappa come una lista DS o come un'altra mappa DS, nel qual caso questi elementi saranno distrutti e la loro memoria ripulita automaticamente.

IMPORTANTE! Quando si crea una struttura di dati, il valore dell'indice per identificarla è un valore intero che inizia con 0. Questo significa che strutture di dati di tipi diversi possono avere lo stesso valore di indice, quindi in caso di dubbio si dovrebbe usare l'opzione ds_exists() prima di accedervi. Notate anche che gli indici vengono riutilizzati, quindi un valore di indice di una struttura di dati distrutta può essere usato da una nuova struttura creata in seguito, quindi raccomandiamo sempre di impostare la variabile che conteneva l'indice DS a -1 dopo la distruzione.

Sintassi:

ds_map_destroy(id);

ArgomentoTipo Descrizione
id The id of the map to destroy.

Ritorna:

Esempio:

ds_map_destroy(inventory);
inventory = -1;

Il codice di cui sopra distruggerà la mappa DS con l'id indicizzato nella variabile "inventario".
