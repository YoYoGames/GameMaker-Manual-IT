Web e HTML5

GameMaker fornisce varie funzioni per interagire con Internet o con un server, e possono essere utilizzate su tutte le piattaforme di destinazione disponibili. Queste funzioni sono:

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

A parte le funzioni generali elencate sopra, il target HTML5 ha alcune funzioni web aggiuntive che sono specificamente progettate per essere utilizzate dai giochi che lo utilizzano:

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

Oltre a queste funzioni per il modulo target HTML5, c'è anche una speciale variabile JavaScript che può essere usata in quelle rare occasioni in cui il contenuto della cartella html5game è memorizzato in un posto completamente diverso (URL). Normalmente questo path è hard-coded nel file JavaScript ed è relativo alla posizione da cui viene chiamato il JavaScript, ma è possibile impostare la variabile g_GameMakerHTML5Dir prima della chiamata al gioco script e il gioco verrà eseguito dalla posizione appena definita. Per esempio:

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

Vale anche la pena notare che è possibile contrassegnare le funzioni script usando il prefisso gmcallback_ che significa che non saranno offuscate quando si compila il progetto per HTML5. Questo è molto utile quando si usano le funzioni clickable_* sopra elencate (per maggiori informazioni, vedere qui).
