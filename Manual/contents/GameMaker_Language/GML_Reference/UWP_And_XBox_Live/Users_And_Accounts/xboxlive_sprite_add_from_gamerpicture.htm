xboxlive_sprite_add_from_gamerpicture

Con questa funzione potete ottenere l'immagine del giocatore per un dato puntatore ID utente. La funzione funziona in modo asincrono, e attiverà un evento asincrono Image Loaded per informarvi che la funzione è finita (come con le chiamate HTML5/URL a sprite_add()). L'evento Asynchronous Image Loaded avrà una mappa DS memorizzata nella variabile incorporata async_load. La mappa contiene le seguenti informazioni:

  • "Nome del file": Il path completo del file richiesto.
  • "id": L'ID della risorsa che avete caricato. Questo sarà lo stesso della variabile a cui avete assegnato la risorsa.
  • "stato": Restituisce un valore inferiore a 0 per un errore.

NOTA: Quando caricate dinamicamente un sprite nel vostro gioco su runtime, dovete ricordarvi di rimuoverlo di nuovo usando sprite_delete() quando non è più necessario, altrimenti c'è il rischio di una perdita di memoria che rallenterà ed eventualmente bloccherà il gioco.

Sintassi:

xboxlive_sprite_add_from_gamerpicture(user_id, imagesize, xorig, yorig);

ArgomentoTipo Descrizione
user_id The user ID (a pointer) to get the gamer picture for
imagesize Size in pixels of the sprite to be returned
xorig Indicate the x position of the origin in the sprite
yorig Indicate the y position of the origin in the sprite

Ritorna:

Async Request ID

Esempio:

var _a = 0;
var _num = xboxlive_get_user_count();
for (var i = 0; i < _num; ++i;)
{
    var _uid = xboxlive_get_user(i);
    if (_uid != pointer_null)
    {
        global.UserName[_a] = xboxlive_gamedisplayname_for_user(_uid);
        global.UserScore[_a] = xboxlive_gamerscore_for_user(_uid);
        global.UserRep[_a] = xboxlive_reputation_for_user(_uid);
        global.UserAvatar[_a] = xboxlive_sprite_add_from_gamerpicture(_uid, 256, 0, 0);
        ++a;
    }
}

Il codice di cui sopra loops attraverso gli utenti registrati e memorizza i loro dati di gioco in vari array globali.
