xboxlive_matchmaking_find

Questa funzione cercherà una sessione multigiocatore utilizzando il modello di sessione, il matchmaking hopper e il modello Secure Device Association, che dovreste aver creato in precedenza nella dashboard XDP. L'ID utente può essere recuperato per un particolare game pad chiamando la xboxlive_user_for_pad() funzione. La funzione restituirà un valore unico di Request ID, che può essere utilizzato per identificare il corretto Evento Sociale Asincrono per questa funzione. Questo evento sarà attivato quando la sessione è stata creata e conterrà una DS Map nella variabile async_load con le seguenti coppie chiave/valore:

  • "requestid" - contiene l'ID della richiesta come restituito dalla funzione chiamante
  • "status" - contiene il string "session_created" per informare su quale tipo di evento è stato attivato.
  • "num_results" - contiene 1 o più se è stata trovata una sessione, altrimenti contiene 0.
  • "sessionid<index> " - contiene l'ID della sessione, o -1 in caso di fallimento(<index/gt; è un valore da 0 a "num_results" - 1).
  • "sessionOwner<index> " - contiene l'ID dell'host della sessione se è stata trovata una sessione(<index> è un valore da 0 a "num_results" - 1).
  • "error" - contiene 0 in caso di successo, o -1 in caso di fallimento

Sintassi:

xboxlive_matchmaking_find(user_id, template, hopper, sdatemplate, [matchattributes]);

Argomento Tipo Descrizione
user_id ID utente Xbox The user ID pointer to use
template Stringa The name of the session template
hopper Stringa The name of the matchmaking hopper
sdatemplate Stringa The name of the secure device association template
matchattributes Real OPTIONAL Gli attributi della partita

Ritorna:

Async Request ID

Esempio:

var userID = xboxone_user_for_pad(global.PadIndex);
result = xboxone_matchmaking_find(userID, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic");

Quanto sopra recupererà l'ID utente per l'utente sul gamepad dato e poi cercherà di trovare una sessione di matchmaking per lui.
