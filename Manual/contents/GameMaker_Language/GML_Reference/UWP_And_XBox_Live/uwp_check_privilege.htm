controlla il privilegio

Con questa funzione potete controllare che un dato utente abbia un privilegio scelto. La funzione restituirà true o false nel System Asynchronous Event a seconda che il privilegio sia abilitato o meno, e se impostate l'argomento attempt_resolution a true e il privilegio non è abilitato, aprirà anche una finestra di dialogo di sistema (sospendendo il gioco) per chiedere all'utente di aggiornare l'account o qualsiasi cosa sia richiesta per ottenere il privilegio. Se l'utente abilita l'opzione richiesta, la funzione restituirà true.

La funzione richiede l'uso di uno dei seguenti elementi Costante di privilegio UWPper la richiesta del privilegio:

Costante di privilegio UWP
Costante Descrizione
uwp_privilege_internet_browsing Controlla se la navigazione in internet è permessa
uwp_privilege_social_network_sharing Controlla se la condivisione sui social network è permessa
uwp_privilege_share_kinect_content Controlla se la condivisione del controller Kinect è consentita
uwp_privilege_video_communications Controlla se la comunicazione video è permessa
uwp_privilege_communications Controlla se la comunicazione via internet è permessa
uwp_privilege_user_created_content Controlla se l'accesso al contenuto creato dall'utente è permesso
uwp_privilege_multiplayer_sessions Controlla se le sessioni multiplayer online sono permesse
uwp_privilege_sessions Controlla se le sessioni online sono permesse
uwp_privilege_fitness_upload Controlla se il caricamento dei dati di fitness è permesso

Una volta che la funzione è stata chiamata, su XBox One scatenerà un evento di sistema asincrono quando il risultato della callback è stato ricevuto, che avrà la speciale mappa DS async_load. Questa mappa dovrebbe poi essere analizzata per la seguente chiave:

  • "event_type" - dovrebbe contenere il valore "check_privilege_result" se l'evento è stato innescato da questa funzione

Se il tipo di evento si riferisce a questa funzione, allora ci saranno anche i seguenti tasti aggiuntivi quando la funzione viene chiamata su XBox:

  • "risultato" - Sarà uno o più dei seguenti Costante di risultato del privilegio di UWPs (valori combinati):
    UWP Privilegio Risultato Costante
    Costante Descrizione Valore
    uwp_privilege_result_no_issue Non ci sono problemi di privilegi per l'utente 0
    uwp_privilege_result_purchase_required L'utente deve acquistare qualcosa di aggiuntivo, di solito un abbonamento, per accedere 1
    uwp_privilege_result_aborted Il controllo è stato interrotto 2
    uwp_privilege_result_banned L'utente è bandito 4
    uwp_privilege_result_restricted L'utente è limitato dall'accesso, di solito attraverso il controllo dei genitori 8

  • "privilegio" - sarà una delle costanti elencate sopra per l'uso nella funzione, in modo da poter controllare quale privilegio avete richiesto nel caso di controlli multipli.

Se il progetto viene eseguito su una piattaforma UWP che non è XBox One, allora l'evento asincrono non verrà attivato, e invece la funzione restituirà immediatamente con -1, 0, o 1 dove:

  • -1 significa -1 l'utente non ha il privilegio
  • 0 significa che la funzione non è stata in grado di interrogare l'utente
  • 1 significa che l'utente ha il privilegio.

Sintassi:

uwp_check_privilege(user_id, privilege, attempt_resolution);

Argomento Tipo Descrizione
user_id ID utente Xbox The User ID pointer to check.
privilege Costante di privilegio UWP The privilege constant to check (see description, above).
attempt_resolution Booleano Enable attempting a resolution if set to true, o ignorare se impostato su false.

Ritorna:

Real

Esempio:

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
{
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
}

Questo controlla per vedere se l'utente ha effettuato l'accesso e, se lo fa, controlla il permesso per le sessioni multiplayer.
