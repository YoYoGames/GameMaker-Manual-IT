mp_potential_path_object

Questa funzione calcola un path per l'istanza dalla sua posizione e orientamento attuali alla posizione specificata dagli argomenti xgoal, ygoal. Utilizza la dimensione di passo indicata mentre cerca di evitare la collisione con le istanze di object specificate dall'argomento "obj". Come nella funzione mp_potential_step() utilizza passi di campo potenziale per realizzare questo e il modo in cui lo fa può essere modificato attraverso la funzione mp_potential_settings(). Il path indicato deve già esistere e sarà sovrascritto dal nuovo path. La funzione restituisce se un path è stato trovato (vero) o no (falso).

Per evitare che la funzione continui a calcolare all'infinito è necessario fornire un fattore maggiore di 1 - la funzione si fermerà e segnalerà il fallimento se non riesce a trovare un path più breve di questo fattore per la distanza tra l'inizio e la meta. Un fattore di 4 è normalmente sufficiente, ma se vi aspettate lunghe deviazioni potreste allungarlo. Se fallisce, viene comunque creato un path che corre nella direzione approssimativa della meta, ma non la raggiunge.

NOTA: Questa funzione non sposta l'istanza. Imposta solo un path ed è necessario utilizzare le funzioni Path per il movimento.

Sintassi:

mp_potential_path_object(path, xgoal, ygoal, stepsize, factor, obj)

ArgomentoTipo Descrizione
path index of the path that is to be used by the function
xgoal The target x position.
ygoal The target y position.
stepsize The speed the instance moves in pixels per step.
factor This number is used to prevent an infinite loop. It should be larger than 1. See the function description for more information.
obj The object that is to block the path. Can be an object index, an instance id or the special keyword ,all

Ritorna:

Esempio:

path = path_add();
mp_potential_path_object(path, obj_Player.x, obj_Player.y, 3, 4, obj_Wall);
path_start(path, 3, 0, 0);

Il codice precedente crea una nuova risorsa path e memorizza il suo indice nella variabile "path". Poi genera un path usando mp_potential_path tra il object con questo codice e il object "obj_Player", controllando le collisioni con il object "obj_Wall". Infine inizia il object lungo quel path, anche se la funzione non è riuscita a restituire un percorso completo per "obj_Player".
