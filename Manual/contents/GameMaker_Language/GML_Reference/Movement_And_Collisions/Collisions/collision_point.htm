punto di collisione

Collision_point controlla il punto specificato dagli argomenti x1,y1 per una collisione con qualsiasi istanza di object specificata dall'argomento "obj". questo controllo può essere preciso o no, ma per abilitare le collisioni precise, il object o l'istanza che si sta controllando deve anche avere collisioni precise abilitate per il suo sprite. In caso contrario, il controllo di default è basato sui bounding box. L'immagine seguente illustra come funziona:

Ricordate, perché le collisioni precise siano considerate sia il object sprite che la funzione di collisione devono avere precise marcate come on. Va anche notato che il valore di ritorno della funzione può essere l'id di una qualsiasi delle istanze considerate in collisione, quindi se tre istanze si sovrappongono in quel punto, uno qualsiasi dei loro id potrebbe essere il valore di ritorno della funzione.

Notate che invece di un indice object potete fornire un'istanza id per controllare un'istanza specifica, o le parole chiave di istanza all, o other (a seconda dell'evento e dello scopo del codice corrente).

Sintassi:

collision_point(x, y, obj, prec, notme);

Argomento Tipo Descrizione
x Real The x coordinate of the point to check.
y Real The y coordinate of the point to check.
obj ID dell'asset dell'oggetto o dell'istanza The object to check for instance collisions.
prec Booleano Whether the check is based on precise collisions (trueche è più lento) o il suo bounding box in generale (false, più veloce).
notme Booleano Whether the calling instance, if relevant, should be excluded (true) o no (false).

Ritorna:

Instance ID or noone

Esempio:

if collision_point(x, y, obj_Cursor, false, true)
{
    score += 10S;
}

Qui stiamo controllando il punto nella posizione del object che ha il codice per il object "obj_Cursor". Se ce n'è uno, allora aggiungiamo 10 alla variabile del punteggio.
