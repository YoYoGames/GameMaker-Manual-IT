Collisioni

Quando si pianificano i movimenti o si decidono certe azioni, è spesso importante vedere se ci sono collisioni con altri objects in certi punti del mondo di gioco, e spesso scegliere la collisione giusta per il lavoro è il compito più importante di tutti. GameMaker ha un certo numero di funzioni incorporate per aiutarvi a gestire le collisioni correttamente e in conformità con le esigenze del vostro progetto.

NOTA Le funzioni di collisione usavano valori interi nelle versioni precedenti di GameMaker, ma ora usano valori in virgola mobile, il che significa che le funzioni di collisione non arrotondano più gli argomenti non interi. Per maggiori informazioni, leggete Collision Compatibility Mode.

Le seguenti funzioni trattano i vari modi in cui potete far controllare ad un'istanza le collisioni con un'altra istanza:

  • place_empty
  • place_free
  • place_meeting
  • position_empty
  • position_meeting
  • position_change
  • position_destroy
  • instance_place
  • instance_place_list
  • instance_position
  • instance_position_list

Controllo avanzato delle collisioni

Le seguenti routine possono essere utilizzate per questo (così come altre funzioni dettagliate nelle sezioni relative a Moving Around e Instances). Queste funzioni di collisione lavorano tutte in modo leggermente diverso, ma mantengono tre argomenti comuni che spiegheremo qui:

  • obj: Qui è dove specifichiamo il object con cui la funzione deve controllare la collisione. Può essere un id di istanza, un tipo object, o la parola chiave speciale GM all. Vale anche la pena notare che se il tipo object scelto è il genitore designato di altri objects, allora anche tutti i figli di quel object saranno inclusi in questo controllo.
  • prec: Questo può essere o true o false e si riferisce alla precisione del controllo delle collisioni. Se impostato su true, allora verranno calcolate le collisioni precise (ma solo se il object che viene controllato ha "precise" marcato per il suo rilevamento delle collisioni nelle sue proprietà della maschera) che è un compito leggermente più complesso per la CPU e quindi dovrebbe essere usato con cautela. Se false allora le collisioni saranno calcolate in base al riquadro di delimitazione del object da controllare.
  • notme: Potresti aver bisogno di controllare le collisioni con un object che ha lo stesso indice del object che esegue il codice, quindi per evitare che il codice trovi una collisione con se stesso, puoi impostare questo a true. Se stai cercando specificamente l'istanza del object che esegue il codice, o non è importante, può essere lasciato come false.

Le funzioni di base restituiranno o l'ID di un'istanza trovata in collisione, o la parola chiave speciale noone quando non c'è collisione, mentre le funzioni di lista restituiranno il numero di istanze che sono in collisione e popoleranno una lista DS predefinita con i loro ID. Si noti che se ci sono collisioni multiple con le aree definite da queste funzioni e le istanze del dato object, solo un ID di istanza viene restituito, e può essere una qualsiasi delle istanze in collisione.

Esistono le seguenti funzioni che si occupano delle collisioni avanzate.

  • collision_circle
  • collision_circle_list
  • collision_ellipse
  • collision_ellipse_list
  • collision_line
  • collision_line_list
  • collision_point
  • collision_point_list
  • collision_rectangle
  • collision_rectangle_list

Controllo delle collisioni senza maschera

Tutte le funzioni di cui sopra sono relative alle istanze di controllo delle collisioni, e come tali si basano sulla maschera di collisione che è definita per l'istanza. Tuttavia, ci sono molti momenti in cui avete bisogno di controllare le "collisioni" con un punto o un'area, specialmente quando la vostra istanza non ha un sprite assegnato, o quando state lavorando con il mouse ecc. Pertanto GameMaker fornisce anche le seguenti funzioni per aiutarvi in queste situazioni:

  • point_in_rectangle
  • point_in_triangle
  • point_in_circle
  • rectangle_in_rectangle
  • rectangle_in_triangle
  • rectangle_in_circle

Collisioni fisiche

Quando si usa la fisica integrata, le funzioni di collisione di cui sopra non sono garantite per funzionare per le istanze abilitate alla fisica. Questo è dovuto principalmente al fatto che queste istanze non usano più la maggior parte delle normali variabili incorporate (invece, le istanze abilitate alla fisica hanno il proprio set di variabili) e non usano nemmeno la maschera di collisione o il bounding box, poiché usano invece le fixture. Tuttavia c'è una funzione speciale per testare le collisioni con la fisica abilitata che controlla le fixture sovrapposte:

  • physics_test_overlap
