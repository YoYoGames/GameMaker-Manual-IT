Potete usare questa funzione per controllare e vedere se l'istanza chiamante entra in collisione con qualsiasi altra istanza di un object o con tutte le istanze del vostro gioco. Affinché questa collisione si risolva correttamente, l'istanza che esegue il codice deve avere una maschera di collisione valida (o per il sprite stesso, o attraverso la funzione mask_index) e registrerà solo collisioni con quelle istanze che hanno anch'esse una maschera valida.

La funzione verifica se non ci sono collisioni nel caso in cui l'istanza chiamante sia posta in una posizione specifica, e si può fornire un argomento opzionale per raffinare il controllo per verificare solo se una posizione è priva di collisioni con istanze del tipo dato. Si noti che se non viene fornito alcun ID opzionale object, il controllo verrà effettuato su tutte le istanze all'interno di room. Il controllo delle collisioni sarà preciso o basato sul riquadro di delimitazione dell'istanza, a seconda del tipo di maschera di collisione selezionata, ma affinché le collisioni precise funzionino correttamente, tutte le istanze nel controllo dovrebbero avere maschere di collisione precise.

Si noti che le coordinate x/y date saranno flottate al numero intero più vicino prima che il controllo venga eseguito.

Sintassi:

place_empty(x, y, [object_id]);

Argomento Tipo Descrizione
x Real The x position to check.
y Real The y position to check.
[object_id] Oggetto Attività OPTIONAL L'oggetto da controllare.

Ritorna:

Boolean

Esempio:

if place_empty(mouse_x, mouse_y, obj_Enemy)
{
    x = mouse_x;
    y = mouse_y;
}

Il codice di cui sopra controllerà la collisione con qualsiasi altra istanza di object "obj_Enemy", come se l'istanza chiamante fosse posta nella stessa posizione del mouse. Se non viene rilevata alcuna collisione, allora l'istanza ha le sue coordinate x/y impostate su quelle del mouse.
