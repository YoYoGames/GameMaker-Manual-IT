Funzioni di runtime

La definizione generale di una funzione è simile a questa:

Una funzione ha un ingresso e un'uscita e l'uscita è correlata all'ingresso.

In GameMaker questo si traduce in due cose diverse, ma entrambe funzionano allo stesso modo:

  • Funzioni di runtime - Una funzione di runtime è una funzione fornita come parte del linguaggio GameMaker (GML). Esse sono integrate nel linguaggio e possono essere utilizzate per far accadere cose nei vostri giochi. Sono disponibili numerose funzioni di GML runtime, tutte spiegate nella sezione Riferimento al linguaggio del manuale.
  • Funzioni e metodi di script - Scrittura funzioni e MVariabili dei metodi sono funzioni create da voi stessi utilizzando gli elementi di base del linguaggio GameMaker (GML), che possono includere le funzioni di runtime come parte del loro codice.

In entrambi i casi una funzione ha la forma di un nome di funzione, seguito dagli argomenti di input tra parentesi () e separati da virgole (se la funzione non ha argomenti di input, si usano solo le parentesi). Questa pagina si concentra sulle funzioni di runtime (cioè quelle che sono incorporate in GameMaker per formare GameMaker Language), ma le regole generali mostrate di seguito si applicano a tutti i tipi di funzione.

Ecco uno schema della struttura di una funzione:

<function>(<arg0>, <arg1> ,... <argN>);

Alcune funzioni possono restituire valori e possono essere utilizzate nelle espressioni, mentre altre eseguono semplicemente dei comandi, come illustrato nei due esempi di funzione runtime che seguono:

// Destroy the calling instance
// This requires no arguments and returns nothing
instance_destroy();

// Get the distance from the current instance position to the mouse position
// This takes four arguments and returns a real value
dist = point_distance(x, y, mouse_x, mouse_y);

NOTA Ogni volta che il manuale riporta "N/A" come valore di ritorno per una funzione, significa che la funzione non è destinata a restituire alcun valore come parte del suo funzionamento. Se si cerca di verificare un valore di ritorno da queste funzioni, si otterrà il valore undefined.

A volte il valore di ritorno di una funzione può essere un valore che si desidera utilizzare in un'assegnazione, ma occorre notare che è impossibile utilizzare una funzione direttamente come lato sinistro di un'assegnazione. Ad esempio, il seguente codice per impostare la velocità di un'istanza darebbe un errore:

instance_nearest(x, y, obj).speed = 0;

Il valore di ritorno dell'espressione in questo esempio di codice è un numero intero (il valore ID univoco dell'istanza più vicina) e quindi deve essere racchiuso tra parentesi per essere usato in questo modo e indirizzare correttamente l'istanza richiesta (per ulteriori informazioni, vedere Indirizzare le variabili in altre istanze ). Il codice di cui sopra sarebbe scritto correttamente come:

(instance_nearest(x, y, obj)).speed = 0;

//or more correctly still

var inst = instance_nearest(x, y, obj);
inst.speed = 0;

function test ()
{
    test();

    if (test)
    {
        test();
    }
}
