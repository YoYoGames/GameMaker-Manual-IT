Codice di commento

Quando si lavora in collaborazione con altre persone, quando si ha a che fare con un grande progetto, o anche solo per il proprio riferimento e debugging, lasciare commenti nel codice degli eventi e scripts è molto importante. In GameMaker avete vari meccanismi per aiutarvi a lasciare note e commenti su sezioni di codice e persino interi blocchi di codice che possono essere visti nella lista degli eventi di object che state attualmente modificando. In questo modo potete lasciare note a voi stessi e ai vostri colleghi, o per spiegare un pezzo di codice particolarmente complicato, o anche solo lasciare un promemoria su cosa fa qualcosa, o cosa volete che qualcosa faccia in futuro.

Per lasciare un semplice commento di una riga, la prima cosa che puoi fare è usare // prima del testo. Per esempio:

//initialize variables
sr = 10;
hp = 100;

Si possono anche lasciare commenti su più righe, per dare credito, per omettere una sezione completa di codice per debugging, o anche per lasciare note su come si vuole cambiare il codice in futuro. Per questo si può usare /* ... */ come questo:

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

Oltre ai commenti si possono usare anche speciali tag #region per marcare una sezione di codice per il Code Folding, e ogni regione può contenere un commento. Come funziona è che si aggiunge un tag #region - insieme a qualsiasi testo di commento - all'inizio di una sezione di codice e poi si aggiunge un tag corrispondente #endregion alla fine della sezione e poi si può "piegare" il codice, cioè: nasconderlo in modo da poter ridurre il numero di linee di codice visibili nell'editor in qualsiasi momento:

Il codice attuale assomiglia a questo:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Infine, vale la pena ricordare che quando si lavora con gli eventi nell'Object Editor, si può usare uno speciale commento "description" all'inizio dell'evento per dire quali sono i contenuti generali dell'evento, e questo verrà mostrato nella finestra degli eventi dell'Editor Object. Questo dovrebbe andare nella primissima linea dell'editor di codice, ed è formattato come segue:

/// @description <This Is An Event Description>

Questo sarà ora mostrato nella finestra dell'evento, ad esempio:

Si noti che è possibile definire un commento predefinito da creare per tutti gli eventi dalle Preferenze dell'editor di oggetti.

Infine, se state scrivendo funzioni di script e volete avere un certo controllo su come vengono trattate da IDE (cioè: avere il completamento automatico o mostrare i parametri script ) allora dovreste scrivere i vostri commenti usando la notazione JSDoc come spiegato qui.
