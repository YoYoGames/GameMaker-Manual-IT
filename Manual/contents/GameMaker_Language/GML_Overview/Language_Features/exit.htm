La dichiarazione exit ha la seguente sintassi:

exit;

exit termina semplicemente l'esecuzione della funzione di script corrente, methodoppure event. Notate che qui c'è una leggera differenza d'uso a seconda dello scopo:

  • Se usate exit in una funzione o metodo script, uscirà semplicemente dalla funzione e ritornerà al codice che ha chiamato la funzione.
  • Se usate exit in un blocco di codice all'interno dell'evento di un oggetto, uscirà dall'intero evento anche se ci sono vari blocchi di codice separati dopo che exit è stato chiamato.
  • Se usate exit in un evento padre e chiamate quell'evento usando event_inherited() in un figlio object, solo l'evento padre uscirà e l'evento figlio (che ha chiamato event_inherited()) continuerà.

Quando è usato in un evento, exit è tipicamente usato per evitare che un'istanza esegua ulteriore codice quando una specifica condizione è stata soddisfatta (o meno). Il codice qui sotto delinea un esempio di come potrebbe essere usato, in questo caso all'interno di un evento Collision, anche se può essere usato in qualsiasi evento.

if (!visible)
{
    exit;
}

other.hp -= attack;
other.coins -= 4;
coins += 4;

Il codice qui sopra controlla se l'istanza corrente non è visibile, in questo caso esce dal blocco di codice, altrimenti va avanti ed esegue il resto del codice.

NOTA: non termina l'esecuzione del gioco. Per questo è necessario utilizzare la funzione game_end().
