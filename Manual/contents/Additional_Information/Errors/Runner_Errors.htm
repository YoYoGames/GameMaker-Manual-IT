Errori del corridore

Anche dopo aver controllato la sintassi nell'editor di codice e dopo aver fatto controllare il codice dal compilatore per verificare la presenza di errori di compilazione, ci sono ancora occasioni in cui qualcosa può andare storto. Nella maggior parte dei casi viene lanciato un errore di runner della macchina virtuale ( chiamato anche eccezione di runtime), che ha l'aspetto seguente:

Gli errori di esecuzione sono generalmente più gravi degli errori di compilazione, poiché significano che c'è qualcosa di gravemente sbagliato nel codice che né l'editor di codice né il compilatore sono stati in grado di rilevare. Quando si verifica un errore, è possibile utilizzare il pulsante Copia del pop-up per copiare l'errore negli appunti e incollarlo in un file di testo (o in un altro posto) per riferimenti futuri.

La struttura di questo errore è la seguente:


  • Mostra l'evento in cui si è verificato l'errore e il sito Object (se applicabile).

  • Mostra l'errore effettivo. Quindi, mostra l'ambito dell'errore e il tipo di errore. Il primo valore tra le parentesi sarà l'ID dell'istanza che stava eseguendo il codice - o potrebbe essere uno dei valori speciali elencati nella tabella sottostante - e il secondo è un valore interno che GameMaker utilizza per identificare i bug e può essere ignorato. Nel caso dell'immagine qui sopra, sta dicendo che la variabile _image non è stata impostata (cioè non esiste prima di essere acceduta) su object obj_EA_DEMO_Control.

  • Questo mostra il numero di riga in cui si verifica l'errore e un frammento di codice che mostra la posizione. Si noti che questo non è sempre preciso al 100% a causa del modo in cui il gioco è compilato, ma dovrebbe essere sufficiente per aiutare a trovare esattamente il problema.

Come già menzionato, alcuni messaggi di errore saranno identificati non da un valore di ID istanza, ma piuttosto da un valore negativo . Questi valori possono essere utilizzati per individuare la natura esatta dell'errore e a cosa si riferisce, con i seguenti valori possibili:

Prefisso Ambito di applicazione
-1 Auto
-2 Altro
-3 Tutti
-4 Nessuno
-5 Globale
-6 Non specificato
-7 Locale

I possibili errori del runner della macchina virtuale sono i seguenti:

Errore Messaggio Operazione Descrizione
DoSet Tipo di confronto non valido Tipi di dati Ciò indica che il corridore ha cercato di confrontare due data types incompatibili, come un numero reale con un string.
DoConv Errore di esecuzione Tipi di dati Denota un errore nella conversione di un tipo di dati in un altro.
DoAdd Errore di esecuzione Matematica Qualcosa è andato storto quando si utilizza l'espressione di addizione (+).
DoMul Errore di esecuzione Matematica Qualcosa è andato storto durante l'utilizzo dell'espressione di moltiplicazione (*).
DoSub Errore di esecuzione Matematica Qualcosa è andato storto quando si utilizza l'espressione di sottrazione (-).
DoSub Execution Engine - Impossibile operare sul tipo string Matematica Si sta cercando di sottrarre il tipo sbagliato di variabili (ad esempio, sottrarre un string da un reale).
DoDiv Errore di esecuzione Matematica Qualcosa è andato storto quando si utilizza l'espressione di divisione (/ o div).
DoDiv Execution Engine - Impossibile operare sul tipo string Matematica Si sta cercando di dividere il tipo sbagliato di variabili (ad esempio dividere un string per un reale).
DoDiv Dividere per zero Matematica Si sta tentando di dividere per 0 (si noti che questo accade solo quando si usa la divisione tra interi, dividere un reale (non nullo) per 0 darà come risposta l'infinito, dividere zero per zero darà come risposta NaN). È possibile verificare questi valori con (is_infinity) e (is_nan).
DoMod Errore di esecuzione Matematica Qualcosa è andato storto quando si utilizza l'espressione modulo (mod).
DoMod Execution Engine - Impossibile operare sul tipo string Matematica Si sta cercando di usare modulo (mod) su un tipo di variabili sbagliato (per esempio mod un string di un reale).
DoAnd Errore di esecuzione In senso stretto Qualcosa è andato storto quando si utilizza l'espressione bitwise "and" (&).
DoAnd Execution Engine - Impossibile operare sul tipo string In senso bitonale Si sta tentando di usare "e" bitwise (&) su variabili di tipo sbagliato (ad esempio, si sta tentando di "e" un string con un reale).
DoOr Errore di esecuzione In senso bitonale Qualcosa è andato storto quando si utilizza l'espressione bitwise "or" (|).
DoOr Execution Engine - Impossibile operare sul tipo string In senso bitonale Si sta cercando di usare "or" (|) su un tipo sbagliato di variabili (ad esempio cercando di "or" un string con un reale).
DoXor Errore di esecuzione In senso bitonale Qualcosa è andato storto quando si utilizza l'espressione bitwise "xor" (^).
DoXor Execution Engine - Impossibile operare sul tipo string In senso bitonale Si sta cercando di usare "xor" (^) su un tipo sbagliato di variabili (ad esempio si sta cercando di "xorare" un string con un reale).
DoShl Errore di esecuzione In senso stretto Qualcosa è andato storto durante lo spostamento di bit a sinistra (<<) di un valore.
DoShl Execution Engine - Impossibile operare sul tipo string In senso bitonale Si sta cercando di bitshiftare a sinistra (<<) il tipo sbagliato di variabili (ad esempio cercando di bitshiftare un string).
DoShr Errore di esecuzione In senso bitonale Qualcosa è andato storto durante il bitshifting verso destra (>>) di un valore.
DoShr Execution Engine - Impossibile operare sul tipo string In senso bitonale Si sta cercando di eseguire il bitshift destro (>>) sul tipo sbagliato di variabili (ad esempio, si sta cercando di eseguire il bitshift di una string).
DoNeg Errore di esecuzione Negare Si sta cercando di trasformare un tipo di variabile in un negativo quando questo tipo non consente tale operazione.
DoNot Errore di esecuzione Negare Si sta cercando di "non" un tipo di variabile quando questo tipo non consente tale operazione.
Push Errore di esecuzione - Indice della variabile fuori dall'intervallo (var) Pila La variabile a cui si accede non rientra nell'intervallo stabilito per il corridore.
Push Ottenere una variabile (var) Pila La variabile data non è stata definita o è sconosciuta.
Pop Indice della variabile fuori dall'intervallo (var) Pila La variabile a cui si accede non rientra nell'intervallo stabilito per il corridore.
Pop Ottenere una variabile (var) Pila La variabile data non è stata definita o è sconosciuta.
With Non si può usare il globale nell'istruzione with Con Si è tentato di utilizzare "global" come variabile all'interno di un'istruzione "with", vale a dire:
with (global)
   {
   //do something;
   }
With Non è possibile utilizzare local nell'istruzione with Con Si è tentato di utilizzare "local" come variabile all'interno di un'istruzione "with", vale a dire:
with (local)
   {
   //do something;
   }
DoCall Errore del tipo di motore di esecuzione Motore Si tratta di un errore non definito all'interno della macchina virtuale. In caso di errore, è necessario segnalare il problema (vedere: il menu Aiuto per informazioni su come farlo).
Stack Overflow - Motore Un overflow dello stack si verifica quando viene utilizzata troppa memoria nello stack delle chiamate e quando il gioco tenta di utilizzare più spazio di quello disponibile nello stack delle chiamate (cioè quando tenta di accedere alla memoria oltre i limiti dello stack delle chiamate, il che è essenzialmente un overflow di buffer ), si dice che lo stack trabocchi, causando un crash del programma. Riavviare il computer e GameMaker e, se l'errore persiste, contattare l'assistenza e/o segnalare un bug (come spiegato sopra).
