Creare un'esercitazione su Marketplace

La creazione di un tutorial è simile a quella di un normale pacchetto Marketplace asset , solo che ora si devono aggiungere file specifici inclusi e includere una pagina iniziale di tutorial (e una pagina video opzionale). Il Tutorial workspace avrà un aspetto simile a questo:Come spiegato in precedenza per il pacchetto generale Marketplace Assets, è necessario selezionare un editore e dare al tutorial un nome e un ID univoco (formato URL inverso), quindi nel menu a tendina "tipo" selezionare Tutorial. In questo modo si potranno aggiungere altre opzioni, come un file HTML di base da cui far partire l'esercitazione, un file video (opzionale) e un URL. Il file video deve utilizzare il modello fornito da Marketplace, così come il resto dell'esercitazione, che si può ottenere qui; quando l'utente carica l'esercitazione, il video verrà caricato in un riquadro separato all'interno di IDE. Si noti che le pagine indicate per la pagina iniziale e la pagina video sono relative paths all'interno dei File inclusi, e si può aprire un explorer per scegliere qualcosa di diverso da quelle predefinite con cui GameMaker popola i campi.

La creazione del tutorial richiede l'aggiunta di file inclusi nel progetto del tutorial, che devono essere aggiunti in un modo molto specifico per consentire a IDE di rilevare correttamente quali file devono essere utilizzati per il tutorial. Abbiamo quindi due cartelle principali:

  • TutorialContent: Qui vanno tutti i file HTML del tutorial, i file CSS per lo stile e le immagini necessarie (non le immagini necessarie per le risorse, ma quelle da visualizzare all'interno del tutorial), nonché il modello del video, se necessario. Si possono avere anche delle sottocartelle, se si vuole dividere il tutorial in capitoli, per esempio, ma si deve avere almeno un file HTML di base da usare come pagina iniziale del tutorial.
  • TutorialResources: Questa cartella contiene altre due sottocartelle per le immagini e le risorse audio . Se l'esercitazione richiede all'utente di aggiungere un'immagine sprite (per esempio) e si desidera includerla nell'esercitazione, la risorsa immagine va collocata nella cartella Images. Queste cartelle vengono importate nel progetto del tutorial e quando l'utente apre l'editor sprite e va ad aggiungere un'immagine, l'path predefinito per la finestra di esplorazione sarà in questa posizione, rendendo più facile per l'utente trovare il assets necessario.

Affinché il tutorial funzioni all'interno di IDE , è necessario disporre di questa struttura di cartelle e di una pagina iniziale di base HTML con un file CSS per lo stile. Esiste un certo numero di collegamenti speciali che possono essere usati all'interno dei progetti di tutorial per passare a parti di un video o per aprire il manuale di una sezione o di una funzione specifica; questi collegamenti sono tutti forniti nei modelli ufficiali di tutorial di Marketplace.

IMPORTANTE! Si consiglia vivamente di scaricare i modelli di tutorial dal Marketplace prima di tentare di creare i propri tutorial. Essi contengono esempi di tutte le parti importanti dell'HTML, nonché CSS e immagini per ricreare pulsanti e menu all'interno del tutorial, lasciandovi liberi di preoccuparvi del contenuto piuttosto che di come visualizzarlo.

Una volta impostati i file inclusi e fornite le informazioni richieste, è possibile fare clic su Ok e il tutorial verrà caricato su Marketplace, aprendo un workspace elenco su asset una volta terminato, in modo da poterlo modificare e aggiungere icone, testo, screenshot, ecc. Una volta pubblicata l'esercitazione asset, gli utenti potranno trovarla nella sezione Esercitazioni della pagina iniziale, dove potrà essere scaricata e installata.
