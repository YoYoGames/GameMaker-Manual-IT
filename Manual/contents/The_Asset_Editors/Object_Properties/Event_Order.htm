Ordine dell'evento

Quando si considerano gli eventi in GameMaker, va notato che l'ordine esatto in cui TUTTI gli eventi si verificheranno in ogni fase non può essere chiaramente indicato, semplicemente perché dipende dal funzionamento interno di GameMaker e questo è soggetto a cambiamenti con lo sviluppo del software. Tuttavia ci sono alcuni eventi che si svolgeranno sempre nello stesso ordine.

La prima serie di eventi che accadranno sempre allo stesso modo sono quelli che si verificano quando un room viene inserito per la prima volta. L'ordine in cui i diversi eventi si innescheranno è:

  • Le Variabili Oggetto / Variabili Istanza sono inizializzate (queste sono le variabili che potete definire nell'IDE dell'Editor Oggetti o dell'Editor Stanze). Se hai un object con delle variabili allegate, queste saranno create prima di qualsiasi altra cosa e poi qualsiasi istanza che ha delle variabili allegate sarà definita dopo, così puoi sovrascrivere object le variabili con quelle specifiche dell'istanza nell'editor della stanza editor.
  • Create Event seguito dall' Instance Creation Code di ogni istanza - così come ogni istanza viene creata, eseguirà prima il suo evento Create e poi il suo Instance Creation Code prima di passare alla prossima istanza da creare
  • Evento di avvio del gioco - questo evento viene attivato una volta nel primissimo room del gioco per tutte le istanze inserite in room dall'editor room (nota che chiamando game_restart() questo evento verrà attivato di nuovo.
  • Codice di creazione della stanza - questo è il codice unico scritto nell'editor di Room per quando un room viene inserito per la prima volta
  • Room Start Event di tutte le istanze - uno della categoria "altro" di eventi e si attiverà per tutte le istanze, persistenti o meno

Vale anche la pena notare che si può anche impostare l'ordine in cui le istanze specifiche sono create all'interno dell'editor room stesso, spostandole in alto o in basso nella lista della finestra Instance Layer Properties.

A parte questi eventi specifici, l'unico ordine noto che si verificherà sempre allo stesso modo, non importa cosa, appartiene agli eventi a tre passi e ai diversi eventi di estrazione . Questi rimarranno sempre coerenti, quindi se avete un codice che si basa su una tempistica specifica durante ogni passo del vostro gioco, dovreste usare:

  • Iniziare il passo evento
  • Evento passo (si noti che l'evento passo viene eseguito appena prima che le istanze siano messe nelle loro nuove posizioni)
  • Evento di fine passo

Anche tutti gli eventi per il disegno sono sempre trattati nello stesso ordine come segue (tranne l'evento Window Resize , che viene attivato in modo diverso):

  • Evento pre-disegno
  • Evento di inizio del disegno
  • Evento di sorteggio
  • Evento finale del disegno
  • Evento post sorteggio
  • Evento di inizio del disegno della GUI
  • Disegnare l'evento GUI
  • Disegnare l'evento finale della GUI
