HTTP

L'evento HTTP è quello che viene attivato dalla richiamata di una delle funzioni di http_*(), come ad esempiohttp_post_string(). Esso genera una mappa DS che è esclusiva di questo evento ed è memorizzata nella variabile speciale async_load (si vedano le singole funzioni per gli esempi di codice che spiegano l'uso di questo evento in modo più dettagliato). Questa mappa DS ha la seguente struttura generale:

  • "id": L'id restituito dal comando. Se si invia una serie di richieste a http_, è necessario sapere quale risposta si sta ricevendo; quindi si userà questo valore per confrontarlo con quello memorizzato al momento dell'invio della richiesta per trovare quella giusta.
  • "status": Restituisce un valore inferiore a 0 per un errore, 0 per un successo e 1 se il contenuto è in fase di download.
  • "result": I dati ricevuti (solostring ), oppure path al file scaricato se si è utilizzato http_get_file(). È possibile ottenere questa chiave solo se lo stato è 0, ma ciò dipende dalla piattaforma.
  • "url": L'URL completo richiesto.
  • "http_status": Il codice di stato http grezzo (se disponibile). Restituisce il codice di stato web standard per la maggior parte dei browser, ad esempio: 304 per "Not Modified" (non modificato) o 204 per "No Content" (nessun contenuto), ecc...

Quanto sopra mostra ciò che si ottiene utilizzando la funzione http_post_string() ma ciascuna delle funzioni di http_ può restituire una mappa leggermente diversa, pertanto si consiglia di consultare la voce del manuale di ciascuna funzione per conoscere i dati precisi che vengono restituiti.

NOTA: Poiché async_load crea una mappa DS, queste funzioni sono particolarmente utili se abbinate alle funzioni json_encode() e json_decode() .

Potrebbero esserci anche dati aggiuntivi forniti da questa mappa se sono stati richiesti dei file da scaricare. In questo caso, il valore di "status" sarà 1 e la mappa DS conterrà queste chiavi aggiuntive:

  • "contentLength": È la dimensione del file che il sito server ha detto di aspettarsi di ricevere (può essere -1 se server non restituisce questo dato).
  • "sizeDownloaded": La dimensione dei dati già scaricati.

Si noti che l'evento non verrà attivato per ogni singolo pacchetto di dati ricevuto, ma si aggiornerà in qualsiasi momento durante il download all'interno del gioco principale loop. Si noti inoltre che al momento questa funzionalità è disponibile solo per le normali piattaforme di destinazione Windows.
