L'editore Extension

Un pacchetto di estensione può essere utilizzato per aggiungere una serie di funzioni e costanti aggiuntive a GameMaker Language (GML) e trasferirle facilmente da un progetto all'altro. Le funzioni di Extension possono anche essere collegate a SDK esterni o a codice sorgente scritto in altri linguaggi (come Java per Android, Objective-C per iOS, ecc.) che possono essere utilizzati per espandere le funzionalità del gioco (di solito vengono utilizzati per implementare API di terze parti, come ad esempio per le pubblicità, gli acquisti in-app e altro).

Le estensioni possono essere usate per creare una libreria di funzioni che si usano frequentemente in tutti i progetti; invece di dover importare lo stesso scripts in ogni progetto, si può semplicemente aggiungere un extension asset che contiene una raccolta delle funzioni e delle costanti preferite. Tali funzioni possono anche essere specifiche per una certa piattaforma e si può facilmente scegliere su quali piattaforme extension può essere utilizzato.

Per un esempio passo-passo sulla creazione di un sito extension, vedere: Creare un'estensione

Per disabilitare extensions in base alla piattaforma di destinazione, vedere: Come disabilitare Extensions?

Importazione di estensioni

Normalmente i pacchetti di terze parti extensions provengono dal Marketplace di GameMaker e quelli a cui ci si è abbonati saranno elencati nella finestra Marketplace Library e potranno essere installati da lì. Se non avete ricevuto il pacchetto extension dal Marketplace, potete comunque aggiungerlo a un progetto facendo clic con il pulsante destro del mouse su in qualsiasi punto del Browser delle risorse e selezionando Aggiungi esistente, quindi navigando fino alla posizione dell'estensione per selezionarla.

NOTA Se si dispone di un pacchetto .yymps/.yymp/.gmez, è possibile trascinarlo in IDE per importarlo, oppure andare nel menu "Strumenti" e selezionare "Importa pacchetto locale".

Piattaforme di destinazione

Una volta aggiunto un extension al Browser Asset, è necessario assegnargli un nome e un numero di versione e impostare le piattaforme in cui si desidera esportarlo. Deselezionando le piattaforme selezionate, si può dire a GameMaker di non esportare il extension come parte dell'eseguibile per quelle piattaforme. È inoltre necessario assicurarsi che le proprietà di extension corrispondano alle piattaforme di destinazione selezionate e selezionare obiettivi di esportazione compatibili per i file extension in base a quanto segue:

  • Tutti i target: GML extensions. Questi sono extensions realizzati con il metodo GameMaker Language (GML) e sono compatibili con tutte le piattaforme di destinazione. (Si noti che questo è l'unico tipo extension che il target Opera GX attualmente supporta.)
  • Windows (Standard, UWP, Xbox): È possibile utilizzare i file Windows DLL (Dynamic Link Libraries). Si noti che se è stata selezionata l'opzione "Usa x64 Windows Runtime" nelle Opzioni di gioco di Windows, tutte le DLL per il progetto devono essere compatibili con i 64 bit. Le DLL a 32 bit non funzioneranno con un eseguibile a 64 bit.
  • macOS: Il target macOS può utilizzare i file standard DyLib per extensions.
  • Ubuntu (Linux): Per l'obiettivo Ubuntu è necessario assicurarsi che il proprio extension utilizzi i file .so.
  • HTML5: Queste piattaforme possono utilizzare JavaScript extensions.
  • Android: Questa piattaforma richiede che extension sia scritto in Java o in Android Framework SDK.
  • iOS / tvOS: Extensions per questa piattaforma è scritto in Objective C e richiede un file *.mm e un file *.h o, se si utilizza un framework, un file *.a nativo precompilato.
  • PlayStation: Per la PlayStation i file extensions devono essere PRX, e ogni destinazione richiede un PRX diverso per conformarsi alle specifiche della destinazione (PS3, PS4 o PS5). Se si lavora su una PSVita, il file extension deve essere SUPRX.

NOTA Per alcuni extensions è necessario seguire specifiche convenzioni di denominazione per GameMaker, in modo da identificare correttamente la piattaforma e l'architettura della CPU per cui sono stati progettati. Per ulteriori informazioni, vedere Convenzioni di denominazione.

Poiché un extension può avere diversi file associati, è possibile creare un singolo extension per più piattaforme; in questo caso, è necessario specificare tutte le piattaforme compatibili nell'editor principale di extension (cioè: spuntare la casella di controllo accanto ad esse) e poi nelle proprietà dei singoli file per ogni parte di extension, impostare la piattaforma in cui tali file devono essere esportati.

Per modificare la piattaforma di destinazione di uno specifico file extension o segnaposto, è necessario prima aprire le proprietà di extension (fare doppio clic su e extension nel Browser Asset ) e poi, una volta nelle proprietà di extension, selezionare da destra il file di risorse che si desidera modificare. Decomprimere su Build

L'opzione "Uncompress on build" può essere usata per i file di archivio compressi destinati a una specifica piattaforma di destinazione. Se questa opzione è attivata, il contenuto dell'archivio verrà estratto nella directory Included Files del pacchetto creato, ma solo dopo essere stato copiato sulla piattaforma di destinazione.

Per esempio, se si attiva questa opzione per un file .zip in extension e poi si avvia una compilazione macOS da Windows, l'archivio .zip sarà copiato come tale da Windows a macOS e sarà quindi decompresso durante la compilazione sulla piattaforma di destinazione (che, in questo contesto, è macOS).

Chiamare le funzioni di Extension

Una volta impostato il proprio extension, è possibile utilizzare le funzioni e le costanti per esso proprio come le funzioni e le costanti incorporate utilizzate in GameMaker Language. Se il gioco è multipiattaforma (ad esempio, è progettato per funzionare su Windows e Android target), è necessario utilizzare le costanti e le funzioni del sistema operativo per verificare la piattaforma corrente prima di chiamare, ad esempio, le funzioni specifiche di extension per evitare problemi:

switch(os_type)
{
    case os_android:
        call_android_extension_function(x, y, z);
    break;

    case os_windows:
        call_windows_extension_function(x, y, z);
    break;
}

Si noti che per piattaforme diverse i file extensions possono condividere i nomi delle funzioni, quindi se si ha un extension con file distinti per (ad esempio) far apparire un messaggio su Windows e per Android, entrambi i file extension possono condividere gli stessi nomi di funzione, purché siano entrambi impostati per esportare solo sulle rispettive piattaforme.

Estensioni ufficiali di terze parti

Le versioni precedenti di GameMaker avevano molte funzionalità di terze parti incorporate, ma questo era inefficiente e difficile da mantenere, poiché se una specifica funzionalità veniva modificata dalle terze parti, l'intero IDE e Runtime di GameMaker doveva essere aggiornato. Pertanto, le versioni attuali hanno adottato l'uso dei pacchetti extensions per aggiungere il supporto di terze parti ai giochi. Ciò significa che è possibile aggiungere solo le funzioni extra necessarie e che è possibile adattarle e modificarle per soddisfare le proprie esigenze specifiche.

La maggior parte di questi extensions viene scaricata e installata quando si seleziona una specifica opzione di gioco (come Facebook o Google Mobile Ads, ad esempio), ma è possibile trovare ulteriori informazioni su di essi nella sezione appropriata per ciascuna piattaforma della Knowledge Base di GameMaker:

  • Piattaforme e configurazione diGameMaker
