Tipi di dati della piuma

Feather utilizza data types per fornire un controllo intelligente della sintassi durante la scrittura del codice, assicurando che non si utilizzi l'indirizzo data type sbagliato per una variabile o un parametro di funzione.

Consente inoltre di specificare data types per i parametri e i valori di ritorno delle proprie funzioni di script, utilizzando i commenti JSDoc.

I tag @param e @return JSDoc consentono di specificare uno dei seguenti tipi:

NOTA I tipi JSDoc non fanno distinzione tra maiuscole e minuscole, quindi Id.DsList e id.dslist sono uguali.

NOTA * La colonna Esempi di specificatori elenca alcuni esempi di specificatori, ma non tutti. Ad esempio, potrebbero esserci più tipi possibili nei gruppi Id, Asset e Constant, che non sono elencati qui.

Quando si usa Feather, si può vedere un tipo Any* nel sito IDE, che indica che il tipo dell'identificatore (che può essere una variabile, un parametro, un valore di ritorno, ecc. Quando verrà aggiunto altro codice per quell'identificatore, esso potrà assumere un tipo specifico.

Specificatori

Dopo la base data type viene aggiunto uno specificatore con un punto . per specificare il tipo esatto di dati in quel gruppo.

Tipi come Id.DsList, Asset.GMObject e Constant.Color utilizzano specificatori.

Tipi di collezione

Tipi come Array e Id.DsList, che sono strutture di dati che contengono più valori, possono specificare un singolo data type per tutti i loro contenuti.

Questo viene fatto usando i tipi di collezione, che vengono aggiunti al tipo usando le parentesi angolari <>.

Ad esempio, un array contenente strings sarebbe Array<String>, e un elenco DS, contenente un array, contenente Reali, sarebbe Id.DsList<Array<Real>>.
